# 라이트 노드 개요

**QoreChain 라이트 노드**는 전체 검증자나 아카이브 노드를 실행하지 않고 QoreChain 네트워크를 따르는 경량 클라이언트입니다. 모든 트랜잭션을 다시 재생하는 대신, 블록 헤더를 암호학적으로 검증하고, 위임과 보상을 추적하며, 실시간 네트워크 텔레메트리를 스트리밍합니다 — 이 모든 것이 작고 독립적인 바이너리로 이루어집니다.

라이트 노드를 실행하면 전체 노드의 저장소, 대역폭, 운영 비용 없이 네트워크의 경제에 참여하고 그 상태를 관찰할 수 있습니다.

## 자체 버전 라인

라이트 노드는 **자체 버전 라인 — 현재 v3.1.1**로 제공되며, 이는 **체인 릴리스 버전과는 별개입니다**(체인은 별도의 `v3.x` 트랙에 있습니다). 라이트 노드의 v3.1.1 라인은 `qorechain-core`와 정렬되어 있습니다: 코어의 서명 검증 동작을 보호하고 지속적 통합에서 이를 실행하는 포스트 양자 암호화(PQC) 회귀 테스트 스위트(keygen, sign, verify, 변조 탐지)를 추가합니다.

문서나 릴리스 노트를 읽을 때, 라이트 노드의 버전(v3.1.1)과 체인의 버전을 우연히 동일한 메이저 시리즈를 공유하는 두 개의 별개 숫자로 취급하세요.

## 라이트 노드를 실행하는 이유

- **블록 보상의 일부를 획득하세요.** 활성화되고 등록된 라이트 노드는 아래에 설명된 **3% 라이트 노드 보상 분배**의 대상이 됩니다.
- **체인을 직접 검증하세요.** 노드는 스키핑 라이트 클라이언트로 헤더 검증을 수행하므로, 원격 API를 신뢰하지 않고도 체인 상태에 대한 암호학적 보증을 얻습니다.
- **위임 및 자동 복리.** 여러 검증자에 걸쳐 위임된 스테이크를 관리하고, 가중치별로 분할하며, 보상을 자동으로 복리화합니다.
- **네트워크를 실시간으로 관찰하세요.** 실시간 텔레메트리는 검증자, 합의, 브리지, 토크노믹스를 다룹니다.
- **첫날부터 포스트 양자.** 키와 서명은 Dilithium-5 (ML-DSA-87)을 사용합니다.

## 두 가지 에디션: SX와 UX

라이트 노드는 동일한 코드베이스에서 빌드된 두 가지 에디션으로 제공됩니다. 노드를 운영하려는 방식에 맞는 것을 선택하세요.

| 에디션 | 바이너리 | 대상 | 인터페이스 |
| --- | --- | --- | --- |
| **SX — Server eXperience** | `lightnode-sx` | 헤드리스 서버 배포 | 전체 CLI (데몬 + 관리 명령어) |
| **UX — User eXperience** | `lightnode-ux` | 데스크톱 및 운영자 사용 | 임베디드 웹 대시보드 |

- **SX 에디션**은 완전한 관리 CLI를 갖춘 헤드리스 데몬입니다. 서버, 자동화, 명령줄에서 활동하는 운영자에게 적합한 선택입니다. [SX 에디션](/light-node/sx-edition)을 참조하세요.
- **UX 에디션**은 동일한 데몬을 실행하지만, 브라우저에서 텔레메트리, 위임, 보상을 관찰할 수 있도록 임베디드 웹 대시보드를 추가합니다. [UX 에디션](/light-node/ux-edition)을 참조하세요.

두 에디션 모두 동일한 `config.toml`을 읽고, 동일한 홈 디렉터리(기본값 `~/.qorechain-lightnode`)에 데이터를 저장하며, 동일한 Dilithium-5 키링을 사용합니다.

## 3% 라이트 노드 보상 분배

QoreChain의 수수료 분배는 네트워크 데이터를 제공하는 라이트 노드에 고정된 **3% 분배**를 할당합니다. 이는 프로토콜의 보상 분할의 일부로 온체인에서 시행되며, 프로젝트의 경제 문서에 기록된 것과 동일한 채널입니다 — 전체 37% / 30% / 20% / 10% / 3% 분류(검증자, 소각, 트레저리, 스테이커, 라이트 노드)는 [토크노믹스](/architecture/tokenomics)를 참조하세요.

이 분배의 대상이 되려면 라이트 노드는 하트비트 증명을 통해 **온체인에 등록되고 적극적으로 활성을 증명**해야 합니다. 등록 및 라이선싱은 [등록 및 라이선싱](/light-node/registration-and-licensing)에서 다루며, 분배가 어떻게 획득되고, 복리화되며, 모니터링되는지는 [보상 및 모니터링](/light-node/rewards-and-monitoring)에서 다룹니다.

## 핵심 기능 한눈에 보기

- **스키핑 라이트 클라이언트** — 전체 블록을 다운로드하지 않고 헤더를 검증하여, 콜드 스타트에서도 빠르게 동기화합니다.
- **위임 스테이킹** — 구성 가능한 분할 가중치로 여러 검증자에 걸쳐 스테이킹합니다.
- **자동 복리 보상** — 구성 가능한 간격으로 보상을 청구하고 재위임합니다.
- **평판 인식 리밸런싱** — 더 높은 평판의 검증자로 위임을 자동으로 이동합니다.
- **실시간 텔레메트리** — 검증자, 합의, 브리지, 토크노믹스를 독립적인 간격으로 새로 고칩니다.
- **온체인 등록** — 노드를 보상 대상으로 유지하는 하트비트 활성 증명과 함께.
- **포스트 양자 암호화** — 전반에 걸친 Dilithium-5 (ML-DSA-87) 키와 서명.
- **로컬 전용 모드** — 가동 중인 체인을 가리키기 전에 전체 PQC 스택을 실행하고 노드를 독립 실행합니다.

라이트 노드는 **Apache 2.0** 라이선스로 배포됩니다.
