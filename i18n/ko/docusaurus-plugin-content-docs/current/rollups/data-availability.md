# 데이터 가용성

데이터 가용성(DA)은 롤업 상태의 배후에 있는 거래 데이터가 누구나 읽을 수 있는 어딘가에 게시된다는 보장입니다 — 그래야 누구나 독립적으로 롤업의 상태를 재구성하고 검증할 수 있습니다. RDK는 세 가지 DA 백엔드를 지원합니다.

| 백엔드 | 무엇인가 |
| ------- | ---------- |
| **`native`** | QoreChain 자체 내의 온체인 블롭 저장소 |
| **`celestia`** | 전용 모듈식 DA 계층인 Celestia로 IBC를 통한 데이터 가용성 |
| **`both`** | 중복성을 위해 native와 Celestia를 함께 사용 |

:::caution
데이터 가용성 백엔드는 활발히 발전 중인 RDK의 일부입니다. 아래의 성숙도 관련 설명은 설계 의도로 간주하고, 프로덕션에서 백엔드에 의존하기 전에 **`qorechain-diana`** 테스트넷에서 검증하세요.
:::

---

## Native DA (온체인 블롭 저장소)

Native DA는 롤업 거래 데이터를 QoreChain에 직접 **블롭**으로 저장합니다. 각 블롭은 커밋되고 주소 지정이 가능하여, 정산 배치 배후의 데이터를 온체인에서 검색하고 검증할 수 있습니다.

주요 개념:

* **블롭(Blobs).** 롤업 거래 데이터는 데이터 가용성 블롭으로 게시되며, 각각 롤업 ID와 블롭 인덱스에 연결됩니다.
* **커밋먼트(Commitments).** 각 블롭은 커밋먼트(블롭 데이터의 해시)를 지니므로, 저장자를 신뢰하지 않고도 커밋된 내용과 블롭을 대조하여 검증할 수 있습니다.
* **네임스페이스(Namespaces).** 블롭은 롤업별 네임스페이스를 지닐 수 있어, 공유 저장소 내에서 각 롤업의 데이터가 논리적으로 분리되도록 유지합니다.
* **보존 및 가지치기(Retention and pruning).** Native 블롭은 제한된 기간 동안 보존된 후 온체인 저장소를 지속 가능하게 유지하기 위해 가지치기됩니다. 가지치기 후에는 원시 블롭 데이터는 제거되지만 커밋먼트 메타데이터는 보존되므로, 바이트가 더 이상 온체인에 저장되지 않더라도 과거 커밋먼트는 검증 가능한 상태로 남습니다.

정확한 최대 블롭 크기와 보존 기간은 실시간 모듈 파라미터에 의해 관리됩니다. 특정 한도를 기준으로 설계하기 전에 이를 조회하세요:

```bash
qorechaind query rdk config
```

Native DA는 가장 단순한 옵션입니다 — 모든 것을 QoreChain 내부에 두어, 호스트 체인의 보안과 양자내성 암호화를 상속하지만, 호스트 체인의 저장소를 소비하는 비용이 따릅니다.

## Celestia DA (Celestia로의 IBC)

`celestia` 백엔드는 전용 모듈식 DA 네트워크인 **Celestia**로 IBC를 통해 데이터 가용성을 게시합니다. 이는 QoreChain에서 블롭 저장을 분리하여 목적에 맞게 구축된 DA 계층으로 옮기면서도, 정산은 여전히 QoreChain에 고정합니다.

:::note
Celestia DA는 성숙해 가는 통합입니다. 현재 릴리스에서는 아직 프로덕션 수준으로 강화되지 않은 것으로 간주해야 합니다 — 테스트넷에서 동작을 검증하고, 오늘 정착된 보장이 필요한 경우 `native` 또는 `both`를 선호하세요.
:::

---

## Both (중복성)

`both` 백엔드는 **native와 Celestia에 함께** 기록하여, 온체인 저장소와 외부 모듈식 DA 계층에 걸친 중복성을 제공합니다. 가장 넓은 가용성 표면을 원하고 데이터를 두 곳에 저장하는 비용을 감수할 의향이 있다면 `both`를 선택하세요.

Celestia 경로가 아직 성숙해 가는 중이므로, `both`는 오늘 두 개의 완전히 독립적인 복사본이 정착된다는 보장이라기보다는 중복성이 진행 중인 native로 간주하세요. 현재 동작은 테스트넷에서 확인하세요.

---

## 백엔드 선택하기

| 원하는 것이... | 선택 |
| -------------- | ------ |
| QoreChain 보안을 상속하는 가장 단순한 완전 온체인 옵션 | **`native`** |
| 전용 모듈식 계층으로 DA를 분리(성숙 중) | **`celestia`** |
| 중복성을 갖춘 최대 가용성 표면(성숙 중) | **`both`** |

DA가 더 넓은 정산 구도에 어떻게 들어맞는지는 **[롤업 개요](/rollups/overview)**를 참조하세요. 더 낮은 수준의 모듈 레퍼런스는 **[Rollup Development Kit](/architecture/rollup-development-kit)** 페이지를 참조하세요.
