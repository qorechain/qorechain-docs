# 주요 기능

다음 표는 QoreChain의 모든 주요 기능을 도입된 릴리스 단계별로 정리한 것입니다. 현재 체인 버전은 **v3.1.80**이며, 메인넷(`qorechain-vladi`, EVM 체인 ID 9801)은 2026년 6월 7일부터 가동 중이고 병렬 테스트넷(`qorechain-diana`, EVM 체인 ID 9800)이 함께 운영됩니다.

| 기능                       | 도입 버전           | 설명                                                                                                                                                                                                                                                                                                                                                                                           |
| -------------------------- | ------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| PQC 하이브리드 서명 (기본값으로 필수) | v1.1.0 (보안)   | 모든 cosmos 경로 트랜잭션에 대한 이중 서명: 고전 secp256k1 (ECDSA) 서명과 ML-DSA-87 (Dilithium-5)을 결합합니다. v3.1.71부터 네트워크 기본값은 **required**입니다(`hybrid_signature_mode = required`, `allow_classical_fallback = false`) — 고전 전용 cosmos 트랜잭션은 거부되며, 제네시스 gentx와 PQC 키 등록/마이그레이션 트랜잭션만 면제됩니다. EVM 트랜잭션은 별도의 `eth_secp256k1` 경로를 사용하며 영향받지 않습니다. 세 가지 거버넌스 제어 시행 모드(disabled / optional / required)는 계속 사용할 수 있습니다. TX 확장 자동 등록을 통한 매끄러운 지갑 온보딩.                                                                              |
| SHAKE-256 기본 해시        | v1.1.0 (보안)       | SHA-3 계열 확장 가능 출력 함수(XOF). v3.1.73부터 SHAKE-256(`qorehash` 패키지를 통해)이 **기본 애플리케이션 해시**이며, PQC 기준선(Dilithium-5 + ML-KEM-1024 + SHAKE-256)을 완성합니다. 가변 길이 해싱, 고정 32바이트 출력, Merkle 내부 노드 연결, 도메인 분리 해싱을 제공합니다 — 모두 FFI 의존성 없이 순수 Go로 구현되었습니다.                                                                                                                 |
| TEE 및 FL 인터페이스       | v1.1.0 (보안)       | 신뢰 실행 환경 어테스테이션(SGX, TDX, SEV-SNP, ARM CCA) 및 연합 학습 조정(FedAvg, FedProx, SCAFFOLD 집계 방법)을 위한 프로덕션급 인터페이스 사양입니다. 암호학적 보장을 갖춘 하드웨어 엔클레이브 AI 추론과 프라이버시 보존 분산 모델 학습을 가능하게 합니다.                                                                |
| 온체인 RL 합의 (PRISM) | v1.0.0 (제네시스) | Go 네이티브 고정 소수점 MLP(73,733개 파라미터)가 블록 생명주기에서 직접 PPO 추론을 실행합니다. PRISM 최적화 계층은 외부 오라클 없이 블록 시간, 가스 한도, 검증자 풀 가중치를 동적으로 튜닝합니다. 결정론적 Taylor 급수 수학은 모든 검증자에서 동일한 결과를 보장합니다. 네 가지 운영 모드: shadow, conservative, autonomous, paused. 안전을 위한 회로 차단기 보호. |
| 트리플 풀 복합 PoS         | v1.0.0 (제네시스)   | 검증자는 QoreChain 합의 엔진에서 1,000블록마다 RPoS(평판 가중), DPoS(위임 가중), PoS(표준) 풀로 분류됩니다. 풀 가중 추첨은 순수 스테이크 지배를 넘어 블록 생성을 다양화합니다. 사용자 지정 본딩 커브는 자기 본딩 스테이크, 충성도 기간, 평판 품질, 프로토콜 단계를 반영합니다.                            |
| QDRW 거버넌스              | v1.0.0 (제네시스)   | 평판 가중을 갖춘 이차 위임(Quadratic Delegation with Reputation Weighting). 투표력은 시그모이드 평판 승수로 감쇠된 제곱근 함수를 사용하여, 고래의 장악을 방지하면서 장기적인 정직한 참여를 보상합니다. 100배의 스테이크 우위는 약 10배의 투표력을 산출합니다. xQORE 보유는 투표 가중치를 두 배로 늘립니다.                                                                               |
| 소각 엔진                  | v1.0.0 (제네시스)   | 10개의 별개 소각 채널: 트랜잭션 수수료, 거버넌스 페널티, 슬래싱, 브리지 수수료, 스팸 억제, 에포크 초과분, 수동 소각, 컨트랙트 콜백, 크로스 VM 수수료, 롤업 생성 소각. 수집된 수수료는 **검증자에게 37%, 영구 소각 30%, 트레저리에 20%, 스테이커에게 10%, 라이트 노드에 3%**로 분할됩니다.                                                            |
| xQORE 스테이킹             | v1.0.0 (제네시스)   | QOR를 잠가 1:1 비율로 xQORE를 발행하여 QDRW 투표에서 거버넌스 가중치를 두 배로 늘립니다. 단계적 출구 페널티(30일 미만 50%, 30~90일 35%, 90~180일 15%, 180일 후 0%)는 PvP 리베이스를 통해 남은 보유자에게 재분배됩니다 — 신념을 보상하고 단기 자본에 페널티를 부과합니다.                                                                                      |
| 고정 공급량 발행          | v1.0.0 (제네시스)   | 총 공급량은 4,500,000,000 QOR로 고정되며(TGE 시 80,000,000 소각), 유한한 스테이킹 보상 예산은 590,000,000 QOR입니다: 1년차 8~12% APY(127,500,000 QOR), 2년차 6~10% APY(106,250,000 QOR), 3~4년차 5~8% APY(연간 85,000,000 QOR), 5년차 이후 거버넌스 결정(~186,000,000 QOR 잔여). 소각 엔진과 결합하여 QOR는 트랜잭션 볼륨이 증가함에 따라 순 디플레이션 행동으로 수렴합니다. |
| EVM 런타임                 | v1.0.0 (제네시스)   | EIP-1559 가스 가격 책정, 포트 8545의 JSON-RPC(`eth_`, `web3_`, `net_`, `txpool_`, `qor_` 네임스페이스), 표준 도구 지원(Hardhat, Foundry, Remix)을 갖춘 완전한 Ethereum 호환성. 기존 Ethereum 워크플로를 사용하여 Solidity 컨트랙트를 배포하고 상호작용하세요.                                                                                                                    |
| CosmWasm 런타임            | v1.0.0 (제네시스)   | Rust 기반 컨트랙트를 위한 WebAssembly 스마트 컨트랙트 엔진. 전체 생명주기 지원: instantiate, execute, query, migrate. 컨트랙트는 결정론적 실행을 갖춘 샌드박스 Wasm 환경에서 실행됩니다.                                                                                                                                                                                       |
| SVM 런타임                 | v1.0.0 (제네시스)   | Rust 기반 실행기를 통한 BPF 프로그램 배포 및 실행. 포트 8899의 Solana 호환 JSON-RPC 서버는 `getAccountInfo`, `getBalance`, `getSlot` 등을 지원합니다. 기존 Solana 클라이언트와 도구가 수정 없이 작동합니다.                                                                                                                                                                    |
| 크로스 VM 브리지           | v1.0.0 (제네시스)   | 세 가지 VM 전반의 매끄러운 상호운용성. EVM 컨트랙트는 프리컴파일을 통해 CosmWasm을 호출하고, CosmWasm 컨트랙트는 사용자 지정 메시지를 통해 EVM을 호출하며, SVM 프로그램은 비동기 이벤트 기반 브리징을 통해 참여합니다. 단일 체인 내에서 동기식 EVM-CosmWasm 호출과 비동기식 SVM 메시징이 가능합니다.                                                                                  |
| 크로스 체인 연결성         | v1.2.0 (상호운용)   | 8개의 IBC 채널(Cosmos Hub, Osmosis, Noble, Celestia, Stride, Akash, Babylon, Injective)과 더불어 **36개 외부 체인을 포괄하는 37개 QCB 구성**(QoreChain 자체를 네이티브 루프백으로 포함). PQC 서명 검증자 어테스테이션, 체인별 확인 깊이, 회로 차단기 볼륨 상한. 현재 테스트넷 / 보류 상태 — 아직 프로덕션이 아닙니다.                               |
| BTC 리스테이킹             | v1.2.0 (상호운용)   | Bitcoin 최종성 보장을 위한 Babylon Protocol 통합. 검증자는 BTC 스테이킹 포지션을 등록합니다(최소 100,000 사토시). QoreChain 에포크 상태 루트는 IBC 릴레이 Babylon 에포크를 통해 주기적으로 Bitcoin에 체크포인트되어, BTC 해시레이트로 뒷받침되는 보조 최종성 계층을 제공합니다.                                                                                           |
| 계정 추상화                | v1.2.0 (상호운용)   | 프로토콜 계층의 프로그래밍 가능한 스마트 계정(ERC-4337과 유사). 세 가지 계정 유형: 멀티시그, 소셜 복구, 세션 기반. 세분화된 권한과 만료를 갖춘 세션 키, 계정별 일일 및 트랜잭션별 지출 규칙, 범위 지정 denom 허용 목록, 합의 시 자동 규칙 시행.                                                                                                      |
| MEV 보호                   | v1.2.0 (상호운용)   | 암호화 멤풀을 위한 FairBlock 임계값 신원 기반 암호화(tIBE) 프레임워크. 트랜잭션은 포함 이후까지 블록 제안자에게 암호학적으로 불투명하여, 프런트러닝과 샌드위치 공격을 제거합니다. FairBlockDecorator ante 핸들러는 연결되어 준비되어 있으며, tIBE 임계값 복호화는 키 의식 배포 후에 활성화됩니다.                                           |
| 가스 추상화                | v1.2.0 (상호운용)   | 다중 토큰 가스 결제는 트랜잭션 수수료를 위해 네이티브 QOR를 보유해야 하는 요구사항을 제거합니다. 사용자는 허용된 IBC 전송 토큰으로 결제할 수 있습니다: ibc/USDC는 1:1 비율, ibc/ATOM은 10:1 비율. GasAbstractionDecorator는 표준 수수료 차감 전에 비네이티브 수수료 denom을 검증하고 변환합니다.                                                                                      |
| 5레인 우선순위 지정        | v1.2.0 (상호운용)   | 블록 공간은 다섯 개의 우선순위 레인으로 정적 분할됩니다: PQC(우선순위 100, 공간 15%), MEV(90, 20%), AI(80, 15%), Default(50, 40%), Free(10, 10%). 보안에 중요한 트랜잭션은 고볼륨 표준 트래픽에 의해 결코 밀려나지 않습니다.                                                                                                                                                                     |
| 온체인 유동성 (AMM)        | v1.2.0 (상호운용)   | 네이티브 자동화된 마켓 메이커(`x/amm`)는 프로토콜 계층에서 온체인 유동성 풀과 스왑을 제공합니다.                                                                                                                                                                                                                                                                                                                                                     |
| RDK 롤업                   | v1.3.0 (롤업)       | 네 가지 정산 패러다임(optimistic, zk, based, sovereign), 다섯 가지 프리셋 프로필(defi, gaming, nft, enterprise, custom), SHA-256 blob 저장 및 자동 가지치기를 갖춘 네이티브 DA 라우터, 구성 가능한 생성 소각률을 갖춘 뱅크 에스크로 생명주기, EndBlocker 자동 최종화, 합의 모듈을 통한 PRISM 지원 구성을 갖춘 롤업 개발 키트. 롤업 기능은 호스트 체인 프레임워크로 제공됩니다.                          |
| 체인 라이선싱              | v1.3.0 (롤업)       | `x/license` 모듈은 프로토콜 네이티브 체인 라이선싱을 제공합니다.                                                                                                                                                                                                                                                                                                                              |

## 버전 기록

<details>

<summary>v1.0.0 — 제네시스 릴리스</summary>

포스트 양자 암호화(Dilithium-5, ML-KEM-1024), PRISM 온체인 강화 학습 합의 계층, 크로스 VM 메시징을 갖춘 트리플 VM 런타임(EVM, CosmWasm, SVM), 고정 공급량 토크노믹스 엔진(소각, xQORE, 유한 발행 예산), 트리플 풀 복합 PoS 검증자 선택, QDRW 이차 거버넌스, AI 트랜잭션 처리 파이프라인으로 핵심 프로토콜을 확립했습니다.

</details>

<details>

<summary>v1.1.0 — 보안 강화 릴리스</summary>

고전 secp256k1 (ECDSA) 서명과 ML-DSA-87을 결합하는 하이브리드 서명 아키텍처(세 가지 거버넌스 제어 시행 모드 포함), 향후 Merkle 트리 교체를 위한 SHAKE-256 포스트 양자 해시 기반, TEE 어테스테이션(SGX, TDX, SEV-SNP, ARM CCA) 및 연합 학습 조정(FedAvg, FedProx, SCAFFOLD)을 위한 프로덕션급 인터페이스 사양을 도입했습니다.

</details>

<details>

<summary>v1.2.0 — 상호운용성 및 UX 릴리스</summary>

크로스 체인 연결성(8개 IBC 채널 + 36개 외부 체인을 포괄하는 37개 QCB 구성, 현재 테스트넷/보류), Babylon Protocol을 통한 BTC 리스테이킹, 세션 키와 소셜 복구를 갖춘 스마트 계정 추상화, FairBlock MEV 보호 프레임워크, 다중 토큰 가스 추상화, 온체인 유동성(`x/amm`), 5레인 블록 공간 우선순위 지정을 추가했습니다.

</details>

<details>

<summary>v1.3.0 — 롤업 생태계 릴리스</summary>

네 가지 정산 패러다임(optimistic, zk, based, sovereign), 다섯 가지 프리셋 배포 프로필(defi, gaming, nft, enterprise, custom), 네이티브 DA 라우터, 뱅크 에스크로 생명주기 관리, EndBlocker 기반 자동 최종화, PRISM 지원 롤업 구성, 체인 라이선싱(`x/license`)을 갖춘 롤업 개발 키트를 출시했습니다. 자동 사이드체인 등록 및 상태 앵커링을 위한 멀티레이어 아키텍처 모듈과의 심층 통합.

</details>
