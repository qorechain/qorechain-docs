# 버전 히스토리

QoreChain의 공개 버전 히스토리입니다. 최신 릴리스는 **v3.1.82**로, 메인넷 **`qorechain-vladi`**(EVM 체인 ID **9801**, 2026년 6월 7일부터 가동 중)에서 실행되고 있습니다. 테스트넷 **`qorechain-diana`**(EVM 체인 ID **9800**)는 프리릴리스 빌드를 추적합니다.

:::note
아래 항목은 상위 수준의 기능 요약입니다. 이전 `v1.x` 항목은 메인넷 이전의 테스트넷 릴리스 라인에 대한 역사적 기록으로 보존되어 있습니다.
:::

---

## v3.1.82 — SVM 네이티브 QOR 가동 + 통합사 지원 (현재 메인넷 릴리스)

**릴리스 초점:** 두 네트워크 모두에서 가동 중인 SVM 네이티브 QOR 통합과 거래소 또는 통합사가 연동에 필요한 모든 것.

* **세 인터페이스 모두에서 통합 네이티브 QOR 잔액 가동** — SVM 통합(v3.1.81)이 메인넷과 테스트넷에서 정상 가동 중임이 확인되었습니다. 동일한 계정이 하나의 잔액을 보유하며, Cosmos에서는 `uqor`(6자리 소수점), EVM에서는 wei 방식의 18자리 소수점, Solana 호환 인터페이스에서는 lamports(9자리 소수점, 1 uqor = 1,000 lamports)로 표시됩니다.
* **검증된 공개 엔드포인트** — 두 네트워크 모두에서 합의 RPC, REST, EVM JSON-RPC, SVM JSON-RPC용 공개 HTTPS 엔드포인트와 공개 [블록 탐색기](https://explore.qore.network)를 제공합니다. [네트워크](/appendix/networks)를 참조하세요.
* **다운로드** — 버전별 노드 바이너리 번들, 메인넷 제네시스, 최신 체인 데이터 스냅샷(SHA-256 체크섬 포함)이 [download.qore.host](https://download.qore.host)에 게시되어 있습니다.
* **클라이언트 스택 전반의 결정적 포스트 양자 서명** — `@qorechain/pqc` 0.1.1은 6개 언어 바인딩 모두에서 ML-DSA-87을 결정적으로(FIPS-204 §3.4) 서명하며, 이는 체인이 수용하는 방식과 일치합니다. `@qorechain/wallet-adapter` 0.1.2는 이를 기반으로 하이브리드 트랜잭션 서명을 지원합니다.
* **통합사 가이드** — 세 인터페이스 전반의 입금, 출금, 노드 운영을 다루는 새로운 [거래소 및 통합사 가이드](/developer-guide/exchange-integration)가 추가되었습니다.

## v3.1.81 — SVM 네이티브 QOR 통합

**릴리스 초점:** Solana 호환 인터페이스에서 네이티브 QOR을 일급 자산으로 지원.

* **SVM의 네이티브 QOR** — SVM 런타임이 이제 별도의 SVM 전용 잔액을 추적하는 대신 계정의 네이티브 QOR 잔액을 직접(lamports 단위로) 표시합니다. `getBalance`와 `getSignaturesForAddress`가 네이티브 자금에 대해 작동하며, System Program 전송이 네이티브 QOR을 이동시킵니다.
* **SVM 주소 매핑** — 계정의 SVM 주소는 20바이트 계정 바이트에서 파생되므로(32바이트로 오른쪽 패딩 후 base58 인코딩), 하나의 키에 대한 Cosmos, EVM, SVM 주소가 동일한 자금을 가리킵니다.

## v3.1.80 — 멀티레이어 스테이트 앵커 쿼리

**릴리스 초점:** 롤업을 위한 읽기 가능하고 오프라인 검증 가능한 정산 앵커.

* **앵커 읽기 쿼리** — `x/multilayer` 쿼리 서비스가 이제 `Anchor`(레이어의 최신 스테이트 앵커)와 `Anchors`(레이어의 앵커 히스토리)를 노출하므로, 클라이언트가 레이어의 정산 앵커를 조회하고 독립적으로 검증할 수 있습니다.
* **멀티레이어용 REST 게이트웨이** — 모든 멀티레이어 쿼리(`params`, `layers`, `layers/{layer_id}`, `anchor/{layer_id}`, `anchors/{layer_id}`, `routing-stats`)를 이제 gRPC뿐 아니라 REST로도 이용할 수 있습니다.
* **양자 안전 정산 영수증 지원 확보** — 각 앵커는 정규 필드에 대한 **ML-DSA-87 (Dilithium-5)** 서명을 포함하여, Rollup Development Kit의 오프라인 정산 영수증 검증을 위한 온체인 기반을 제공합니다.

## v3.1.79 — 브리지 네트워크용 밸리데이터 자동 프로비저닝

**릴리스 초점:** 라이선스를 보유한 밸리데이터를 위한 연결된 네트워크 턴키 참여.

* **네트워크 드라이버 프레임워크** — 선언형 드라이버 프레임워크를 통해, 해당 `validator_<chain>`(또는 `qcb_bridge`) 라이선스를 보유한 QoreChain 밸리데이터는 라이선스가 활성화된 이후에만 QoreChain 오케스트레이션 하에서 동일한 노드에 해당 외부 네트워크 클라이언트가 프로비저닝, 구성, 실행되도록 할 수 있습니다.
* **37개 브리지 네트워크 전체용 드라이버** — 참여 모델(퍼미션리스 밸리데이터, 상한/선출/승인, L2 풀 노드, 논스테이킹/트러스트 리스트 역할)별로 분류된 모든 연결 네트워크를 지원합니다. 외부 네트워크 스테이크 및 서명 키는 네트워크별로 운영자가 제공해야 하며, QoreChain은 프레임워크와 강제되는 라이선스 게이트를 제공합니다.

## v3.1.78 — 배포 전 준비

**릴리스 초점:** 지갑, 브리지, IBC, 라이선싱이 출시 시점부터 모두 작동 — 배포 후 거버넌스 불필요.

* **트러스트리스 배포 후 브리지 활성화** — `bridge_admin` 키(또는 `qcb_bridge` 라이선스 보유자)가 단일 서명 트랜잭션(`tx bridge update-chain-config` / `set-verifier-bootstrap`)으로 연결된 체인의 브리지를 활성화할 수 있으며 — 컨트랙트 주소, 컨펌 수, 아키텍처, 상태, 활성 검증자, 검증자 트러스트 루트를 설정 — 거버넌스 제안이나 체인 업그레이드가 필요 없습니다.
* **밸리데이터 네트워크 라이선스 게이트** — 오케스트레이터가 이제 외부 네트워크 클라이언트를 시작하기 전에 `validator_<chain>` / `qcb_bridge` 라이선스를 강제(페일 클로즈드)합니다.
* **지갑 통합 패키지** — `@qorechain/wallet-adapter`와 `@qorechain/connect`가 npm(v0.1.0)에 게시되어, 원콜 MetaMask 네트워크 등록(EIP-3085, EVM 레일에서 **18자리 소수점** 네이티브 QOR)과 Keplr 가스 가격 구성이 추가되었습니다.
* **IBC 턴키 릴레이어** — 8개 IBC 상대 체인을 위한 즉시 실행 가능한 릴레이어 구성 및 채널 부트스트랩 도구를 제공하여, 별도 설정 없이 배포 후 채널이 열립니다.

## v3.1.77 — 브리지 및 소각 REST 엔드포인트

**릴리스 초점:** 크로스체인 및 공급 모듈에 대한 읽기 전용 REST 액세스.

* **브리지 REST 엔드포인트** — 브리지 모듈용 읽기 전용 HTTP 쿼리 엔드포인트로, gRPC뿐 아니라 표준 REST로도 브리지 상태를 노출합니다.
* **소각 REST 엔드포인트** — 소각 모듈용 읽기 전용 HTTP 쿼리 엔드포인트로, 수수료 분배 및 공급 데이터를 표준 REST로 쿼리할 수 있습니다.

## v3.1.76 — SVM 툴체인 현대화

**릴리스 초점:** Solana Virtual Machine 호환성 갱신.

* **최신 툴체인 프로그램 지원** — SVM 실행이 현대화되어 최신 Solana 툴체인으로 빌드된 프로그램이 QoreChain SVM 런타임에서 실행됩니다.

## v3.1.75 — 기본 활성화된 SVM JSON-RPC

**릴리스 초점:** 즉시 사용 가능한 Solana 호환 RPC.

* **Solana 호환 JSON-RPC** — SVM JSON-RPC 서버가 이제 기본 활성화(포트 **8899**)되어 노드와 함께 자동으로 시작되며, SVM 도구를 위한 Solana 호환 RPC 인터페이스를 제공합니다.

## v3.1.74 — 롤업 프로필 프리셋

**릴리스 초점:** Rollup Development Kit의 사용성 및 정산.

* **프로필 프리셋 적용** — 롤업 생성 시 이제 선택한 프로필의 프리셋(DeFi, 게임, NFT, 엔터프라이즈, 완전 커스텀)이 적용되어, 새 롤업이 해당 사용 사례에 맞는 합리적인 기본값을 상속합니다.
* **낙관적 정산** — 낙관적 정산 경로(배치 제출 및 챌린지)가 엔드투엔드로 운영됩니다.

## v3.1.73 — 포스트 양자 해시 베이스라인

**릴리스 초점:** 기본 포스트 양자 암호화 베이스라인 완성.

* **SHAKE-256 기본 해시** — SHAKE-256(SHA-3 계열)이 기본 애플리케이션 해시로 채택되어, **ML-DSA-87 (Dilithium-5)** 서명, **ML-KEM-1024** 키 캡슐화, **SHAKE-256** 해싱으로 구성된 기본 포스트 양자 베이스라인이 완성되었습니다.

## v3.1.72 — 안정성 및 유지보수

**릴리스 초점:** 정기적인 안정성 및 빌드 파이프라인 유지보수.

* **안정성 개선** — 외부에서 관찰 가능한 동작 변경 없이 내부 안정성, 의존성, 빌드 파이프라인 유지보수를 수행했습니다.

## v3.1.71 — 기본 강제화된 PQC 하이브리드 서명

**릴리스 초점:** Cosmos 트랜잭션 경로에서 기본으로 활성화된 포스트 양자 보안.

* **기본으로 요구되는 하이브리드 서명** — 이제 Cosmos 트랜잭션 경로에서 포스트 양자 하이브리드 서명이 기본으로 강제됩니다. 각 트랜잭션은 클래식 **secp256k1** 서명과 함께 포스트 양자 **ML-DSA-87 (Dilithium-5)** 서명을 포함합니다.
* **거버넌스 제어 강제화** — 강제화 모드는 거버넌스로 제어되며, 기본값은 **required**로 설정되어 있습니다.

## v3.1.70 — 프로덕션 강화

**릴리스 초점:** 가동 중인 메인넷을 위한 프로덕션 강화 및 합의 최적화.

* **PRISM 합의 최적화** — 서킷 브레이커 안전 제어와 함께, 실시간 네트워크 조건에서의 적응형 파라미터 튜닝을 위한 PRISM 강화학습 최적화 레이어의 지속적인 개선.
* **성능 및 안정성** — 밸리데이터와 풀 노드 전반의 처리량, 지연 시간, 리소스 사용량 개선.
* **운영 도구** — 메인넷 운영자를 위한 모니터링, 쿼리, 노드 운영 편의성 개선.
* **토크노믹스 v2.1 정렬** — 수수료 분배 및 발행 메커니즘을 고정 공급·유한 발행 경제 모델에 맞게 정렬했습니다.

## v3.0.0 — 메인넷 제네시스

**릴리스 초점:** 메인넷 출시 및 토큰 생성 이벤트.

* **메인넷 제네시스** — QoreChain 메인넷(`qorechain-vladi`, EVM 체인 ID 9801)이 **2026년 6월 7일**에 출시되었으며, 토큰 생성 이벤트(TGE)가 제네시스에서 이루어졌습니다.
* **5방향 수수료 분배** — 밸리데이터, 소각, 트레저리, 스테이커, 라이트 노드 전반에 걸친 프로토콜 수수료 분배(**37 / 30 / 20 / 10 / 3**)로, 라이트 노드 전용 몫이 추가되었습니다.
* **온체인 AMM** — 온체인 유동성 풀과 스왑을 위한 네이티브 자동화 마켓 메이커 모듈(`x/amm`).
* **체인 라이선싱** — 프로토콜 권한 등록 및 관리를 위한 온체인 라이선스 모듈(`x/license`).
* **강화된 정산 패러다임** — RDK 정산 모드가 낙관적(optimistic), zk, 기반(based), 소버린(sovereign)으로 확정되었습니다.

## v1.4.0 — 메인넷 이전 확장

**릴리스 초점:** 메인넷을 앞둔 크로스체인 커버리지 확대 및 릴리스 후보 안정화.

* **확장된 크로스체인 커버리지** — 더 넓은 범위의 외부 네트워크에 대한 추가 IBC 및 브리지 연결.
* **라이트 노드 참여** — 라이트 노드와 그 수수료 분배 보상을 위한 기반 도입.
* **릴리스 후보 강화** — 메인넷 제네시스를 준비하며 모든 핵심 모듈에 대한 광범위한 테스트, 감사, 안정화 수행.

## v1.3.0 — Rollup Development Kit

**릴리스 초점:** 소버린 및 공유 보안 롤업 배포를 위한 네이티브 롤업 인프라.

* **x/rdk 모듈** — 네 가지 정산 패러다임(낙관적, zk, 기반, 소버린)을 갖춘 완전한 Rollup Development Kit
* **5개 프리셋 프로필** — DeFi, 게임, NFT, 엔터프라이즈, 완전 커스텀 사용 사례를 위한 사전 구성된 롤업 템플릿
* **네이티브 데이터 가용성** — 블롭 스토리지, 보존 관리, 프루닝 라이프사이클을 갖춘 온체인 DA 레이어
* **EndBlocker 자동 확정** — 챌린지 기간 만료 시 운영자 개입 없이 자동으로 배치가 확정됨
* **AI 지원 프로필 선택** — 의도한 사용 사례를 기반으로 최적의 롤업 구성을 추천하는 `suggest-profile` 쿼리
* **멀티레이어 통합** — 롤업이 멀티레이어 아키텍처의 레이어로 등록되어 라우팅, 앵커링, 챌린지 메커니즘을 상속함
* **뱅크 에스크로 라이프사이클** — 운영자 스테이크는 롤업 운영 기간 동안 에스크로에 보관되며, 정상 종료 시 반환되거나 슬래싱 시 몰수됨

## v1.2.0 — IBC 및 브리지

## v1.1.0 — PQC 하이브리드 서명

**릴리스 초점:** 포스트 양자 암호화 보안 및 알고리즘 민첩성.

* **이중 secp256k1(ECDSA) + ML-DSA-87 서명** — 모든 트랜잭션이 클래식 서명과 포스트 양자 서명을 모두 포함하며, AnteHandler 체인에서 검증됨
* **3가지 강제화 모드** — 구성 가능한 하이브리드 서명 강제화: off(모드 0), permissive(모드 1, PQC 선택), mandatory(모드 2, PQC 필수)
* **자동 등록** — PQC 공개 키가 첫 하이브리드 트랜잭션에서 자동으로 등록되어 별도의 등록 단계가 필요 없음
* **SHAKE-256 해시 기반** — 모든 PQC 관련 해싱 연산이 양자 내성 주소 파생을 위해 SHAKE-256(SHA-3 계열)을 사용
* **TEE 증명 인터페이스** — PQC 키 생성 무결성 증명을 위한 신뢰 실행 환경(Trusted Execution Environment) 증명 지원
* **알고리즘 민첩성 프레임워크** — 체인 업그레이드 없이 거버넌스를 통해 향후 PQC 알고리즘을 추가할 수 있는 플러그형 알고리즘 레지스트리

## v1.0.0 — 제네시스 (토크노믹스 엔진)

**릴리스 초점:** 완전한 토크노믹스, 멀티 VM 실행, AI 지원 운영을 갖춘 초기 프로토콜 출시.

* **x/burn 모듈** — 밸리데이터, 소각, 트레저리, 스테이커에 걸친 4방향 분배를 갖춘 다중 채널 수수료 소각 메커니즘
* **x/xqore 모듈** — 단계별 조기 언락 페널티와 PvP 리베이스 재분배를 갖춘 거버넌스 스테이킹 파생 상품
* **x/inflation 모듈** — 유한 발행 경제 모델이 관장하는 연간 감쇠 기반의 에포크별 발행
* **PRISM 합의 레이어** — 서킷 브레이커 안전 제어를 갖춘 동적 체인 파라미터 튜닝을 위한 강화학습 최적화(PPO)
* **트리플 풀 CPoS** — 평판 점수로 가중치가 부여된 Emerald, Sapphire, Ruby 밸리데이터 풀을 갖춘 분류형 지분 증명(Classified Proof-of-Stake)
* **QDRW 거버넌스** — 풀 전반의 보상 분배를 거버넌스 승인 하에 조정할 수 있는 동적 보상 가중치(Dynamic Reward Weighting) 시스템
* **EVM + CosmWasm + SVM 런타임** — 세 가지 동시 실행 환경: QoreChain EVM Engine, CosmWasm 스마트 컨트랙트, Solana Virtual Machine
* **크로스 VM 브리지** — 단일 블록 내에서 EVM, CosmWasm, SVM 런타임 간 메시지 전달 및 자산 이동
* **포스트 양자 암호화** — 고성능 PQC 라이브러리가 뒷받침하는 양자 내성 서명
* **QCAI** — 사기 탐지, 수수료 추정, 네트워크 최적화를 위한 온체인 휴리스틱 분석과 선택적 오프체인 사이드카
* **컨테이너화 배포** — 사이드카 서비스와 블록 인덱서를 포함한 완전한 멀티 밸리데이터 테스트넷 배포
* **블록 인덱서** — 과거 데이터 쿼리 및 분석을 위한 영구 저장소를 갖춘 블록 리스너
