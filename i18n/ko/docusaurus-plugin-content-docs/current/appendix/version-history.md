# 버전 히스토리

QoreChain의 공개 버전 히스토리입니다. 최신 릴리스는 **v3.1.80**이며, 메인넷 **`qorechain-vladi`**(EVM 체인 ID **9801**, 2026년 6월 7일부터 가동)에서 실행되고 있습니다. 테스트넷 **`qorechain-diana`**(EVM 체인 ID **9800**)는 사전 릴리스 빌드를 추적합니다.

:::note
아래 항목은 상위 수준의 기능 요약입니다. 초기 `v1.x` 항목은 메인넷에 앞선 테스트넷 릴리스 라인의 역사적 기록으로 보존되어 있습니다.
:::

---

## v3.1.80 — 멀티레이어 상태-앵커 쿼리 (현재 메인넷 릴리스)

**릴리스 초점:** 롤업을 위한 읽기 가능하고 오프라인으로 검증 가능한 정산 앵커.

* **앵커 읽기 쿼리** — `x/multilayer` 쿼리 서비스가 이제 `Anchor`(레이어의 최신 상태 앵커)와 `Anchors`(레이어의 앵커 히스토리)를 노출하므로, 클라이언트는 레이어의 정산 앵커를 가져와 독립적으로 검증할 수 있습니다.
* **멀티레이어용 REST 게이트웨이** — 모든 멀티레이어 쿼리(`params`, `layers`, `layers/{layer_id}`, `anchor/{layer_id}`, `anchors/{layer_id}`, `routing-stats`)가 이제 gRPC에 더해 REST를 통해서도 제공됩니다.
* **양자 안전 정산 영수증 차단 해제** — 각 앵커는 정규화된 필드에 대한 **ML-DSA-87 (Dilithium-5)** 서명을 포함하며, 이는 Rollup Development Kit의 오프라인 정산-영수증 검증을 위한 온체인 기반을 제공합니다.

## v3.1.79 — 브리지 네트워크용 검증자 자동 프로비저닝

**릴리스 초점:** 라이선스를 보유한 검증자를 위한 연결된 네트워크에서의 즉시 사용 가능한 참여.

* **네트워크 드라이버 프레임워크** — 선언적 드라이버 프레임워크는 관련 `validator_<chain>`(또는 `qcb_bridge`) 라이선스를 보유한 QoreChain 검증자가 해당 외부 네트워크 클라이언트를 동일한 노드에서 QoreChain 오케스트레이션 하에 프로비저닝, 구성, 실행할 수 있도록 합니다 — 단, 라이선스가 활성화된 경우에만 가능합니다.
* **37개 모든 브리지 네트워크용 드라이버** — 모든 연결된 네트워크를 포괄하며, 참여 모델(퍼미션리스 검증자, 상한/선출/심사, L2 풀노드, 비스테이킹/신뢰 목록 역할)별로 분류됩니다. 외부 네트워크 스테이크와 서명 키는 네트워크별로 운영자가 제공하며, QoreChain은 프레임워크와 강제되는 라이선스 게이트를 제공합니다.

## v3.1.78 — 사전 배포 준비

**릴리스 초점:** 지갑, 브리지, IBC, 라이선싱이 출시 시점에 모두 작동 — 배포 후 거버넌스 없이.

* **신뢰 불필요(trustless) 배포 후 브리지 활성화** — `bridge_admin` 키(또는 `qcb_bridge` 라이선스 보유자)는 단일 서명된 트랜잭션(`tx bridge update-chain-config` / `set-verifier-bootstrap`)으로 연결된 체인의 브리지를 활성화할 수 있으며 — 컨트랙트 주소, 확인 수, 아키텍처, 상태, 활성 검증기, 검증기 신뢰 루트를 설정합니다 — 거버넌스 제안이나 체인 업그레이드 없이 가능합니다.
* **검증자-네트워크 라이선스 게이트** — 오케스트레이터는 이제 외부 네트워크 클라이언트를 시작하기 전에 `validator_<chain>` / `qcb_bridge` 라이선스를 강제합니다(fail-closed).
* **지갑 통합 패키지** — `@qorechain/wallet-adapter`와 `@qorechain/connect`가 npm(v0.1.0)에 게시되어, 원클릭 MetaMask 네트워크 등록(EIP-3085, EVM 레일에서 **18자리 소수점** 네이티브 QOR)과 Keplr 가스 가격 구성을 추가합니다.
* **IBC 즉시 사용 가능 릴레이어** — 8개 IBC 카운터파티를 위한 즉시 실행 가능한 릴레이어 구성과 채널 부트스트랩 도구로, 별도의 맞춤 설정 없이 배포 후 채널이 구성됩니다.

## v3.1.77 — 브리지 & 번 REST 엔드포인트

**릴리스 초점:** 크로스체인 및 공급량 모듈을 위한 읽기 전용 REST 접근.

* **브리지 REST 엔드포인트** — 브리지 모듈을 위한 읽기 전용 HTTP 쿼리 엔드포인트로, gRPC에 더해 표준 REST를 통해 브리지 상태를 노출합니다.
* **번 REST 엔드포인트** — 번 모듈을 위한 읽기 전용 HTTP 쿼리 엔드포인트로, 수수료 분배 및 공급량 데이터를 표준 REST를 통해 쿼리할 수 있게 합니다.

## v3.1.76 — SVM 툴체인 현대화

**릴리스 초점:** Solana Virtual Machine 호환성 갱신.

* **현재 툴체인 프로그램 지원** — 현재 Solana 툴체인으로 빌드된 프로그램이 QoreChain SVM 런타임에서 실행되도록 SVM 실행이 현대화되었습니다.

## v3.1.75 — 기본값으로 SVM JSON-RPC

**릴리스 초점:** 즉시 사용 가능한 Solana 호환 RPC.

* **Solana 호환 JSON-RPC** — SVM JSON-RPC 서버가 이제 기본적으로 활성화되며(포트 **8899**) 노드와 함께 자동으로 시작되어, SVM 도구를 위한 Solana 호환 RPC 인터페이스를 제공합니다.

## v3.1.74 — 롤업 프로파일 프리셋

**릴리스 초점:** Rollup Development Kit의 사용성과 정산.

* **프로파일-프리셋 적용** — 롤업 생성 시 이제 선택한 프로파일의 프리셋(DeFi, 게이밍, NFT, 엔터프라이즈, 또는 완전 맞춤형)을 적용하므로, 새 롤업은 해당 사용 사례에 맞는 합리적인 기본값을 상속합니다.
* **낙관적(optimistic) 정산** — 낙관적 정산 경로(배치 제출 및 챌린지)가 엔드 투 엔드로 작동합니다.

## v3.1.73 — 포스트 양자 해시 베이스라인

**릴리스 초점:** 기본 포스트 양자 암호화 베이스라인 완성.

* **SHAKE-256 기본 해시** — SHAKE-256(SHA-3 계열)이 기본 애플리케이션 해시로 채택되어, **ML-DSA-87 (Dilithium-5)** 서명, **ML-KEM-1024** 키 캡슐화, **SHAKE-256** 해싱으로 구성된 기본 포스트 양자 베이스라인을 완성합니다.

## v3.1.72 — 안정성 & 유지보수

**릴리스 초점:** 일상적인 안정성 및 빌드 파이프라인 유지보수.

* **안정성 개선** — 외부에 보이는 동작 변경 없이 내부 안정성, 의존성, 빌드 파이프라인 유지보수.

## v3.1.71 — 기본값으로 강제되는 PQC 하이브리드 서명

**릴리스 초점:** Cosmos 트랜잭션 경로에서 기본적으로 활성화되는 포스트 양자 보안.

* **기본값으로 요구되는 하이브리드 서명** — 포스트 양자 하이브리드 서명이 이제 Cosmos 트랜잭션 경로에서 기본적으로 강제됩니다: 각 트랜잭션은 고전적 **secp256k1** 서명과 함께 포스트 양자 **ML-DSA-87 (Dilithium-5)** 서명을 포함합니다.
* **거버넌스 제어 강제** — 강제 모드는 거버넌스에 의해 제어되며, 기본값은 **required**로 설정됩니다.

## v3.1.70 — 프로덕션 하드닝

**릴리스 초점:** 가동 중인 메인넷을 위한 프로덕션 하드닝 및 합의 최적화.

* **PRISM 합의 최적화** — 가동 중인 네트워크 조건에서 적응형 파라미터 튜닝을 위한 PRISM 강화학습 최적화 레이어의 지속적인 개선과 서킷 브레이커 안전 제어.
* **성능 및 안정성** — 검증자와 풀노드 전반에 걸친 처리량, 지연 시간, 리소스 사용량 개선.
* **운영 도구** — 메인넷 운영자를 위한 모니터링, 쿼리, 노드 운영 편의성 개선.
* **Tokenomics v2.1 정렬** — 고정 공급량, 유한 발행 경제 모델에 맞춰 수수료 분배 및 발행 메커니즘 정렬.

## v3.0.0 — 메인넷 제네시스

**릴리스 초점:** 메인넷 출시 및 토큰 생성 이벤트.

* **메인넷 제네시스** — QoreChain 메인넷(`qorechain-vladi`, EVM 체인 ID 9801)이 **2026년 6월 7일**에 출시되었으며, 제네시스 시점에 토큰 생성 이벤트(TGE)가 진행되었습니다.
* **5방향 수수료 분할** — 검증자, 번, 트레저리, 스테이커, 라이트 노드 전반에 걸친 프로토콜 수수료 분배(**37 / 30 / 20 / 10 / 3**)로, 전용 라이트 노드 몫을 추가합니다.
* **온체인 AMM** — 온체인 유동성 풀 및 스왑을 위한 네이티브 자동 마켓 메이커 모듈(`x/amm`).
* **체인 라이선싱** — 프로토콜 권한을 등록하고 관리하기 위한 온체인 라이선스 모듈(`x/license`).
* **하드닝된 정산 패러다임** — RDK 정산 모드가 optimistic, zk, based, sovereign으로 확정되었습니다.

* **확장된 크로스체인 커버리지** — 더 광범위한 외부 네트워크 집합에 대한 추가 IBC 및 브리지 연결.

## v1.3.0 — Rollup Development Kit

**릴리스 초점:** 소버린 및 공유 보안 롤업 배포를 위한 네이티브 롤업 인프라.

* **x/rdk 모듈** — 네 가지 정산 패러다임(optimistic, zk, based, sovereign)을 갖춘 완전한 Rollup Development Kit
* **5개 프리셋 프로파일** — DeFi, 게이밍, NFT, 엔터프라이즈, 완전 맞춤형 사용 사례를 위한 사전 구성된 롤업 템플릿
* **네이티브 데이터 가용성** — 블롭 스토리지, 보존 관리, 프루닝 라이프사이클을 갖춘 온체인 DA 레이어
* **EndBlocker 자동 확정** — 챌린지 윈도우가 만료될 때 운영자 개입 없이 자동 배치 확정
* **AI 지원 프로파일 선택** — 의도한 사용 사례를 기반으로 최적의 롤업 구성을 추천하는 `suggest-profile` 쿼리
* **멀티레이어 통합** — 롤업은 멀티레이어 아키텍처에서 레이어로 등록되어, 라우팅, 앵커링, 챌린지 메커니즘을 상속합니다
* **뱅크 에스크로 라이프사이클** — 운영자 스테이크는 롤업 운영 중 에스크로에 보관되며, 정상 종료 시 해제되거나 슬래싱 시 몰수됩니다

## v1.2.0 — IBC & 브리지

**릴리스 초점:** 크로스체인 연결성 및 고급 계정 추상화.

* **25개 크로스체인 연결** — 외부 네트워크에 대한 8개 IBC 채널 및 17개 QoreChain Bridge (QCB) 연결
* **x/babylon 모듈** — 비트코인 보유자가 QoreChain 스테이킹 보안에 참여할 수 있도록 하는 BTC 리스테이킹 통합
* **x/abstractaccount 모듈** — 프로그래밍 가능한 지출 규칙, 세션 키, 맞춤 인증 로직을 갖춘 스마트 계정 프레임워크
* **x/fairblock 모듈** — MEV 저항 트랜잭션 암호화를 위한 임계값 신원 기반 암호화(tIBE)
* **x/gasabstraction 모듈** — 네이티브 QOR, IBC 브리지 USDC, IBC 브리지 ATOM을 지원하는 다중 토큰 가스 결제
* **5레인 TX 우선순위화** — 우선순위별로 정렬된 트랜잭션 레인: 시스템, 거버넌스, 스테이킹, 브리지, 일반
* **IBC 릴레이어 구성** — 지원되는 모든 IBC 채널을 위한 사전 구성된 릴레이어 설정
* **브리지-투-번 통합** — 브리지 수수료는 번 모듈의 수수료 분배를 통해 라우팅됩니다

## v1.1.0 — PQC 하이브리드 서명

**릴리스 초점:** 포스트 양자 암호화 보안 및 알고리즘 민첩성.

* **이중 secp256k1 (ECDSA) + ML-DSA-87 서명** — 모든 트랜잭션은 고전적 서명과 포스트 양자 서명을 모두 포함하며, AnteHandler 체인에서 검증됩니다
* **3가지 강제 모드** — 구성 가능한 하이브리드 서명 강제: off(모드 0), permissive(모드 1, PQC 선택), mandatory(모드 2, PQC 필수)
* **자동 등록** — PQC 공개 키는 첫 하이브리드 트랜잭션 시 자동으로 등록되어 별도의 등록 단계를 제거합니다
* **SHAKE-256 해시 기반** — 모든 PQC 관련 해싱 작업은 양자 저항 주소 파생을 위해 SHAKE-256(SHA-3 계열)을 사용합니다
* **TEE 증명 인터페이스** — PQC 키 생성 무결성을 입증하기 위한 신뢰 실행 환경(Trusted Execution Environment) 증명 지원
* **알고리즘 민첩성 프레임워크** — 향후 PQC 알고리즘을 체인 업그레이드 없이 거버넌스를 통해 추가할 수 있도록 하는 플러그 가능한 알고리즘 레지스트리

## v1.0.0 — 제네시스 (토크노믹스 엔진)

**릴리스 초점:** 완전한 토크노믹스, 다중 VM 실행, AI 지원 운영을 갖춘 초기 프로토콜 출시.

* **x/burn 모듈** — 검증자, 번, 트레저리, 스테이커 전반의 4방향 분배를 갖춘 다중 채널 수수료 번 메커니즘
* **x/xqore 모듈** — 단계별 조기 잠금 해제 페널티 및 PvP 리베이스 재분배를 갖춘 거버넌스 스테이킹 파생상품
* **x/inflation 모듈** — 유한 발행 경제 모델에 의해 통제되는 연간 감쇠를 갖춘 에포크 기반 발행
* **PRISM 합의 레이어** — 서킷 브레이커 안전 제어를 갖춘 동적 체인 파라미터 튜닝을 위한 강화학습 최적화(PPO)
* **트리플 풀 CPoS** — 평판 점수로 가중치가 부여된 Emerald, Sapphire, Ruby 검증자 풀을 갖춘 분류형 지분 증명(Classified Proof-of-Stake)
* **QDRW 거버넌스** — 풀 전반의 보상 분배에 대한 거버넌스 승인 조정을 허용하는 동적 보상 가중치(Dynamic Reward Weighting) 시스템
* **EVM + CosmWasm + SVM 런타임** — 세 가지 동시 실행 환경: QoreChain EVM Engine, CosmWasm 스마트 컨트랙트, Solana Virtual Machine
* **크로스 VM 브리지** — 단일 블록 내에서 EVM, CosmWasm, SVM 런타임 간 메시지 전달 및 자산 이전
* **포스트 양자 암호화** — 고성능 PQC 라이브러리로 뒷받침되는 양자 저항 서명
* **QCAI** — 사기 탐지, 수수료 추정, 네트워크 최적화를 위한 선택적 오프체인 사이드카를 갖춘 온체인 휴리스틱 분석
* **컨테이너화된 배포** — 사이드카 서비스 및 블록 인덱서를 갖춘 완전한 다중 검증자 테스트넷 배포
* **블록 인덱서** — 과거 쿼리 및 분석을 위한 영구 스토리지를 갖춘 블록 리스너
