# إصدار UX — لوحة تحكم الويب

يشغِّل إصدار **UX (تجربة المستخدم - User eXperience)** نفس عفريت العقدة الخفيفة الذي يشغِّله إصدار SX، لكنه يضيف **لوحة تحكم ويب مدمجة** بحيث يمكنك مراقبة العقدة والشبكة في المتصفح. الثنائي هو `lightnode-ux`. مثل إصدار SX، هذا هو خط **v3.1.1** للعقدة الخفيفة (إصدارها الخاص، منفصل عن إصدار السلسلة).

إصدار UX هو الخيار الصحيح للاستخدام على سطح المكتب وللمشغِّلين الذين يفضلون واجهة بصرية على سطر الأوامر.

## التثبيت

### البناء من المصدر

يتطلب إصدار UX **Go 1.26.1** ويُبنى مع تفعيل CGO من أجل المكتبة الأصلية لما بعد الكمّي:

```bash
CGO_ENABLED=1 go build -o build/lightnode-ux ./cmd/lightnode-ux/
```

ينتج هذا `build/lightnode-ux`.

### Docker

تُبنى خدمة UX من `Dockerfile.ux`:

```bash
docker compose up lightnode-ux
```

تحفظ حاوية UX البيانات في وحدة تخزين مسمّاة عند `/root/.qorechain-lightnode` وتقرأ عنوان RPC للسلسلة من متغير البيئة `QORECHAIN_RPC_ADDR`.

## التشغيل

ابدأ عقدة UX:

```bash
build/lightnode-ux start
```

يطلق هذا العفريت وخادم لوحة التحكم المدمجة معًا. يفعِّل إصدار UX لوحة التحكم دائمًا. عند بدء التشغيل يطبع الثنائي رابط لوحة التحكم.

يتشارك إصدار UX إعداده مع إصدار SX: يقرأ نفس `config.toml` من `~/.qorechain-lightnode` ويستخدم نفس حلقة مفاتيح Dilithium-5. إذا لم تكن قد هيّأت العقدة بعد، شغِّل معالج SX أولًا (`lightnode-sx onboard`) لكتابة الإعدادات واستيراد أو توليد مفتاحك — راجع [إصدار SX](/light-node/sx-edition).

## لوحة تحكم الويب على المنفذ 8420

تُعرَض لوحة التحكم على **المنفذ 8420**. هذا هو المنفذ الذي تعلنه صورة Docker الخاصة بـ `lightnode-ux` (`EXPOSE 8420`) والافتراضي الذي يرتبط به الثنائي، لذا عند التشغيل عبر Docker تُنشَر لوحة التحكم على `8420`:

```
http://localhost:8420
```

:::caution تحقق من تعيين المنفذ في ملف compose الخاص بك
تشير بعض النصوص في أماكن أخرى إلى المنفذ 8080 للوحة التحكم. القيمة المعتمدة هي **8420** — وهي ما تعرضه الصورة فعليًا وما يرتبط به العفريت افتراضيًا. إذا عدّلت ملف `docker-compose.yml` الخاص بك أو وكيلًا عكسيًا، عيِّن إلى **8420**، وليس 8080.
:::

## ما تعرضه لوحة التحكم

تُنظَّم لوحة التحكم في العروض التالية:

- **Overview** — ارتفاع الكتل وحالة العقدة بلمحة سريعة.
- **Validators** — مجموعة المُدققين المربوطين.
- **Delegation** — تفويضاتك الحالية وتقسيمها.
- **Network** — القياس عن بُعد الحي للشبكة والترويسات المتزامنة حديثًا.
- **Bridge** — القياس عن بُعد للجسر عبر السلاسل.
- **Tokenomics** — القياس عن بُعد لاقتصاد الرمز.
- **Settings** — التهيئة الفعلية للعقدة.

يُحدَّث القياس عن بُعد في الوقت الفعلي، إذ يحدِّث العفريت بيانات المُدققين والشبكة والجسر واقتصاد الرمز على فترات مستقلة (قابلة للتهيئة تحت `[telemetry]` في `config.toml`).

### شريط الوضع المحلي فقط

إذا لم يكن للعقدة **نقطة نهاية RPC للسلسلة مهيَّأة**، تعمل لوحة التحكم في **الوضع المحلي فقط** وتعرض شريطًا بارزًا يشرح الحالة: حزمة PQC مُتحقَّق منها، لكن العقدة لا تتزامن مع أي سلسلة، لذا يبقى ارتفاع الكتل عند `0`. يحثك الشريط على تشغيل معالج التهيئة الأولية على المضيف:

```bash
lightnode-sx onboard
```

يشغِّل المعالج الاختبار الذاتي لـ PQC، ويطلب نقطة نهاية سلسلتك، ويستورد أو يولِّد مفتاح المُدقق الخاص بك. بمجرد تهيئة نقطة نهاية، أعد تشغيل العقدة وتبدأ لوحة التحكم بعرض بيانات السلسلة الحية.
