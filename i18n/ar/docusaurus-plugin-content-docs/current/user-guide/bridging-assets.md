# ربط الأصول عبر الجسور

يغطّي هذا الدليل كيفية نقل الأصول بين QoreChain وشبكات بلوكتشين أخرى. تتألّف طبقة قابلية التشغيل البيني في QoreChain من **37 تهيئة QCB (جسر QoreChain)** (بما في ذلك حلقة استرجاع QoreChain) للشبكات غير المتجانسة إضافةً إلى **8 قنوات IBC** لسلاسل منظومة Cosmos.

:::caution
يوجد الجسر عبر السلاسل حالياً في مرحلة **اختبار / ما قبل الإنتاج**. تخضع جاهزية الاتصالات والأصول المدعومة ومعاملات النهائية للتغيير، ولا ينبغي اعتبارها جاهزة للإنتاج. تحقّق من جميع عمليات النقل على **`qorechain-diana`** قبل الاعتماد عليها.
:::

:::note
تستخدم الأوامر أدناه شبكة الاختبار **`qorechain-diana`** (معرّف سلسلة EVM **9800**). أمّا الشبكة الرئيسية (**`qorechain-vladi`**، معرّف سلسلة EVM **9801**) فهي قيد التشغيل منذ 7 يونيو 2026 وتعمل بإصدار السلسلة **v3.1.80** — استبدل معرّف سلسلة الشبكة الرئيسية ونقاط النهاية من صفحة **الاتصال بالشبكة الرئيسية** حيث جرى تفعيل دعم الجسر.
:::

## نظرة عامة على الاتصالات

توفّر QoreChain بروتوكولَي ربط:

| البروتوكول                                 | الاتصالات          | حالة الاستخدام                                                                 |
| ---------------------------------------- | ------------------ | ------------------------------------------------------------------------ |
| **IBC** (التواصل بين سلاسل البلوكتشين) | 8 قنوات         | قابلية تشغيل بيني أصلية مع السلاسل المفعّلة لـ IBC                          |
| **QCB** (جسر QoreChain)               | 37 تهيئة  | عمليات نقل عبر السلاسل مع شبكات غير IBC عبر إثباتات مؤمّنة بـ PQC |

يوجد سرد كامل لكل تهيئة QCB وقناة IBC في صفحة **بنية الجسر**. يركّز هذا الدليل على الاستخدام اليومي للجسر.

## قنوات IBC

أنشأت السلاسل التالية المفعّلة لـ IBC قنوات مع QoreChain:

| السلسلة                | القناة     | الحالة |
| -------------------- | ----------- | ------ |
| Cosmos Hub           | `channel-0` | نشطة |
| Osmosis              | `channel-1` | نشطة |
| Noble                | `channel-2` | نشطة |
| Celestia             | `channel-3` | نشطة |
| Stride               | `channel-4` | نشطة |
| Akash                | `channel-5` | نشطة |
| Babylon              | `channel-6` | نشطة |
| QoreChain (حلقة استرجاع) | `channel-7` | نشطة |

تستخدم عمليات نقل IBC وحدة `ibc-transfer` القياسية:

```bash
qorechaind tx ibc-transfer transfer transfer <channel> <recipient> <amount>uqor \
  --from mykey \
  --chain-id qorechain-diana \
  --fees 500uqor
```

## تدفّق الإيداع (من سلسلة خارجية إلى QoreChain)

يتّبع إيداع الأصول من سلسلة خارجية إلى QoreChain هذا التسلسل:

1. **القفل** — اقفل الرموز على السلسلة الخارجية بإرسالها إلى عقد أو عنوان جسر QCB.
2. **الإثبات** — يراقب مدقّقو الجسر معاملة القفل وينتجون إثباتات موقّعة بـ PQC.
3. **العتبة** — بمجرّد جمع **7 من أصل 10** إثباتات مدقّقين، ينهي الجسر الإيداع.
4. **السكّ** — تُسَكّ الرموز المغلّفة المكافئة على QoreChain وتُضاف إلى رصيد عنوانك `qor1...`.

**أمر CLI:**

```bash
qorechaind tx bridge deposit \
  --chain ethereum \
  --amount 1000000 \
  --from mykey \
  --chain-id qorechain-diana \
  --fees 500uqor
```

## تدفّق السحب (من QoreChain إلى سلسلة خارجية)

سحب الأصول من QoreChain إلى سلسلة خارجية:

1. **الحرق** — احرق الرموز المغلّفة على QoreChain.
2. **الإثبات** — يراقب مدقّقو الجسر عملية الحرق وينتجون إثباتات موقّعة بـ PQC.
3. **العتبة** — بمجرّد جمع **7 من أصل 10** إثباتات، يُنهى السحب.
4. **فك القفل** — تُحرَّر الرموز الأصلية على السلسلة الخارجية إلى عنوان الوجهة المحدّد.

**أمر CLI:**

```bash
qorechaind tx bridge withdraw \
  --chain ethereum \
  --amount 1000000 \
  --to 0xYourEthereumAddress \
  --from mykey \
  --chain-id qorechain-diana \
  --fees 500uqor
```

## نموذج الأمان

يُؤمَّن جسر QoreChain بطبقات دفاع متعددة:

| الآلية                    | الوصف                                                                                                                                           |
| ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| **التوقيع المتعدد PQC 7 من 10**     | تتطلّب كل عملية جسر إثباتات من 7 من أصل 10 مدقّقي جسر على الأقل، يستخدم كل منهم توقيعات تشفير مقاومة للكم.               |
| **فترة طعن مدتها 24 ساعة** | تدخل عمليات السحب التي تتجاوز عتبة قابلة للتهيئة نافذة طعن مدتها 24 ساعة يمكن خلالها للمدقّقين أو المراقبين الإبلاغ عن المعاملات الاحتيالية. |
| **قواطع الدائرة**         | تُوقِف محدِّدات المعدّل الآلية عمليات الجسر إذا اكتُشِف حجم غير طبيعي أو أنماط مشبوهة. تُستأنف عمليات الجسر بعد المراجعة اليدوية.  |

## الاستعلام عن حالة الجسر

تحقّق من حالة عملية جسر معلّقة:

```bash
qorechaind query bridge pending-deposits --address <your_qor_address>
```

```bash
qorechaind query bridge pending-withdrawals --address <your_qor_address>
```

اسرد جميع اتصالات الجسر النشطة:

```bash
qorechaind query bridge connections
```

## نصائح

* عادةً ما تُنهى إيداعات الجسر خلال دقائق بمجرّد جمع الإثباتات المطلوبة 7 من 10.
* تُطلِق عمليات السحب الكبيرة فترة الطعن البالغة 24 ساعة تلقائياً. خطّط مسبقاً لعمليات النقل الحساسة للوقت.
* تحقّق دائماً من أن تنسيق عنوان الوجهة يطابق السلسلة المستهدفة (مثل `0x...` لسلاسل EVM، وbase58 لـ Solana).
* عمليات نقل IBC أسرع عموماً من عمليات نقل QCB لأنها تستخدم تواصلاً أصلياً على مستوى البروتوكول.
* تُحرق رسوم الجسر عبر قناة الحرق `bridge_fee` (راجع [عمليات الرمز](/user-guide/token-operations)).
