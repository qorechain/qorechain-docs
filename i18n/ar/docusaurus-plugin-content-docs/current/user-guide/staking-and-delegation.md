# التخزين والتفويض

يغطي هذا الدليل كيفية تفويض رموز QOR إلى المُدقِّقين، وإعادة التفويض بين المُدقِّقين، وفك ارتباط حصتك، والمطالبة بالمكافآت، وفهم بنية التخزين ثلاثية المجمّعات (Triple-Pool) في QoreChain.

:::note
تستخدم الأوامر أدناه شبكة الاختبار **`qorechain-diana`** (معرّف سلسلة EVM رقم **9800**). الشبكة الرئيسية (**`qorechain-vladi`**، معرّف سلسلة EVM رقم **9801**) أصبحت فعّالة منذ 7 يونيو 2026 وتعمل بإصدار السلسلة **v3.1.80** — استبدل معرّف السلسلة ونقاط النهاية الخاصة بالشبكة الرئيسية من صفحة **Connecting to Mainnet** عند التخزين على الشبكة الرئيسية.
:::

---

## تفويض الرموز

فوّض QOR إلى مُدقِّق لكسب مكافآت التخزين والمشاركة في أمان الشبكة:

```bash
qorechaind tx staking delegate <validator_address> <amount>uqor \
  --from mykey \
  --chain-id qorechain-diana \
  --fees 500uqor
```

**مثال:** تفويض 100 QOR إلى مُدقِّق:

```bash
qorechaind tx staking delegate qorvaloper1abc...xyz 100000000uqor \
  --from mykey \
  --chain-id qorechain-diana \
  --fees 500uqor
```

---

## إعادة التفويض

انقل تفويضك من مُدقِّق إلى آخر دون انتظار فترة فك الارتباط:

```bash
qorechaind tx staking redelegate <source_validator> <destination_validator> <amount>uqor \
  --from mykey \
  --chain-id qorechain-diana \
  --fees 500uqor
```

**مثال:**

```bash
qorechaind tx staking redelegate qorvaloper1src... qorvaloper1dst... 50000000uqor \
  --from mykey \
  --chain-id qorechain-diana \
  --fees 500uqor
```

:::caution
لا يمكنك إعادة تفويض رموز موجودة بالفعل في حالة عبور إعادة تفويض. انتظر اكتمال إعادة التفويض الحالية قبل بدء أخرى.
:::

---

## فك الارتباط

اسحب رموزك المفوَّضة من مُدقِّق. يستغرق فك الارتباط **21 يومًا** لإكماله، وخلالها لا تكسب الرموز مكافآت ولا يمكن تحويلها.

```bash
qorechaind tx staking unbond <validator_address> <amount>uqor \
  --from mykey \
  --chain-id qorechain-diana \
  --fees 500uqor
```

**مثال:**

```bash
qorechaind tx staking unbond qorvaloper1abc...xyz 25000000uqor \
  --from mykey \
  --chain-id qorechain-diana \
  --fees 500uqor
```

بعد فترة فك الارتباط البالغة 21 يومًا، تُعاد الرموز تلقائيًا إلى حسابك.

---

## المطالبة بالمكافآت

اسحب جميع مكافآت التخزين المتراكمة من كل مُدقِّق فوّضت إليه:

```bash
qorechaind tx distribution withdraw-all-rewards \
  --from mykey \
  --chain-id qorechain-diana \
  --fees 500uqor
```

لسحب المكافآت من مُدقِّق محدد فقط:

```bash
qorechaind tx distribution withdraw-rewards <validator_address> \
  --from mykey \
  --chain-id qorechain-diana \
  --fees 500uqor
```

تُموَّل مكافآت التخزين من مجمّع التخزين الخاص بالبروتوكول البالغ 590 مليون QOR وفقًا لجدول Tokenomics v2.1، إلى جانب حصة المُخزِّنين (10%) من كل رسوم معاملة.

---

## تصنيف ثلاثي المجمّعات

تستخدم QoreChain نموذج تخزين **ثلاثي المجمّعات** (Triple-Pool) يصنّف المُدقِّقين إلى ثلاثة مجمّعات بناءً على سمعتهم ومستويات التفويض لديهم. يتلقى كل مجمّع حصة موزونة من مكافآت الكتل.

| المجمّع                               | معايير الدخول                                              | وزن المكافأة |
| ------------------------------------ | ----------------------------------------------------------- | ------------- |
| **RPoS** (إثبات الحصة بالسمعة) | درجة السمعة >= المئين السبعين **و** الحصة >= الوسيط | 40%           |
| **DPoS** (إثبات الحصة المفوَّضة)  | إجمالي التفويض >= 10,000 QOR                              | 35%           |
| **PoS** (إثبات الحصة)             | جميع المُدقِّقين المتبقّين                                    | 25%           |

يُعاد تصنيف المُدقِّقين عند كل حدود حقبة (epoch). تتم ترقية المُدقِّق الذي يبني سمعة قوية ويراكم حصة كافية إلى مجمّع RPoS، فيكسب أعلى حصة من المكافآت.

---

## مكافآت منحنى الربط

تُحسب مكافآت التخزين الفردية باستخدام صيغة منحنى الربط (bonding curve) الخاصة بـ QoreChain:

```
R = beta * S * (1 + alpha * log(1 + L)) * Q(r) * P(t)
```

| المتغير  | الوصف                                                          |
| -------- | -------------------------------------------------------------------- |
| `R`      | مبلغ المكافأة للفترة                                         |
| `beta`   | معدل المكافأة الأساسي (معامل البروتوكول)                                |
| `S`      | المبلغ المُخزَّن                                                        |
| `alpha`  | معامل الولاء (معامل البروتوكول)                             |
| `L`      | مدة القفل بالحقب                                              |
| `Q(r)`   | مضاعِف الجودة المستمد من درجة سمعة المُدقِّق `r` |
| `P(t)`   | مضاعِف المجمّع عند الزمن `t` (40% أو 35% أو 25% حسب المجمّع)     |

ينتج عن مدد القفل الأطول ودرجات السمعة الأعلى مكافآت أكبر تناسبيًا، مما يحفّز الالتزام طويل الأمد وحُسن سلوك المُدقِّق.

---

## الاستعلام عن معلومات المُدقِّق

ابحث عن تفاصيل أي مُدقِّق:

```bash
qorechaind query staking validator <validator_operator_address>
```

**مثال:**

```bash
qorechaind query staking validator qorvaloper1abc...xyz
```

اعرض جميع المُدقِّقين النشطين:

```bash
qorechaind query staking validators --status bonded
```

استعلم عن تفويضاتك الحالية:

```bash
qorechaind query staking delegations <delegator_address>
```

---

:::tip

* يحقق التفويض لمُدقِّقين في **مجمّع RPoS** أعلى المكافآت بسبب وزن المجمّع البالغ 40%.
* يستغرق بناء سمعة المُدقِّق وقتًا. ضع في اعتبارك سجل المُدقِّق قبل التفويض.
* إعادة التفويض فورية لكنها تخضع لقيود فترة تهدئة. خطّط لتحركاتك بعناية.
* فترة فك الارتباط البالغة 21 يومًا هي إجراء أمني. خلال هذا الوقت، لا يزال بإمكان أحداث الخصم (slashing) التأثير على رموزك.

:::
