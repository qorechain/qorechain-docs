# الحوكمة

يغطّي هذا الدليل كيفية عمل الحوكمة على السلسلة في QoreChain، بما في ذلك نظام التصويت الموزون بالتفويض والسمعة التربيعي (QDRW)، وكيفية تقديم المقترحات، وكيفية التصويت.

:::note
تستخدم الأوامر أدناه شبكة الاختبار **`qorechain-diana`** (معرّف سلسلة EVM **9800**). أمّا الشبكة الرئيسية (**`qorechain-vladi`**، معرّف سلسلة EVM **9801**) فهي قيد التشغيل منذ 7 يونيو 2026 وتعمل بإصدار السلسلة **v3.1.80** — استبدل معرّف سلسلة الشبكة الرئيسية ونقاط النهاية من صفحة **الاتصال بالشبكة الرئيسية** عند المشاركة في الحوكمة على الشبكة الرئيسية.
:::

---

## قوة التصويت: صيغة QDRW

تستخدم QoreChain صيغة **الموزون بالتفويض والسمعة التربيعي (QDRW)** لحساب قوة التصويت. يمنع هذا النظام هيمنة الحيتان بينما يكافئ المشاركين الذين اكتسبوا درجات سمعة عالية والتزموا بالحوكمة عبر مراهنة xQORE.

```
VP = sqrt(staked + 2 * xQORE) * ReputationMultiplier(r)
```

| المتغيّر                  | الوصف                                                                                                                      |
| ------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `VP`                      | قوة التصويت الفعّالة                                                                                                           |
| `staked`                  | إجمالي رموز QOR المرهونة من قِبل الناخب                                                                                             |
| `xQORE`                   | كمية رموز حوكمة xQORE المملوكة (راجع [مراهنة xQORE](/user-guide/xqore-staking))                                          |
| `r`                       | درجة سمعة الناخب، مُطبَّعة إلى \[0, 1]                                                                                  |
| `ReputationMultiplier(r)` | دالة سيغمويد تُطابق السمعة بمُضاعِف في المدى \[0.5, 2.0]                                                                     |

### الخصائص الرئيسية

* **التخميد التربيعي:** يحصل المالك الذي يمتلك ضعف حصّة ناخب آخر بمقدار 100 مرة على \~10 أضعاف قوة التصويت فقط، وليس 100 ضعف. يضمن هذا أن يتوسّع نفوذ الحوكمة بشكل دون خطّي مع الثروة.
* **مكافأة xQORE:** تُحتسب رموز xQORE بوزن **مضاعف (2x)** داخل الجذر التربيعي، ما يمنح المشاركين الملتزمين بالحوكمة ميزة ذات معنى.
* **مُضاعِف السمعة:** يُطابق درجة سمعة الناخب من \[0, 1] بمُضاعِف في \[0.5, 2.0] باستخدام منحنى سيغمويد. يمكن للمشاركين ذوي السمعة العالية مضاعفة قوة تصويتهم الفعّالة، بينما يرى المشاركون ذوو السمعة المنخفضة نفوذهم منقسماً إلى النصف.

## تقديم مقترح

يمكن لأي مالك QOR تقديم مقترح حوكمة. يلزم إيداع بحد أدنى لكي يدخل المقترح فترة التصويت.

```bash
qorechaind tx gov submit-proposal <proposal_file.json> \
  --from mykey \
  --chain-id qorechain-diana \
  --fees 500uqor
```

**مثال على ملف مقترح** (`proposal.json`):

```json
{
  "title": "Increase Maximum Validator Count",
  "description": "This proposal increases the maximum active validator set from 100 to 150 to improve decentralization.",
  "type": "parameter_change",
  "changes": [
    {
      "subspace": "staking",
      "key": "MaxValidators",
      "value": "150"
    }
  ],
  "deposit": "10000000uqor"
}
```

## التصويت على المقترحات

بمجرّد دخول المقترح فترة التصويت، يمكن لأي مراهن الإدلاء بصوته:

```bash
qorechaind tx gov vote <proposal_id> <option> \
  --from mykey \
  --chain-id qorechain-diana \
  --fees 500uqor
```

**خيارات التصويت:**

| الخيار         | الوصف                                                                                              |
| -------------- | -------------------------------------------------------------------------------------------------------- |
| `yes`          | تأييد المقترح                                                                                     |
| `no`           | معارضة المقترح                                                                                      |
| `abstain`      | الإقرار بالمقترح دون اتخاذ موقف                                                       |
| `no_with_veto` | معارضة المقترح والإشارة إلى أنه ما كان ينبغي تقديمه (يحرق الإيداع إذا استُوفيت العتبة) |

**مثال:**

```bash
qorechaind tx gov vote 1 yes \
  --from mykey \
  --chain-id qorechain-diana \
  --fees 500uqor
```

## أنواع المقترحات

تدعم QoreChain أنواع مقترحات الحوكمة التالية:

| النوع                 | الوصف                                                                                     |
| -------------------- | ----------------------------------------------------------------------------------------------- |
| **نصّي**             | مقترح إشاري دون تنفيذ آلي على السلسلة. يُستخدم لفحص مشاعر المجتمع. |
| **تغيير معامِل** | يعدّل واحداً أو أكثر من معاملات البروتوكول على السلسلة (مثل الحد الأقصى للمدقّقين، معدّل الإصدار).        |
| **ترقية برمجية** | يجدول ترقية سلسلة منسّقة عند ارتفاع كتلة محدّد.                              |
| **إنفاق مجتمعي**  | يطلب أموالاً من خزينة المجتمع لعنوان مستلِم محدّد.                   |

---

## الاستعلام عن المقترحات

اسرد جميع المقترحات:

```bash
qorechaind query gov proposals
```

استعلم عن مقترح محدّد بالمعرّف:

```bash
qorechaind query gov proposal <proposal_id>
```

تحقّق من العدّ الحالي للأصوات على مقترح:

```bash
qorechaind query gov tally <proposal_id>
```

اعرض صوتك أنت على مقترح:

```bash
qorechaind query gov vote <proposal_id> <voter_address>
```

## معاملات الحوكمة

استعلم عن معاملات الحوكمة الحالية:

```bash
qorechaind query gov params
```

تشمل المعاملات الرئيسية:

| المعامِل            | الوصف                                                      |
| -------------------- | ---------------------------------------------------------------- |
| `min_deposit`        | الحد الأدنى للإيداع المطلوب لدخول المقترح في التصويت          |
| `max_deposit_period` | النافذة الزمنية للوصول إلى الحد الأدنى للإيداع                     |
| `voting_period`      | مدة فترة التصويت بمجرّد أن يصبح المقترح نشطاً          |
| `quorum`             | الحد الأدنى للمشاركة المطلوب لصحّة التصويت                  |
| `threshold`          | الحد الأدنى لنسبة "نعم" للنجاح (باستثناء الامتناع)            |
| `veto_threshold`     | الحد الأدنى لنسبة "لا مع النقض" للرفض وحرق الإيداع |

---

:::tip

* ابنِ سمعتك قبل أصوات الحوكمة الكبرى لتعظيم مُضاعِف قوة تصويتك.
* اقفل QOR في xQORE للحصول على مكافأة وزن حوكمة مضاعفة (2x) داخل صيغة QDRW.
* استخدم `no_with_veto` بحذر. إذا تم بلوغ عتبة النقض، يُحرق إيداع المقترح.
* تُزال المقترحات التي لا تبلغ الحد الأدنى للإيداع خلال فترة الإيداع تلقائياً.

:::
