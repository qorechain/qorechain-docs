# تجريد الغاز

يغطّي هذا الدليل ميزة تجريد الغاز في QoreChain، التي تتيح للمستخدمين دفع رسوم المعاملات برموز غير أصلية بدلاً من QOR.

:::note
تستخدم الأوامر أدناه شبكة الاختبار **`qorechain-diana`** (معرّف سلسلة EVM **9800**). أمّا الشبكة الرئيسية (**`qorechain-vladi`**، معرّف سلسلة EVM **9801**) فهي قيد التشغيل منذ 7 يونيو 2026 وتعمل بإصدار السلسلة **v3.1.82** — استبدل معرّف سلسلة الشبكة الرئيسية ونقاط النهاية من صفحة **الاتصال بالشبكة الرئيسية** عند إجراء المعاملات على الشبكة الرئيسية.
:::

---

## نظرة عامة

يزيل تجريد الغاز شرط امتلاك رموز QOR لدفع رسوم المعاملات. يمكن للمستخدمين الذين يمتلكون رموزاً بديلة مقبولة (مثل USDC أو ATOM المنقولة عبر IBC) استخدام تلك الرموز مباشرةً كدفع للرسوم. يحوّل البروتوكول مبلغ الرسوم تلقائياً إلى مكافئه الأصلي قبل المعالجة.

---

## الرموز المقبولة

الرموز التالية مقبولة لدفع الرسوم:

| الرمز              | التسمية | معدّل التحويل | مثال على الرسوم          |
| ------------------ | ------------ | --------------- | -------------------- |
| **QOR**            | `uqor`       | 1.0 (أصلي)    | `--fees 500uqor`     |
| **USDC** (عبر IBC) | `ibc/USDC`   | 1.0             | `--fees 500ibc/USDC` |
| **ATOM** (عبر IBC) | `ibc/ATOM`   | 10.0            | `--fees 50ibc/ATOM`  |

:::note
تعكس معدّلات التحويل نسبة الصرف المحدّدة من البروتوكول، وليس أسعار السوق. يعني معدّل قدره 10.0 لـ ATOM أن وحدة واحدة من ibc/ATOM تكافئ 10 وحدات من uqor لأغراض الرسوم.
:::

---

## كيفية عمله

يجري دمج `GasAbstractionDecorator` الخاص بـ QoreChain في خط أنابيب معالجة المعاملات. عندما تتضمّن معاملة رسوماً بتسمية غير أصلية، يحدث ما يلي:

1. **فحص الرسوم** — يتحقّق الـ decorator من تسمية الرسوم المحدّدة في المعاملة.
2. **البحث عن المعدّل** — إذا كانت التسمية ضمن قائمة الرموز المقبولة، يبحث البروتوكول عن معدّل التحويل المقابل.
3. **التحويل** — يُحوَّل مبلغ الرسوم إلى مكافئه الأصلي بـ uqor باستخدام معدّل التحويل.
4. **المعالجة القياسية** — تُمرَّر الرسوم المحوّلة إلى معالج `DeductFee` القياسي لخصمها من حساب المرسِل. التحويل شفّاف لبقية خط أنابيب المعاملة. تعمل جميع معالجات الرسوم اللاحقة (التوزيع على المدقّقين، والحرق، وتخصيص الخزينة، ومكافآت المراهنين، ومكافآت العقد الخفيفة) على المكافئ الأصلي بـ uqor.

---

## أمثلة على الاستخدام

### دفع الرسوم بـ USDC

أرسِل عملية نقل رموز مع دفع الرسوم بـ USDC:

```bash
qorechaind tx bank send mykey qor1recipient... 5000000uqor \
  --chain-id qorechain-diana \
  --fees 500ibc/USDC
```

نظراً لأن USDC له معدّل تحويل 1.0، فإن 500 ibc/USDC تكافئ 500 uqor.

### دفع الرسوم بـ ATOM

أرسِل عملية نقل رموز مع دفع الرسوم بـ ATOM:

```bash
qorechaind tx bank send mykey qor1recipient... 5000000uqor \
  --chain-id qorechain-diana \
  --fees 50ibc/ATOM
```

نظراً لأن ATOM له معدّل تحويل 10.0، فإن 50 ibc/ATOM تكافئ 500 uqor.

---

## الاستعلام عن الرموز المقبولة

استرجِع قائمة الرموز المقبولة حالياً لتجريد الغاز، إلى جانب معدّلات تحويلها:

```bash
qorechaind query gasabstraction accepted-tokens
```

**مثال على المُخرَج:**

```yaml
accepted_tokens:
- denom: uqor
  conversion_rate: "1.000000000000000000"
- denom: ibc/USDC
  conversion_rate: "1.000000000000000000"
- denom: ibc/ATOM
  conversion_rate: "10.000000000000000000"
```

---

## الوصول عبر JSON-RPC

للتطبيقات المتكاملة عبر JSON-RPC، استعلم عن تهيئة تجريد الغاز:

```
qor_getGasAbstractionConfig
```

**الطلب:**

```json
{
  "jsonrpc": "2.0",
  "method": "qor_getGasAbstractionConfig",
  "params": [],
  "id": 1
}
```

**الاستجابة:**

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "accepted_tokens": [
      { "denom": "uqor", "conversion_rate": "1.0" },
      { "denom": "ibc/USDC", "conversion_rate": "1.0" },
      { "denom": "ibc/ATOM", "conversion_rate": "10.0" }
    ]
  }
}
```

---

:::tip

* تجريد الغاز مثالي للمستخدمين القادمين من منظومات أخرى الذين قد لا يمتلكون QOR بعد.
* تُحدَّد معدّلات التحويل من قِبل الحوكمة وقد يجري تحديثها عبر مقترحات تغيير المعاملات.
* إذا كنت تمتلك عدة رموز مقبولة، فيمكن استخدام أيٍّ منها للرسوم في أي نوع من المعاملات.
* يُخصم الرمز الفعلي المحدّد في `--fees` من حسابك. يُستخدم التحويل فقط للتحقّق من أن الرسوم تستوفي الحد الأدنى المطلوب.

:::
