# إعادة تخزين BTC (Babylon)

تدمج وحدة `x/babylon` بين QoreChain وبروتوكول Babylon لوراثة ضمانات النهائية القائمة على إثبات العمل (proof-of-work) لـ Bitcoin. ومن خلال إعادة تخزين BTC، تكتسب QoreChain طبقة نهائية ثانوية مدعومة بقوة تجزئة (hashrate) Bitcoin — دون الحاجة إلى أي تغييرات في بروتوكول Bitcoin نفسه.

## نظرة عامة

يمكّن بروتوكول Babylon سلاسل إثبات الحصة (proof-of-stake) من الاستفادة من أمان Bitcoin عبر آلية للطابع الزمني ونقاط التفتيش. يعمل تكامل QoreChain على النحو التالي:

1. يقفل **مخزِّنو BTC** عملة Bitcoin في معاملات تخزين Babylon ويسجّلون مراكزهم على QoreChain.
2. تُنقل **نقاط تفتيش الحقب (epoch checkpoints)** من QoreChain دورياً إلى Babylon، التي تختمها بطابع زمني على Bitcoin.
3. **وراثة النهائية**: بمجرد وضع نقطة تفتيش لحقبة QoreChain على Bitcoin، ترث الحالة التي تغطيها تلك الحقبة ضمانات نهائية إثبات العمل لـ Bitcoin.

يوفر هذا دفاعاً ضد هجمات المدى الطويل والتواقيع المتناقضة، مرتكزاً على قوة التجزئة المتراكمة لـ Bitcoin بدلاً من الاعتماد فقط على مجموعة مدققي QoreChain الخاصة.

## مراكز تخزين BTC

يمكن للمستخدمين تسجيل مراكز تخزين BTC على QoreChain بتقديم معاملة `MsgBTCRestake` تشير إلى معاملة تخزين Bitcoin.

### متطلبات التسجيل

| المعامل               | القيمة                        | الوصف                                       |
| ----------------------- | ---------------------------- | ------------------------------------------------- |
| **الحد الأدنى للتخزين**       | 100,000 ساتوشي (0.001 BTC) | الحد الأدنى من BTC المطلوب لكل مركز تخزين         |
| **فترة فك التخزين (unbonding)**    | 144 كتلة BTC (\~يوم واحد)     | فترة الانتظار قبل إمكانية سحب BTC المخزَّن |
| **فاصل نقطة التفتيش** | كل 10 حقب QoreChain    | عدد المرات التي توضع فيها نقطة تفتيش للحالة على Babylon        |

### بنية مركز التخزين

يتتبع كل مركز تخزين BTC الحالة التالية على السلسلة:

| الحقل              | الوصف                                                     |
| ------------------ | --------------------------------------------------------------- |
| `staker_address`   | عنوان QoreChain للمخزِّن (`qor1...`)                     |
| `btc_tx_hash`      | تجزئة معاملة Bitcoin لمعاملة التخزين             |
| `amount_satoshis`  | مقدار BTC المخزَّن بالساتوشي                                |
| `status`           | حالة دورة حياة المركز: `active` أو `unbonding` أو `withdrawn` |
| `staked_at`        | الطابع الزمني لتسجيل المركز                                |
| `unbonding_height` | ارتفاع الكتلة الذي بدأ عنده فك التخزين (إن وُجد)   |
| `validator_addr`   | عنوان مدقق QoreChain المُفوَّض إليه هذا التخزين          |

### تدفق التسجيل

1. **إنشاء معاملة تخزين BTC** — على شبكة Bitcoin، أنشئ معاملة تخزين BTC.
2. **تقديم MsgBTCRestake على QoreChain** — على QoreChain، قدّم `MsgBTCRestake` مع `btc_tx_hash` و`amount` و`validator`.
3. **تسجيل المركز** — يُسجَّل المركز على السلسلة بحالة "active".

## نقاط تفتيش الحقب

تُوضع جذور حالة حقب QoreChain دورياً كنقاط تفتيش على Bitcoin عبر سلسلة ترحيل Babylon.

### تدفق نقطة التفتيش

1. **تقديم نقطة التفتيش** — يقدّم مدقق QoreChain `MsgSubmitBTCCheckpoint` يتضمن رقم الحقبة، وتجزئة كتلة BTC، وارتفاع كتلة BTC، وجذر حالة QoreChain.
2. **ترحيل IBC** — تُنقل بيانات نقطة التفتيش إلى سلسلة Babylon عبر IBC.
3. **الختم الزمني على Bitcoin** — تُضمِّن Babylon نقطة التفتيش في معاملة Bitcoin، مرسيةً حالة QoreChain على سلسلة كتل Bitcoin.
4. **التأكيد** — بمجرد تأكيد معاملة Bitcoin، تتدفق النهائية رجوعاً عبر Babylon إلى QoreChain.
5. **الإنهاء** — تنتقل حالة نقطة التفتيش من `pending` إلى `confirmed` إلى `finalized`.

### بنية نقطة التفتيش

| الحقل              | الوصف                                              |
| ------------------ | -------------------------------------------------------- |
| `epoch_num`        | رقم حقبة QoreChain التي توضع لها نقطة تفتيش                |
| `btc_block_hash`   | تجزئة كتلة Bitcoin التي تحتوي على نقطة التفتيش             |
| `btc_block_height` | ارتفاع كتلة Bitcoin                                     |
| `state_root`       | جذر حالة QoreChain عند حدود الحقبة               |
| `submitted_at`     | الطابع الزمني لتقديم نقطة التفتيش                       |
| `status`           | حالة نقطة التفتيش: `pending` أو `confirmed` أو `finalized` |

### لقطات الحقب

عند كل حدود نقطة تفتيش، تلتقط لقطة حقبة حالة الشبكة الإجمالية:

| الحقل              | الوصف                                      |
| ------------------ | ------------------------------------------------ |
| `total_staked`     | إجمالي BTC المخزَّن عبر جميع المراكز (ساتوشي) |
| `active_positions` | عدد مراكز التخزين النشطة               |
| `validator_count`  | عدد المدققين ذوي التفويضات المدعومة بـ BTC |
| `block_height`     | ارتفاع كتلة QoreChain عند اللقطة               |

## طبقة النهائية الثانوية

يوفر تكامل Babylon **ضمان نهائية ثانوي** يكمّل نهائية الإجماع الأصلية لـ QoreChain:

| طبقة النهائية | المصدر                     | السرعة        | الأمان                                |
| -------------- | -------------------------- | ------------ | --------------------------------------- |
| **الأساسية**    | محرك إجماع QoreChain | \~5 ثوانٍ  | مدعومة بحصة QOR + تواقيع PQC    |
| **الثانوية**  | Babylon + Bitcoin          | \~60 دقيقة | مدعومة بقوة التجزئة التراكمية لـ Bitcoin |

تكون الطبقة الثانوية ذات قيمة خاصة لـ:

* **منع هجمات المدى الطويل**: حتى لو راكم مهاجم حصة QOR كبيرة، لا يمكنه إعادة كتابة تاريخ وُضعت له نقطة تفتيش على Bitcoin.
* **أمان الجسر عبر السلاسل**: يمكن لعمليات الجسر التي تشمل قيماً كبيرة الانتظار للنهائية على مستوى Bitcoin قبل تحرير الأموال.
* **ثقة المؤسسات**: يوفر الطابع الزمني لـ Bitcoin إثباتاً قابلاً للتحقق بشكل مستقل لتاريخ حالة QoreChain.

## التهيئة

| المعامل             | الافتراضي          | الوصف                               |
| --------------------- | ---------------- | ----------------------------------------- |
| `enabled`             | `false`          | مفتاح رئيسي لميزات إعادة تخزين BTC  |
| `min_stake_amount`    | 100,000 ساتوشي | الحد الأدنى من BTC لكل مركز تخزين          |
| `unbonding_period`    | 144 كتلة BTC   | مدة فك التخزين المُقوَّمة بـ BTC        |
| `checkpoint_interval` | 10 حقب        | الحقب بين نقاط تفتيش Babylon        |
| `babylon_chain_id`    | `bbn-1`          | معرّف سلسلة شبكة Babylon المتصلة |

## الأحداث

تُصدر الوحدة الأحداث التالية على السلسلة:

| نوع الحدث               | السمات                               | الوصف                                    |
| ------------------------ | ---------------------------------------- | ---------------------------------------------- |
| `btc_restake`            | staker، btc\_tx\_hash، amount، validator | تسجيل مركز تخزين BTC جديد            |
| `btc_unbond`             | staker، amount                           | دخول مركز تخزين BTC في فك التخزين         |
| `btc_checkpoint`         | epoch، checkpoint\_id                    | تقديم نقطة تفتيش حقبة إلى Babylon          |
| `babylon_epoch_complete` | epoch                                    | إنهاء حقبة Babylon بطابع زمني من Bitcoin |

## أوامر CLI

### أوامر الاستعلام

```bash
# Query BTC restaking configuration
qorechaind query babylon config

# Query a staking position by address
qorechaind query babylon position <staker-address>
```

### أوامر المعاملات

```bash
# Register a BTC restaking position
qorechaind tx babylon restake \
  --btc-tx-hash <hash> \
  --amount-satoshis <amount> \
  --validator <qorvaloper1...> \
  --from <key-name>

# Submit a BTC checkpoint
qorechaind tx babylon submit-checkpoint \
  --epoch <epoch-number> \
  --btc-block-hash <hash> \
  --btc-block-height <height> \
  --state-root <root> \
  --from <key-name>
```
