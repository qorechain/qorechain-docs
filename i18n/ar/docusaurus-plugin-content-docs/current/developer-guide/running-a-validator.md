# تشغيل مُدقّق

يشرح هذا الدليل كيفية إنشاء مُدقّق على شبكة QoreChain، وفهم نظام تصنيف المجمّعات، وتسجيل مفتاح PQC للحصول على أمان مقاوم للحوسبة الكمّية، ومراقبة العقدة الخاصة بك.

:::note
يستهدف هذا الدليل الشبكة الرئيسية **`qorechain-vladi`** (مُعرّف سلسلة EVM رقم **9801**)، وهي قيد التشغيل منذ 7 يونيو 2026 بإصدار السلسلة **v3.1.80**. يُوصى باستخدام شبكة الاختبار **`qorechain-diana`** (مُعرّف سلسلة EVM رقم **9800**) للتدرّب على إعدادك قبل الانتقال إلى التشغيل الفعلي. استبدل القيمة المناسبة لـ `--chain-id` الخاصة بالشبكة المستهدفة.
:::

---

## المتطلبات المسبقة

* عقدة `qorechaind` متزامنة بالكامل (انظر [الاتصال بشبكة الاختبار](/getting-started/connecting-to-testnet))
* حساب مُموّل بما لا يقل عن **1,000 QOR** (1,000,000,000 uqor) من أجل التفويض الذاتي الأولي
* الإلمام بنموذج [التخزين والتفويض](/user-guide/staking-and-delegation)

## إنشاء مُدقّق

```bash
qorechaind tx staking create-validator \
  --amount 1000000000uqor \
  --pubkey $(qorechaind comet show-validator) \
  --moniker "my-validator" \
  --commission-rate 0.10 \
  --commission-max-rate 0.20 \
  --commission-max-change-rate 0.01 \
  --min-self-delegation 1 \
  --from mykey \
  --gas auto \
  --gas-adjustment 1.3 \
  -y
```

| المُعامل                        | الوصف                                              |
| ------------------------------ | -------------------------------------------------- |
| `--amount`                     | مقدار التفويض الذاتي (الحد الأدنى للتخزين)          |
| `--pubkey`                     | المفتاح العام للإجماع الخاص بالمُدقّق (ed25519)     |
| `--moniker`                    | اسم قابل للقراءة البشرية للمُدقّق الخاص بك          |
| `--commission-rate`            | معدل العمولة الأولي (مثال: 0.10 = 10%)             |
| `--commission-max-rate`        | الحد الأقصى لمعدل العمولة (غير قابل للتغيير بعد الإنشاء) |
| `--commission-max-change-rate` | الحد الأقصى لمعدل التغيير اليومي للعمولة            |
| `--min-self-delegation`        | الحد الأدنى من الرموز التي يجب أن يفوّضها المُشغّل ذاتياً |

بعد تأكيد المعاملة، تحقّق من المُدقّق الخاص بك:

```bash
qorechaind query staking validator $(qorechaind keys show mykey --bech val -a)
```

## تصنيف المجمّعات

تستخدم QoreChain **نظام تصنيف ثلاثي المجمّعات** يُدار بواسطة وحدة `x/qca` (تخصيص الإجماع الكمّي). كل **1,000 كتلة**، تتم إعادة تصنيف المُدقّقين إلى أحد ثلاثة مجمّعات بناءً على سمعتهم وحصة التخزين:

| المجمّع                              | المعايير                                          | تخصيص الكتل      |
| ------------------------------------ | ------------------------------------------------- | ---------------- |
| **RPoS** (إثبات الحصة بالسمعة)        | السمعة >= المئين السبعون و حصة التخزين >= الوسيط   | 40% من الكتل     |
| **DPoS** (إثبات الحصة المُفوّض)       | إجمالي التفويض >= 10,000 QOR                       | 35% من الكتل     |
| **PoS** (إثبات الحصة)                 | جميع المُدقّقين النشطين المتبقّين                  | 25% من الكتل     |

داخل كل مجمّع، يتم اختيار مقترحي الكتل باستخدام **الاختيار العشوائي المرجّح** بما يتناسب مع حصة التخزين الفعلية لديهم. يضمن التصنيف أن يحظى كل من المُدقّقين ذوي السمعة العالية والمُدقّقين ذوي التفويض العالي بتمثيل عادل، مع السماح في الوقت ذاته للمُدقّقين الأصغر بالمشاركة.

### الاستعلام عن تصنيف المجمّع الخاص بك

```bash
qorechaind query qca pool-classification $(qorechaind keys show mykey --bech val -a)
```

عبر JSON-RPC:

```bash
curl -X POST http://localhost:8545 \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "qor_getPoolClassification",
    "params": ["qorvaloper1..."],
    "id": 1
  }'
```

## منحنى الربط

تُحدَّد مكافأة التخزين للمُدقّق بواسطة منحنى ربط يدمج عدة عوامل:

```
R = beta * S * (1 + alpha * log(1 + L)) * Q(r) * P(t)
```

| المتغيّر | الوصف                                                      |
| -------- | ---------------------------------------------------------- |
| `R`      | مقدار المكافأة                                             |
| `beta`   | معدل المكافأة الأساسي                                      |
| `S`      | حصة التخزين الفعلية                                        |
| `alpha`  | ثابت تحجيم الولاء                                          |
| `L`      | مدة الولاء (مدة التخزين المتواصلة)                          |
| `Q(r)`   | عامل جودة السمعة، النطاق \[0.75 - 1.25]                    |
| `P(t)`   | مُضاعِف مرحلة البروتوكول (يتعدّل عبر دورة حياة الشبكة)      |

**أبرز النقاط:**

* **مكافأة مدة الولاء:** يحصل المُدقّقون الذين يخزّنون بشكل متواصل على مكافآت متزايدة عبر حد الولاء اللوغاريتمي. يحفّز ذلك على الالتزام طويل الأمد.
* **عامل جودة السمعة:** يتراوح من 0.75 (سمعة ضعيفة) إلى 1.25 (سمعة ممتازة). تُحتسب السمعة من وقت التشغيل، والمقترحات الناجحة، والمشاركة المجتمعية، وجودة التحقق من المعاملات.
* **مُضاعِف مرحلة البروتوكول:** يتعدّل مع نضج الشبكة عبر مراحل مختلفة (التمهيد، النمو، النضج).

## التخفيض التدريجي

تستخدم QoreChain نموذج **تخفيض تدريجي** يصعّد العقوبات على المخالفين المتكررين مع السماح للمُدقّقين بالتعافي بمرور الوقت:

```
penalty = base_rate * escalation^effective_count * severity
```

| المُعامل                       | القيمة          |
| ---------------------------- | -------------- |
| الحد الأقصى للعقوبة لكل حدث     | 33% من الحصة   |
| عمر النصف للاضمحلال            | 100,000 كتلة   |
| خطورة التوقف                   | 1.0            |
| خطورة التوقيع المزدوج          | 2.0            |
| خطورة هجوم العميل الخفيف        | 3.0            |

1. **كل مخالفة تزيد العدد الفعلي.** كل مخالفة (توقف، توقيع مزدوج، إلخ) تزيد من العدد الفعلي للمُدقّق، مما يؤثر على العقوبات المستقبلية.

2. **العقوبة تتصاعد أُسّياً.** تتصاعد العقوبة بناءً على العدد الفعلي باستخدام الصيغة أعلاه، لذا يواجه المخالفون المتكررون عقوبات أكبر بكثير.

3. **العدد الفعلي يضمحلّ بمرور الوقت.** يضمحلّ العدد الفعلي بعمر نصف يبلغ 100,000 كتلة (\~7 أيام عند كتل بمعدل 6 ثوانٍ)، مما يسمح للمُدقّقين بالتعافي بعد فترة من السلوك الجيد.

4. **الأحداث المنفردة مقابل المخالفات المتكررة.** يؤدي حدث توقف عرضي منفرد إلى عقوبة طفيفة، بينما تؤدي المخالفات المتكررة إلى عواقب تتزايد أُسّياً.

## تسجيل مفتاح PQC

يمكن للمُدقّقين اختيارياً تسجيل **مفتاح عام للتشفير ما بعد الكمّي (PQC)** باستخدام خوارزمية ML-DSA-87. يوفّر ذلك أماناً مقاوماً للحوسبة الكمّية لهوية المُدقّق ويمكن استخدامه للتوقيع الهجين.

```bash
qorechaind tx pqc register-key <pubkey-hex> hybrid \
  --from mykey \
  --gas auto \
  -y
```

| المُعامل        | الوصف                                             |
| -------------- | ------------------------------------------------- |
| `<pubkey-hex>` | مفتاح عام ML-DSA-87 بحجم 2592 بايت بترميز سداسي عشري |
| `hybrid`       | وضع التسجيل (hybrid = كلاسيكي + PQC معاً)          |

تحقّق من التسجيل:

```bash
qorechaind query pqc key <account-address>
```

:::tip
**توصية:** تسجيل مفتاح PQC اختياري لكنه موصى به بشدة للمُدقّقين العاملين على الشبكة الرئيسية. فهو يوفّر دفاعاً استشرافياً ضد تهديدات الحوسبة الكمّية.
:::

## المراقبة

### مقاييس Prometheus

تكشف QoreChain مقاييس Prometheus على المنفذ **26660**:

```
http://localhost:26660/metrics
```

المقاييس الرئيسية التي ينبغي مراقبتها:

| المقياس                          | الوصف                                            |
| ------------------------------- | ----------------------------------------------- |
| `qorechain_missed_blocks_total` | إجمالي الكتل التي فوّتها المُدقّق الخاص بك        |
| `qorechain_validator_uptime`    | نسبة وقت التشغيل عبر آخر N من الكتل               |
| `qorechain_reputation_score`    | درجة السمعة الحالية                              |
| `qorechain_pool_classification` | تخصيص المجمّع الحالي (0=PoS، 1=DPoS، 2=RPoS)     |
| `qorechain_consecutive_signed`  | الكتل المُوقّعة المتتالية                        |
| `consensus_height`              | ارتفاع الكتلة الحالي                             |
| `consensus_rounds`              | جولات الإجماع للارتفاع الحالي                    |

### الاستعلام عن درجة السمعة

```bash
qorechaind query reputation score $(qorechaind keys show mykey --bech val -a)
```

عبر JSON-RPC:

```bash
curl -X POST http://localhost:8545 \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "qor_getReputationScore",
    "params": ["qorvaloper1..."],
    "id": 1
  }'
```

### فحوصات السلامة

```bash
# Node status
qorechaind status | jq '.sync_info'

# Validator signing info (uptime, missed blocks)
qorechaind query slashing signing-info $(qorechaind comet show-validator)

# Check if your validator is in the active set
qorechaind query staking validators --status bonded | grep "my-validator"
```

## أفضل الممارسات التشغيلية

1. **استخدم بنية عقدة الحراسة (sentry node).** شغّل المُدقّق الخاص بك خلف عقد حراسة لحمايته من هجمات حجب الخدمة الموزّعة (DDoS). اكشف فقط عقد الحراسة للشبكة العامة.

2. **أعدّ نظام التنبيهات.** هيّئ تنبيهات للكتل المُفوّتة، وانخفاض وقت التشغيل، وعمليات إعادة التشغيل غير المتوقعة. تفويت بضع كتل أمر طبيعي؛ أما التفويتات المستمرة فستؤدي إلى تخفيض.

3. **حافظ على وقت تشغيل مرتفع.** يكافئ نظام السمعة وقت التشغيل المتسق. يؤدي التوقف الممتد إلى تدهور عامل جودة السمعة لديك، مما يقلّل المكافآت.

4. **حافظ على تحديث البرمجيات.** تابع إصدارات QoreChain وطبّق التحديثات على الفور. نسّق مع مجتمع المُدقّقين لترقيات السلسلة.

5. **أمّن مفاتيحك.** استخدم وحدة أمان عتادية (HSM) أو موقّعاً عن بُعد لمفتاح الإجماع الخاص بالمُدقّق. لا تخزّن المفاتيح أبداً على الجهاز نفسه الذي تعمل عليه العقدة.

6. **سجّل مفتاح PQC.** اجعل المُدقّق الخاص بك جاهزاً للمستقبل ضد التهديدات الكمّية عبر تسجيل مفتاح ML-DSA-87.

7. **راقب مجمّعك.** تابع تصنيف مجمّعك كل 1,000 كتلة. يمكن أن يؤدي تحسين سمعتك إلى نقلك من PoS إلى RPoS، مما يزيد بشكل كبير فرص اقتراح الكتل.

## مرجع أوامر المُدقّق

```bash
# Edit validator metadata
qorechaind tx staking edit-validator \
  --moniker "new-name" \
  --website "https://myvalidator.com" \
  --details "Description of my validator" \
  --from mykey -y

# Unjail after downtime slashing
qorechaind tx slashing unjail --from mykey -y

# Delegate additional stake
qorechaind tx staking delegate $(qorechaind keys show mykey --bech val -a) \
  500000000uqor --from mykey -y

# Withdraw rewards
qorechaind tx distribution withdraw-rewards $(qorechaind keys show mykey --bech val -a) \
  --commission --from mykey -y
```

## التحقق من الشبكات المتصلة {#connected-networks}

اعتباراً من إصدار السلسلة **v3.1.80**، يمكن لمُدقّق QoreChain أيضاً المساعدة في التحقق من الشبكات المتصلة عبر [الجسر](/architecture/bridge-architecture). هذا الأمر **مقيّد بالترخيص وقائم على الاشتراك الاختياري**:

1. **احتفظ بالترخيص.** يجب أن يحتفظ المُدقّق بترخيص `validator_<chain>` (أو `qcb_bridge`) نشط للشبكة المستهدفة. يرفض المُنسّق بدء تشغيل عميل خارجي بدونه (فشل مغلق).
2. **التفعيل يوفّر العميل تلقائياً.** عند تفعيل الترخيص، توفّر QoreChain عميل الشبكة المطابق على عقدتك — بتنزيل العميل المُثبّت، وتوليد إعداده، وتشغيله ضمن تنسيق QoreChain. لا يتم جلب أي شيء حتى التفعيل.
3. **وفّر مفاتيح الشبكة وحصة التخزين.** مفاتيح المُدقّق/حصة التخزين ومفاتيح التوقيع للشبكة الخارجية **يوفّرها المُشغّل** لكل شبكة؛ تقدّم QoreChain إطار عمل المُشغّلات وبوابة الترخيص المفروضة، وليس حصة التخزين الخاصة بك على السلسلة الخارجية.

تتوفّر مُشغّلات لجميع **37 شبكة جسر**، مُصنّفة بحسب كيفية مشاركة المُدقّق:

| الفئة | المشاركة | الأمثلة |
| ----- | ------------- | -------- |
| مُدقّق بلا أذونات | التخزين والتشغيل | Solana, Ethereum, Avalanche, Sui, Aptos, Cardano, Tezos, Algorand, Starknet |
| مُقيّد / مُنتخَب / قبول | التخزين، خاضع لسقف أو انتخاب | BSC, Polygon, Polkadot, TRON, Sei, Injective, NEAR, Hedera |
| عقدة كاملة L2 | تشغيل عقدة كاملة (بلا تخزين) | Optimism, Base, zkSync Era, Linea, Scroll, Arbitrum |
| بلا تخزين / قائمة ثقة | المراقبة / المشاركة بلا تخزين | Bitcoin, Filecoin, XRPL, Stellar |

:::note
تثبيتات إصدارات العميل تُبذل بأفضل جهد؛ تحقّق من إصدار العميل المنبع للشبكة المستهدفة قبل التفعيل في بيئة الإنتاج.
:::
