# تشغيل مدقق

يغطّي هذا الدليل كيفية إنشاء مدقق على شبكة QoreChain، وفهم نظام تصنيف المجمّعات، وتسجيل مفتاح PQC للأمان المقاوم للكمّ، ومراقبة عقدتك.

:::note
يستهدف هذا الدليل الشبكة الرئيسية **`qorechain-vladi`** (معرّف سلسلة EVM **9801**)، قيد التشغيل منذ 7 يونيو 2026 بإصدار السلسلة **v3.1.77**. يُوصى بشبكة الاختبار **`qorechain-diana`** (معرّف سلسلة EVM **9800**) للتدرّب على إعدادك قبل الانطلاق. استبدل `--chain-id` المناسب لشبكتك المستهدفة.
:::

---

## المتطلبات المسبقة

* عقدة `qorechaind` متزامنة بالكامل (راجع [الاتصال بشبكة الاختبار](/getting-started/connecting-to-testnet))
* حساب مموّل بما لا يقل عن **1,000 QOR** (1,000,000,000 uqor) للتفويض الذاتي الأولي
* الإلمام بنموذج [الرهان والتفويض](/user-guide/staking-and-delegation)

## إنشاء مدقق

```bash
qorechaind tx staking create-validator \
  --amount 1000000000uqor \
  --pubkey $(qorechaind comet show-validator) \
  --moniker "my-validator" \
  --commission-rate 0.10 \
  --commission-max-rate 0.20 \
  --commission-max-change-rate 0.01 \
  --min-self-delegation 1 \
  --from mykey \
  --gas auto \
  --gas-adjustment 1.3 \
  -y
```

| المعامل                        | الوصف                                              |
| ------------------------------ | -------------------------------------------------- |
| `--amount`                     | مبلغ التفويض الذاتي (الحد الأدنى للرهان)            |
| `--pubkey`                     | مفتاح إجماع المدقق العام (ed25519)                  |
| `--moniker`                    | اسم قابل للقراءة لمدققك                             |
| `--commission-rate`            | معدل العمولة الأولي (مثلًا، 0.10 = 10%)            |
| `--commission-max-rate`        | الحد الأقصى لمعدل العمولة (ثابت بعد الإنشاء)        |
| `--commission-max-change-rate` | الحد الأقصى لمعدل تغيير العمولة اليومي              |
| `--min-self-delegation`        | الحد الأدنى من الرموز التي يجب أن يفوّضها المشغّل ذاتيًا |

بعد تأكيد المعاملة، تحقق من مدققك:

```bash
qorechaind query staking validator $(qorechaind keys show mykey --bech val -a)
```

## تصنيف المجمّعات

تستخدم QoreChain **نظام تصنيف بثلاثة مجمّعات** تديره وحدة `x/qca` (تخصيص الإجماع الكمّي). كل **1,000 كتلة**، يُعاد تصنيف المدققين إلى أحد ثلاثة مجمّعات بناءً على سمعتهم ورهانهم:

| المجمّع                               | المعايير                                          | تخصيص الكتل      |
| ------------------------------------ | ------------------------------------------------- | ---------------- |
| **RPoS** (إثبات الحصة بالسمعة)        | السمعة >= المئين 70 و الرهان >= الوسيط             | 40% من الكتل     |
| **DPoS** (إثبات الحصة المفوّض)        | إجمالي التفويض >= 10,000 QOR                       | 35% من الكتل     |
| **PoS** (إثبات الحصة)                 | جميع المدققين النشطين المتبقّين                    | 25% من الكتل     |

داخل كل مجمّع، يُختار مقترحو الكتل باستخدام **اختيار عشوائي مرجّح** متناسب مع رهانهم الفعلي. يضمن التصنيف أن يحظى كل من المدققين عالي السمعة وعالي التفويض بتمثيل عادل، مع السماح في الوقت ذاته للمدققين الأصغر بالمشاركة.

### الاستعلام عن تصنيف مجمّعك

```bash
qorechaind query qca pool-classification $(qorechaind keys show mykey --bech val -a)
```

عبر JSON-RPC:

```bash
curl -X POST http://localhost:8545 \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "qor_getPoolClassification",
    "params": ["qorvaloper1..."],
    "id": 1
  }'
```

## منحنى الربط

تُحدَّد مكافأة الرهان لمدقق بمنحنى ربط يدمج عوامل متعددة:

```
R = beta * S * (1 + alpha * log(1 + L)) * Q(r) * P(t)
```

| المتغير  | الوصف                                                      |
| -------- | ---------------------------------------------------------- |
| `R`      | مبلغ المكافأة                                              |
| `beta`   | معدل المكافأة الأساسي                                      |
| `S`      | الرهان الفعلي                                              |
| `alpha`  | ثابت تحجيم الولاء                                          |
| `L`      | مدة الولاء (زمن الرهان المتواصل)                            |
| `Q(r)`   | عامل جودة السمعة، النطاق \[0.75 - 1.25]                     |
| `P(t)`   | مضاعِف مرحلة البروتوكول (يتكيّف عبر دورة حياة الشبكة)         |

**النقاط الرئيسية:**

* **مكافأة مدة الولاء:** المدققون الذين يراهنون باستمرار يتلقّون مكافآت متزايدة عبر مصطلح الولاء اللوغاريتمي. يحفّز هذا الالتزام طويل المدى.
* **عامل جودة السمعة:** يتراوح من 0.75 (سمعة ضعيفة) إلى 1.25 (سمعة ممتازة). تُحتسب السمعة من زمن التشغيل، والمقترحات الناجحة، والمشاركة المجتمعية، وجودة التحقق من المعاملات.
* **مضاعِف مرحلة البروتوكول:** يتكيّف مع نضوج الشبكة عبر مراحل مختلفة (التمهيد، النمو، النضج).

## التخفيض التدرّجي

تستخدم QoreChain نموذج **تخفيض تدرّجي** يصعّد العقوبات على المخالفين المتكررين مع السماح للمدققين بالتعافي مع الوقت:

```
penalty = base_rate * escalation^effective_count * severity
```

| المعامل                      | القيمة         |
| ---------------------------- | -------------- |
| الحد الأقصى للعقوبة لكل حدث   | 33% من الرهان  |
| نصف عمر الاضمحلال            | 100,000 كتلة   |
| شدّة التعطّل                  | 1.0            |
| شدّة التوقيع المزدوج          | 2.0            |
| شدّة هجوم العميل الخفيف       | 3.0            |

1. **كل مخالفة تزيد العدّ الفعلي.** كل مخالفة (تعطّل، توقيع مزدوج، إلخ) تزيد العدّ الفعلي للمدقق، ما يؤثر على العقوبات المستقبلية.

2. **العقوبة تتصاعد أسّيًا.** تتصاعد العقوبة بناءً على العدّ الفعلي باستخدام الصيغة أعلاه، لذا يواجه المخالفون المتكررون عقوبات أكبر بكثير.

3. **العدّ الفعلي يضمحلّ مع الوقت.** يضمحلّ العدّ الفعلي بنصف عمر قدره 100,000 كتلة (\~7 أيام عند كتل بزمن 6 ثوانٍ)، ما يتيح للمدققين التعافي بعد فترة من السلوك الجيد.

4. **الأحداث الفردية مقابل المخالفات المتكررة.** يؤدي حدث تعطّل عرضي واحد إلى عقوبة طفيفة، بينما تُطلق المخالفات المتكررة عواقب متزايدة أسّيًا.

## تسجيل مفتاح PQC

يمكن للمدققين اختياريًا تسجيل **مفتاح تشفير ما بعد كمّي (PQC) عام** باستخدام خوارزمية ML-DSA-87. يوفّر هذا أمانًا مقاومًا للكمّ لهوية المدقق ويمكن استخدامه للتوقيع الهجين.

```bash
qorechaind tx pqc register-key <pubkey-hex> hybrid \
  --from mykey \
  --gas auto \
  -y
```

| المعامل        | الوصف                                             |
| -------------- | ------------------------------------------------- |
| `<pubkey-hex>` | مفتاح ML-DSA-87 عام بحجم 2592 بايت بترميز سداسي     |
| `hybrid`       | وضع التسجيل (hybrid = كلاسيكي + PQC معًا)          |

تحقق من التسجيل:

```bash
qorechaind query pqc key <account-address>
```

:::tip
**التوصية:** تسجيل مفتاح PQC اختياري لكنه موصى به بشدة للمدققين العاملين على الشبكة الرئيسية. يوفّر دفاعًا استشرافيًا ضد تهديدات الحوسبة الكمّية.
:::

## المراقبة

### مقاييس Prometheus

تكشف QoreChain مقاييس Prometheus على المنفذ **26660**:

```
http://localhost:26660/metrics
```

المقاييس الرئيسية للمراقبة:

| المقياس                         | الوصف                                           |
| ------------------------------- | ----------------------------------------------- |
| `qorechain_missed_blocks_total` | إجمالي الكتل التي فوّتها مدققك                   |
| `qorechain_validator_uptime`    | نسبة زمن التشغيل خلال آخر N كتلة                  |
| `qorechain_reputation_score`    | درجة السمعة الحالية                              |
| `qorechain_pool_classification` | تخصيص المجمّع الحالي (0=PoS, 1=DPoS, 2=RPoS)     |
| `qorechain_consecutive_signed`  | الكتل المتتالية الموقّعة                          |
| `consensus_height`              | ارتفاع الكتلة الحالي                             |
| `consensus_rounds`              | جولات الإجماع للارتفاع الحالي                     |

### الاستعلام عن درجة السمعة

```bash
qorechaind query reputation score $(qorechaind keys show mykey --bech val -a)
```

عبر JSON-RPC:

```bash
curl -X POST http://localhost:8545 \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "qor_getReputationScore",
    "params": ["qorvaloper1..."],
    "id": 1
  }'
```

### فحوص الصحة

```bash
# Node status
qorechaind status | jq '.sync_info'

# Validator signing info (uptime, missed blocks)
qorechaind query slashing signing-info $(qorechaind comet show-validator)

# Check if your validator is in the active set
qorechaind query staking validators --status bonded | grep "my-validator"
```

## أفضل الممارسات التشغيلية

1. **استخدم بنية عقدة الحراسة.** شغّل مدققك خلف عقد حراسة لحمايته من هجمات DDoS. اكشف عقد الحراسة فقط للشبكة العامة.

2. **اضبط التنبيهات.** كوّن تنبيهات للكتل المفوّتة، وزمن التشغيل المنخفض، وإعادات التشغيل غير المتوقعة. بضع كتل مفوّتة أمر طبيعي؛ التفويت المستمر سيُطلق التخفيض.

3. **حافظ على زمن تشغيل عالٍ.** يكافئ نظام السمعة زمن التشغيل المتسق. يؤدي التعطّل الممتد إلى تدهور عامل جودة سمعتك، ما يقلّل المكافآت.

4. **أبقِ البرمجيات محدّثة.** تتبّع إصدارات QoreChain وطبّق التحديثات بسرعة. نسّق مع مجتمع المدققين لترقيات السلسلة.

5. **أمّن مفاتيحك.** استخدم وحدة أمان عتادية (HSM) أو موقّعًا عن بُعد لمفتاح إجماع المدقق. لا تخزّن المفاتيح أبدًا على الجهاز نفسه الذي تعمل عليه العقدة.

6. **سجّل مفتاح PQC.** حصّن مدققك مستقبليًا ضد التهديدات الكمّية بتسجيل مفتاح ML-DSA-87.

7. **راقب مجمّعك.** تتبّع تصنيف مجمّعك كل 1,000 كتلة. تحسين سمعتك قد ينقلك من PoS إلى RPoS، ما يزيد فرص اقتراح الكتل بشكل كبير.

## مرجع أوامر المدقق

```bash
# Edit validator metadata
qorechaind tx staking edit-validator \
  --moniker "new-name" \
  --website "https://myvalidator.com" \
  --details "Description of my validator" \
  --from mykey -y

# Unjail after downtime slashing
qorechaind tx slashing unjail --from mykey -y

# Delegate additional stake
qorechaind tx staking delegate $(qorechaind keys show mykey --bech val -a) \
  500000000uqor --from mykey -y

# Withdraw rewards
qorechaind tx distribution withdraw-rewards $(qorechaind keys show mykey --bech val -a) \
  --commission --from mykey -y
```
