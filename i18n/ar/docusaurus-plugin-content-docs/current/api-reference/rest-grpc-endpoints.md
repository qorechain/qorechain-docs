# نقاط نهاية REST / gRPC

توفّر QoreChain ثلاث واجهات رئيسية للوصول البرمجي:

| الواجهة | المنفذ الافتراضي | البروتوكول | الوصف                                    |
| ------- | ---------------- | ---------- | ---------------------------------------- |
| REST    | `1317`           | HTTP/1.1   | واجهة REST API لخدمة LCD (Light Client Daemon) |
| gRPC    | `9090`           | HTTP/2     | خدمة gRPC مرمَّزة بصيغة Protobuf          |
| RPC     | `26657`          | HTTP + WS  | واجهة RPC لمحرك إجماع QoreChain          |

تُعيد جميع نقاط نهاية REST بيانات بصيغة JSON. أما نقاط نهاية gRPC فتستخدم Protocol Buffers ويمكن استهلاكها بأي عميل gRPC. وتوفّر واجهة RPC استعلامات على مستوى الإجماع وبثّ المعاملات.

:::note
هذه الواجهات متاحة على كلٍّ من الشبكة الرئيسية **`qorechain-vladi`** (تعمل منذ 7 يونيو 2026 على إصدار السلسلة **v3.1.82**) وشبكة الاختبار **`qorechain-diana`**. تفترض عناوين URL الأساسية أدناه وجود عقدة تعمل محليًا؛ أما نقاط النهاية العامة المستضافة (`rpc/api/evm/svm.qore.host` ونسخها بلاحقة `-testnet`) فهي مدرجة في [الشبكات](/appendix/networks#public-endpoints).
:::

## عناوين URL الأساسية

```
REST:  http://localhost:1317
gRPC:  localhost:9090
RPC:   http://localhost:26657
```

## وحدة الجسر {#bridge-module}

اعتبارًا من إصدار السلسلة **v3.1.77**، أصبحت حالة القراءة فقط لوحدة الجسر متاحة عبر REST من خلال grpc-gateway تحت البادئة `/qorechain/bridge/v1/...` (كانت سابقًا متاحة عبر gRPC فقط). تقدّم هذه النقاط بيانات JSON حقيقية من على السلسلة عبر HTTP لمستكشفات السلسلة وقياسات العقد الخفيفة عن بُعد. يُظهر `config` الخاص بالجسر على سبيل المثال `min_validators=10` و `threshold=7`.

| الطريقة | نقطة النهاية                                | الوصف                                    |
| ------- | ------------------------------------------- | ----------------------------------------- |
| GET     | `/qorechain/bridge/v1/config`               | التكوين الحالي لوحدة الجسر                |
| GET     | `/qorechain/bridge/v1/chains`               | تسرد جميع سلاسل الجسر المسجَّلة           |
| GET     | `/qorechain/bridge/v1/chains/{chain_id}`    | تفاصيل سلسلة مجسّرة محددة                 |
| GET     | `/qorechain/bridge/v1/validators`           | تسرد مدقّقي الجسر المسجَّلين              |
| GET     | `/qorechain/bridge/v1/validators/{address}` | تفاصيل مدقّق جسر محدد                     |
| GET     | `/qorechain/bridge/v1/operations`           | تسرد عمليات الجسر                         |
| GET     | `/qorechain/bridge/v1/operations/{id}`      | تفاصيل عملية جسر محددة                    |

وتبقى نقاط النهاية التالية ذات المسارات الأقصر متاحة:

## وحدة الطبقات المتعددة {#multilayer-module}

اعتبارًا من إصدار السلسلة **v3.1.80**، أصبحت خدمة الاستعلام الكاملة لوحدة الطبقات المتعددة متاحة عبر REST من خلال grpc-gateway تحت البادئة `/qorechain/multilayer/v1/...` (كانت سابقًا متاحة عبر gRPC فقط)، بما في ذلك **استعلاما قراءة لمراسي الحالة**: يُعيد `anchor/{layer_id}` أحدث مرساة تسوية لطبقة ما، ويُعيد `anchors/{layer_id}` سجلّ مراسيها التاريخي. تحمل كل مرساة توقيع **ML-DSA-87 (Dilithium-5)** على حقولها القياسية، بحيث يمكن للعميل جلب المرساة والتحقق منها بشكل مستقل — وهذا هو الأساس على السلسلة الذي تعتمد عليه [إيصالات التسوية](/rollups/settlement-receipts) في Rollup Development Kit.

| الطريقة | نقطة النهاية                                     | الوصف                                          |
| ------- | ------------------------------------------------ | ----------------------------------------------- |
| GET     | `/qorechain/multilayer/v1/params`                | المعاملات الحالية لوحدة الطبقات المتعددة        |
| GET     | `/qorechain/multilayer/v1/layers`                | تسرد جميع الطبقات المسجَّلة                     |
| GET     | `/qorechain/multilayer/v1/layers/{layer_id}`     | تفاصيل طبقة محددة                               |
| GET     | `/qorechain/multilayer/v1/anchor/{layer_id}`     | أحدث مرساة حالة لطبقة ما                        |
| GET     | `/qorechain/multilayer/v1/anchors/{layer_id}`    | سجلّ مراسي الحالة لطبقة ما                      |
| GET     | `/qorechain/multilayer/v1/routing-stats`         | إحصاءات توجيه المعاملات عبر الطبقات             |

يحتوي `StateAnchorView` على `layer_id` و `layer_height` و `state_root` و `validator_set_hash` و `main_chain_height` و `anchored_at` و `pqc_aggregate_signature` و `transaction_count` و `compressed_state_proof`. والرسالة القياسية الموقَّعة هي `layer_id || layer_height || state_root || validator_set_hash`، ويجري التحقق منها مقابل مفتاح PQC المسجَّل لمنشئ الطبقة.

وتبقى نقاط النهاية التالية ذات المسارات الأقصر متاحة:

| الطريقة | نقطة النهاية                    | الوصف                                     |
| ------- | ------------------------------- | ------------------------------------------ |
| GET     | `/multilayer/v1/layer/{id}`     | تفاصيل طبقة محددة                          |
| GET     | `/multilayer/v1/layers`         | تسرد جميع الطبقات المسجَّلة                |
| GET     | `/multilayer/v1/anchor/{id}`    | تفاصيل سجل مرساة محدد                      |
| GET     | `/multilayer/v1/anchors`        | تسرد أحدث المراسي المقدَّمة                |
| GET     | `/multilayer/v1/routing-stats`  | إحصاءات توجيه المعاملات عبر الطبقات        |
| GET     | `/multilayer/v1/params`         | المعاملات الحالية لوحدة الطبقات المتعددة   |

## انعكاس gRPC

انعكاس خادم gRPC مفعَّل افتراضيًا، ما يتيح لأدوات مثل `grpcurl` اكتشاف الخدمات المتاحة:

```bash
grpcurl -plaintext localhost:9090 list
```

للاستعلام عن خدمة محددة:

```bash
grpcurl -plaintext localhost:9090 qorechain.pqc.v1.Query/Params
```

## المصادقة

جميع نقاط نهاية REST و gRPC غير مُصادَق عليها افتراضيًا. لعمليات النشر الإنتاجية، ضع وكيلًا عكسيًا (مثل Nginx أو Caddy) أمام العقدة للتعامل مع إنهاء TLS والتحكم في الوصول.
