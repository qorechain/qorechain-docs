# نقاط نهاية REST / gRPC

تكشف QoreChain عن ثلاث واجهات أساسية للوصول البرمجي:

| الواجهة | المنفذ الافتراضي | البروتوكول  | الوصف                        |
| --------- | ------------ | --------- | ---------------------------------- |
| REST      | `1317`       | HTTP/1.1  | واجهة LCD (عفريت العميل الخفيف) REST API |
| gRPC      | `9090`       | HTTP/2    | خدمة gRPC مُرمَّزة بـ Protobuf      |
| RPC       | `26657`      | HTTP + WS | RPC لمحرّك إجماع QoreChain     |

تُرجِع جميع نقاط نهاية REST بيانات بتنسيق JSON. تستخدم نقاط نهاية gRPC تنسيق Protocol Buffers ويمكن استهلاكها مع أي عميل gRPC. توفّر واجهة RPC استعلامات على مستوى الإجماع وبثّ المعاملات.

:::note
تتوفّر هذه الواجهات على كلٍّ من شبكة **`qorechain-vladi`** الرئيسية (تعمل منذ 7 يونيو 2026 على إصدار السلسلة **v3.1.77**) وشبكة **`qorechain-diana`** الاختبارية. تفترض عناوين URL الأساسية أدناه عقدة تعمل محليًا؛ استبدلها بمضيف الشبكة الرئيسية أو الاختبارية الخاص بمزوّدك للوصول عن بُعد.
:::

## عناوين URL الأساسية

```
REST:  http://localhost:1317
gRPC:  localhost:9090
RPC:   http://localhost:26657
```

## وحدة الجسر {#bridge-module}

اعتبارًا من إصدار السلسلة **v3.1.77**، تُكشَف حالة وحدة الجسر للقراءة فقط عبر REST من خلال grpc-gateway تحت البادئة `/qorechain/bridge/v1/...` (كانت سابقًا متاحة عبر gRPC فقط). تقدّم هذه النقاط بيانات JSON حقيقية على السلسلة عبر HTTP لأدوات الاستكشاف وقياسات العقد الخفيفة عن بُعد. يُبلِغ الجسر `config` على سبيل المثال عن `min_validators=10` و`threshold=7`.

| الطريقة | نقطة النهاية                               | الوصف                              |
| ------ | ------------------------------------------ | ---------------------------------------- |
| GET    | `/qorechain/bridge/v1/config`              | تهيئة وحدة الجسر الحالية      |
| GET    | `/qorechain/bridge/v1/chains`              | يسرد جميع سلاسل الجسر المسجَّلة       |
| GET    | `/qorechain/bridge/v1/chains/{chain_id}`   | تفاصيل سلسلة مجسورة محدّدة     |
| GET    | `/qorechain/bridge/v1/validators`          | يسرد مُدقِّقي الجسر المسجَّلين       |
| GET    | `/qorechain/bridge/v1/validators/{address}`| تفاصيل مُدقِّق جسر محدّد  |
| GET    | `/qorechain/bridge/v1/operations`          | يسرد عمليات الجسر                  |
| GET    | `/qorechain/bridge/v1/operations/{id}`     | تفاصيل عملية جسر محدّدة  |

تظل نقاط النهاية ذات المسارات الأقصر التالية متاحة:

| الطريقة | نقطة النهاية                        | الوصف                                    |
| ------ | ----------------------------------- | ---------------------------------------------- |
| GET    | `/bridge/v1/chains`                 | يسرد جميع سلاسل الجسر المسجَّلة             |
| GET    | `/bridge/v1/chains/{id}`            | تفاصيل سلسلة مجسورة محدّدة           |
| GET    | `/bridge/v1/validators`             | يسرد مُدقِّقي الجسر النشطين                 |
| GET    | `/bridge/v1/operations`             | يسرد عمليات الجسر الأخيرة                  |
| GET    | `/bridge/v1/operations/{id}`        | تفاصيل عملية جسر محدّدة        |
| GET    | `/bridge/v1/locked/{chain}/{asset}` | القيمة الإجمالية المقفلة لزوج سلسلة/أصل      |
| GET    | `/bridge/v1/limits/{chain}`         | حدود المعدّل والعتبات لسلسلة مجسورة |
| GET    | `/bridge/v1/estimate`               | تقدير رسوم الجسر ووقت التحويل         |

## انعكاس gRPC

انعكاس خادم gRPC مُفعَّل افتراضيًا، مما يتيح لأدوات مثل `grpcurl` اكتشاف الخدمات المتاحة:

```bash
grpcurl -plaintext localhost:9090 list
```

للاستعلام عن خدمة محدّدة:

```bash
grpcurl -plaintext localhost:9090 qorechain.pqc.v1.Query/Params
```

## المصادقة

جميع نقاط نهاية REST وgRPC غير مُصادَق عليها افتراضيًا. لعمليات النشر الإنتاجية، ضع وكيلًا عكسيًا (مثل Nginx أو Caddy) أمام العقدة للتعامل مع إنهاء TLS والتحكّم في الوصول.
