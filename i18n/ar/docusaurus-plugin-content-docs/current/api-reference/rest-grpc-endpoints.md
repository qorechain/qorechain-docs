# نقاط نهاية REST / gRPC

تكشف QoreChain عن ثلاث واجهات أساسية للوصول البرمجي:

| الواجهة | المنفذ الافتراضي | البروتوكول | الوصف                              |
| --------- | ------------ | --------- | ---------------------------------- |
| REST      | `1317`       | HTTP/1.1  | واجهة LCD (Light Client Daemon) REST API |
| gRPC      | `9090`       | HTTP/2    | خدمة gRPC مُرمّزة بـ Protobuf      |
| RPC       | `26657`      | HTTP + WS | QoreChain Consensus Engine RPC     |

تُعيد جميع نقاط نهاية REST بيانات JSON. تستخدم نقاط نهاية gRPC تنسيق Protocol Buffers ويمكن استهلاكها بأي عميل gRPC. توفّر واجهة RPC استعلامات على مستوى الإجماع وبثّ المعاملات.

:::note
تتوفّر هذه الواجهات على كلٍّ من شبكة **`qorechain-vladi`** الرئيسية (الفعّالة منذ 7 يونيو 2026 على إصدار السلسلة **v3.1.80**) وشبكة **`qorechain-diana`** الاختبارية. تفترض عناوين URL الأساسية أدناه وجود عقدة تعمل محلياً؛ استبدل بها مضيف الشبكة الرئيسية أو الاختبارية الخاص بمزوّدك للوصول عن بُعد.
:::

## عناوين URL الأساسية

```
REST:  http://localhost:1317
gRPC:  localhost:9090
RPC:   http://localhost:26657
```

## وحدة الجسر (Bridge Module) {#bridge-module}

اعتباراً من إصدار السلسلة **v3.1.77**، تُكشف الحالة للقراءة فقط لوحدة الجسر عبر REST من خلال grpc-gateway تحت البادئة `/qorechain/bridge/v1/...` (كانت سابقاً متاحة عبر gRPC فقط). تقدّم نقاط النهاية هذه بيانات JSON حقيقية على السلسلة عبر HTTP للمستكشفات وقياس عن بُعد للعُقد الخفيفة. يُبلّغ `config` الخاص بالجسر مثلاً عن `min_validators=10` و`threshold=7`.

| الطريقة | نقطة النهاية                                | الوصف                                    |
| ------ | ------------------------------------------ | ---------------------------------------- |
| GET    | `/qorechain/bridge/v1/config`              | الإعدادات الحالية لوحدة الجسر            |
| GET    | `/qorechain/bridge/v1/chains`              | يسرد جميع السلاسل المسجّلة للجسر          |
| GET    | `/qorechain/bridge/v1/chains/{chain_id}`   | تفاصيل سلسلة مجسورة محدّدة                |
| GET    | `/qorechain/bridge/v1/validators`          | يسرد المُحقّقين المسجّلين للجسر           |
| GET    | `/qorechain/bridge/v1/validators/{address}`| تفاصيل مُحقّق جسر محدّد                   |
| GET    | `/qorechain/bridge/v1/operations`          | يسرد عمليات الجسر                         |
| GET    | `/qorechain/bridge/v1/operations/{id}`     | تفاصيل عملية جسر محدّدة                   |

تظل نقاط النهاية التالية ذات المسار الأقصر متاحة:

| الطريقة | نقطة النهاية                         | الوصف                                          |
| ------ | ----------------------------------- | ---------------------------------------------- |
| GET    | `/bridge/v1/chains`                 | يسرد جميع السلاسل المسجّلة للجسر                |
| GET    | `/bridge/v1/chains/{id}`            | تفاصيل سلسلة مجسورة محدّدة                      |
| GET    | `/bridge/v1/validators`             | يسرد مُحقّقي الجسر النشطين                      |

| GET    | `/bridge/v1/operations/{id}`        | تفاصيل عملية جسر محدّدة                         |

## الوحدة متعددة الطبقات (Multilayer Module) {#multilayer-module}

اعتباراً من إصدار السلسلة **v3.1.80**، تُكشف خدمة الاستعلام الكاملة للوحدة متعددة الطبقات عبر REST من خلال grpc-gateway تحت البادئة `/qorechain/multilayer/v1/...` (كانت سابقاً متاحة عبر gRPC فقط)، بما في ذلك استعلامان للقراءة لـ**مرساة الحالة (state-anchor)**: يُعيد `anchor/{layer_id}` أحدث مرساة تسوية لطبقة، ويُعيد `anchors/{layer_id}` سجلّ مراسيها. تحمل كل مرساة توقيع **ML-DSA-87 (Dilithium-5)** على حقولها القانونية، بحيث يمكن للعميل جلب مرساة والتحقق منها بشكل مستقل — وهو الأساس على السلسلة لـ[إيصالات التسوية](/rollups/settlement-receipts) في Rollup Development Kit.

| الطريقة | نقطة النهاية                                     | الوصف                                             |
| ------ | ----------------------------------------------- | ------------------------------------------------- |
| GET    | `/qorechain/multilayer/v1/params`               | معاملات الوحدة متعددة الطبقات الحالية              |
| GET    | `/qorechain/multilayer/v1/layers`               | يسرد جميع الطبقات المسجّلة                         |
| GET    | `/qorechain/multilayer/v1/layers/{layer_id}`    | تفاصيل طبقة محدّدة                                 |
| GET    | `/qorechain/multilayer/v1/anchor/{layer_id}`    | أحدث مرساة حالة لطبقة                              |
| GET    | `/qorechain/multilayer/v1/anchors/{layer_id}`   | سجلّ مراسي الحالة لطبقة                            |
| GET    | `/qorechain/multilayer/v1/routing-stats`        | إحصائيات توجيه المعاملات عبر الطبقات               |

يحتوي `StateAnchorView` على `layer_id` و`layer_height` و`state_root` و`validator_set_hash` و`main_chain_height` و`anchored_at` و`pqc_aggregate_signature` و`transaction_count` و`compressed_state_proof`. الرسالة القانونية الموقّعة هي `layer_id || layer_height || state_root || validator_set_hash`، ويتم التحقق منها مقابل مفتاح PQC المسجّل لمُنشئ الطبقة.

تظل نقاط النهاية التالية ذات المسار الأقصر متاحة:

| الطريقة | نقطة النهاية                    | الوصف                                        |
| ------ | ------------------------------ | -------------------------------------------- |
| GET    | `/multilayer/v1/layer/{id}`    | تفاصيل طبقة محدّدة                            |
| GET    | `/multilayer/v1/layers`        | يسرد جميع الطبقات المسجّلة                    |
| GET    | `/multilayer/v1/anchor/{id}`   | تفاصيل سجلّ مرساة محدّد                       |
| GET    | `/multilayer/v1/anchors`       | يسرد عمليات إرسال المراسي الأخيرة             |
| GET    | `/multilayer/v1/routing-stats` | إحصائيات توجيه المعاملات عبر الطبقات          |
| GET    | `/multilayer/v1/params`        | معاملات الوحدة متعددة الطبقات الحالية         |

## انعكاس gRPC (gRPC Reflection)

انعكاس خادم gRPC مُفعّل افتراضياً، مما يتيح لأدوات مثل `grpcurl` اكتشاف الخدمات المتاحة:

```bash
grpcurl -plaintext localhost:9090 list
```

للاستعلام عن خدمة محدّدة:

```bash
grpcurl -plaintext localhost:9090 qorechain.pqc.v1.Query/Params
```

## المصادقة

جميع نقاط نهاية REST وgRPC غير مُصادَق عليها افتراضياً. لعمليات النشر الإنتاجية، ضع وكيلاً عكسياً (reverse proxy) مثل Nginx أو Caddy أمام العقدة للتعامل مع إنهاء TLS والتحكم في الوصول.
