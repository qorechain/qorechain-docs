# الاتصال بالشبكة الرئيسية

انضم إلى شبكة QoreChain Vladi الرئيسية النشطة عبر تهيئة عقدتك بملف التكوين الجيني الصحيح والأقران وإعدادات الشبكة.

:::note
تغطّي هذه الصفحة شبكة **`qorechain-vladi`** الرئيسية (معرّف سلسلة EVM **9801**، الست عشري `0x2649`)، النشطة منذ **7 يونيو 2026 الساعة 23:59 UTC** وتشغّل إصدار السلسلة **v3.1.80** على Cosmos SDK v0.53. لشبكة **`qorechain-diana`** للاختبار (معرّف سلسلة EVM **9800**)، راجع [الاتصال بشبكة الاختبار](/getting-started/connecting-to-testnet) وتدرّب على إعدادك هناك قبل الانتقال إلى الإنتاج.
:::

:::warning
تُنشر العقد البذرية للشبكة الرئيسية، والأقران الدائمون، وعنوان URL للتكوين الجيني، ومجموع التحقق SHA-256 الخاص به مع كل إصدار رسمي للشبكة الرئيسية. **احصل دائمًا على هذه القيم الحالية من المستودع/الإصدار الرسمي للشبكة الرئيسية** وتحقّق من مجموع التحقق للتكوين الجيني قبل البدء. يجب استبدال العناصر النائبة أدناه (`<MAINNET_SEED_NODE_ID>@<host>:26656`، وعنوان URL للتكوين الجيني، وعناوين URL للقطات) بالقيم الحقيقية المنشورة — لا تُشغّل عقدة الشبكة الرئيسية على أقران أو تكوين جيني غير مُتحقَّق منهما.
:::

---

## التثبيت

ثبّت ملف `qorechaind` التنفيذي إمّا عبر البناء من المصدر أو عبر سحب صورة Docker الرسمية.

### البناء من المصدر

```bash
git clone https://github.com/qorechain/qorechain-core.git
cd qorechain-core
CGO_ENABLED=1 go build -o qorechaind ./cmd/qorechaind/
```

راجع [البناء من المصدر](/developer-guide/building-from-source) للاطّلاع على المتطلبات المسبقة الكاملة (Go 1.26+، CGO، سلسلة أدوات Rust، المكتبات الأصلية).

### تهيئة العقدة

```bash
./qorechaind init my-node --chain-id qorechain-vladi
```

ينشئ هذا التكوين الافتراضي وأدلة البيانات ضمن `~/.qorechaind/`.

## تنزيل التكوين الجيني

استبدل ملف التكوين الجيني المحلي بالتكوين الجيني الرسمي للشبكة الرئيسية:

```bash
curl -o ~/.qorechaind/config/genesis.json \
  <MAINNET_GENESIS_URL>
```

تحقّق من مجموع التحقق للتكوين الجيني مقابل القيمة المنشورة في الإصدار الرسمي للشبكة الرئيسية قبل المتابعة:

```bash
sha256sum ~/.qorechaind/config/genesis.json
# Compare against <MAINNET_GENESIS_SHA256> from the official release
```

يحدّد هذا الملف الحالة الأولية لشبكة Vladi الرئيسية، بما في ذلك مجموعة المُحقّقين الجينية، وتخصيصات الرموز (حدث توليد الرمز عند التكوين الجيني)، ومعاملات الوحدات.

:::note
`<MAINNET_GENESIS_URL>` و `<MAINNET_GENESIS_SHA256>` عنصران نائبان. احصل على عنوان URL الحالي للتكوين الجيني ومجموع التحقق SHA-256 الخاص به من الإصدار/المستودع الرسمي للشبكة الرئيسية وتحقّق من تطابق مجموع التحقق قبل تشغيل عقدتك.
:::

## تهيئة الأقران

عدّل تكوين عقدتك للاتصال بأقران الشبكة الرئيسية الموجودين.

افتح `~/.qorechaind/config/config.toml` واضبط الحقلين `seeds` و `persistent_peers`:

```toml
seeds = "<MAINNET_SEED_NODE_ID>@<host>:26656"
persistent_peers = "<PEER_NODE_ID_1>@<host1>:26656,<PEER_NODE_ID_2>@<host2>:26656"
```

:::note
قيم البذرة والأقران الدائمين أعلاه عناصر نائبة. احصل على معرّف العقدة البذرية الحالي للشبكة الرئيسية والمضيف والمنفذ من المستودع/الإصدار الرسمي للشبكة الرئيسية. لا تتصل بأقران غير مُتحقَّق منهم.
:::

### الإعدادات الموصى بها

قد ترغب أيضًا في تعديل ما يلي في `config.toml`:

```toml
[mempool]
size = 5000

[consensus]
timeout_propose = "3s"
timeout_commit = "5s"
```

هذه القيم مُضبطة لأوقات الكتل والإنتاجية في شبكة Vladi الرئيسية.

## تشغيل العقدة

شغّل عقدتك لبدء المزامنة مع الشبكة:

```bash
./qorechaind start
```

تتصل العقدة بالأقران وتبدأ بتنزيل الكتل من التكوين الجيني. يعتمد وقت المزامنة الأولي على ارتفاع السلسلة الحالي وسرعة شبكتك. للحصول على تشغيل أسرع، يستخدم المشغّلون عادةً مزامنة الحالة أو لقطة حديثة — راجع [تشغيل عقدة](/developer-guide/running-a-node) للاطّلاع على سير عمل مزامنة الحالة واللقطات الكامل.

---

## التحقق من حالة المزامنة

تأكّد من أن عقدتك تلحق بأحدث كتلة:

```bash
curl localhost:26657/status | jq '.result.sync_info.catching_up'
```

* `true` — لا تزال العقدة تتزامن. انتظر حتى تلحق بالشبكة.
* `false` — العقدة متزامنة بالكامل وتعالج الكتل الجديدة.

يمكنك أيضًا التحقق من ارتفاع أحدث كتلة:

```bash
curl localhost:26657/status | jq '.result.sync_info.latest_block_height'
```

تأكّد من أنك على الشبكة الصحيحة — يجب أن يُبلّغ حقل `network` عن `qorechain-vladi`:

```bash
curl localhost:26657/status | jq '.result.node_info.network'
```

## المراقبة

تعرض QoreChain عدّة نقاط نهاية لمراقبة سلامة العقدة وأدائها.

### مقاييس Prometheus

تتوفر المقاييس الخام على:

```
http://localhost:26660/metrics
```

يمكن جمع هذه المقاييس بواسطة أي أداة جمع متوافقة مع Prometheus.

### لوحات معلومات Grafana

إذا كنت تشغّل عبر Docker Compose، فإن Grafana متاحة على:

```
http://localhost:3001
```

عند تسجيل الدخول لأول مرة، اضبط بيانات اعتمادك الخاصة عند المطالبة — لا تترك القيم الافتراضية كما هي. تعرض لوحات المعلومات المُهيّأة مسبقًا إنتاج الكتل، وإنتاجية المعاملات، واتصالات الأقران، واستخدام الموارد.

### فحص سلامة REST

توفّر واجهة REST API نقطة نهاية حالة سريعة:

```
http://localhost:1317
```

## مرجع المنافذ

| المنفذ    | البروتوكول  | الوصف                                              |
| ------- | --------- | ------------------------------------------------------- |
| `26657` | TCP       | RPC — استعلام وبثّ المعاملات                  |
| `26656` | TCP       | P2P — اتصالات شبكة الند للند                |
| `1317`  | HTTP      | REST API — استعلام حالة السلسلة عبر HTTP                   |
| `9090`  | gRPC      | gRPC API — وصول برمجي إلى السلسلة                    |
| `8545`  | HTTP      | EVM JSON-RPC — RPC متوافق مع إيثريوم (معرّف السلسلة `9801`) |
| `8546`  | WebSocket | EVM WebSocket — اشتراكات أحداث EVM في الوقت الفعلي       |
| `8899`  | HTTP      | SVM RPC — RPC متوافق مع سولانا                         |
| `26660` | HTTP      | نقطة نهاية مقاييس Prometheus                             |
