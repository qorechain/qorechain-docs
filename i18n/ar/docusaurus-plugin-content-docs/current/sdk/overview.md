# QoreChain SDK

إن QoreChain SDK هي مجموعة أدوات المطورين الرسمية متعددة اللغات لبناء
تطبيقات لامركزية على **QoreChain** — وهي شبكة Layer 1 آمنة كمومياً وثلاثية الأجهزة الافتراضية.

تغطي هذه الوثائق كيفية تثبيت SDK، والاتصال بالشبكة، وقراءة
الحالة على السلسلة، واشتقاق الحسابات، وتوقيع المعاملات وإرسالها، والعمل مع كل
آلة افتراضية من آلات QoreChain.

## ما هي QoreChain؟

QoreChain هي بلوكتشين من نوع Layer 1 مزودة بثلاثة أنظمة تشغيل للعقود الذكية
من الدرجة الأولى على سلسلة واحدة:

- **CosmWasm** — عقود ذكية بصيغة Wasm عبر Cosmos SDK.
- **QoreChain EVM Engine** — تنفيذ متوافق مع Ethereum (Solidity وviem
  وJSON-RPC القياسي).
- **SVM** — نظام تشغيل متوافق مع Solana مع JSON-RPC بأسلوب Solana.

تتم مشاركة الحسابات والأرصدة والرموز عبر أنظمة التشغيل، وتدعم
السلسلة IBC للتشغيل البيني عبر السلاسل.

### آمنة كمومياً بحكم التصميم

توفر QoreChain أساسيات التشفير ما بعد الكمومي (PQC) المستندة إلى
**ML-DSA-87** (Dilithium-5، FIPS 204). إلى جانب التوقيع الكلاسيكي بـ secp256k1،
تدعم السلسلة وضعية توقيع **هجينة** تحمل فيها المعاملة
توقيعاً كلاسيكياً *و* توقيعاً ما بعد كمومي معاً، فتبقى صالحة
بموجب التحقق الكلاسيكي اليوم مع اكتساب الحماية ما بعد الكمومية.

تكشف SDK اليوم عن توليد مفاتيح ML-DSA-87 والتوقيع والتحقق، بالإضافة
إلى اللبنات الأساسية للمعاملات الهجينة. راجع
[الحسابات وتوقيع PQC](/sdk/concepts/accounts-pqc) للتفاصيل. لا توجد ادعاءات
تسويقية هنا — تكشف SDK تماماً عن الأساسيات التي تنفذها السلسلة.

## ما الذي يميز هذه الـ SDK

إلى جانب التكافؤ الكامل عبر السلاسل المتعددة، هناك ثلاث قدرات **ممكنة فقط على
QoreChain**، لأنها مبنية على ميزات بروتوكول لا تمتلكها أي شبكة Layer 1 أخرى:

- **تسجيل المخاطر قبل الإطلاق بالذكاء الاصطناعي** — افحص المعاملة بالذكاء الاصطناعي على السلسلة قبل
  بثها. تُرجع `simulateWithRiskScore` الغاز إضافة إلى حكم مخاطر/شذوذ
  من العقود المسبقة (precompiles) الحتمية في EVM، بحيث يمكن للمحفظة أو التطبيق اللامركزي التحذير (أو الحظر)
  *قبل* التوقيع. راجع [الفحص المسبق بالذكاء الاصطناعي](/sdk/guides/ai-preflight).
- **استدعاءات موحدة عبر الأجهزة الافتراضية** — حساب واحد، ثلاث أجهزة افتراضية، معاملة واحدة.
  تستدعي `createCrossVMClient` عقداً على أي جهاز افتراضي، وتحزم `callAtomic` عدة
  استدعاءات عبر الأجهزة الافتراضية في معاملة ذرية واحدة موقّعة مرة واحدة. راجع
  [الاستدعاءات عبر الأجهزة الافتراضية](/sdk/guides/cross-vm).
- **تجربة مطور آمنة كمومياً** — اجعل أداة التوقيع محمية ما بعد كمومياً في استدعاء واحد
  عديم التأثير الجانبي (`ensurePqcRegistered` / `migrateToHybrid`)، مع شارة React جاهزة للإدراج.
  راجع [آمنة كمومياً](/sdk/guides/quantum-safe).

تجعل مجموعة **`@qorechain/react`** الجديدة (provider وhooks و`ConnectButton`
و`QuantumSafeBadge`) بناءَ تطبيق لامركزي آمن كمومياً المسارَ الافتراضي — راجع
[دليل مجموعة React](/sdk/guides/react). وللاطلاع على الحالة الكاملة، اقرأ
[لماذا QoreChain SDK](/sdk/why).

## عائلة الـ SDK

تُشحن الـ SDK كعائلة من الحزم بحيث يمكنك البناء باللغة التي تختارها.
وهي تشترك في نفس إعدادات الشبكة المسبقة، ومخططات الاشتقاق، وحسابات
الفئات النقدية، وأسطح القراءة.

| الحزمة | اللغة | التثبيت | الحالة |
| --- | --- | --- | --- |
| `@qorechain/sdk` | TypeScript | `npm i @qorechain/sdk` | منشورة (npm، v0.5.0) |
| `qorechain-sdk` | Python | `pip install qorechain-sdk` (استيراد `qorsdk`) | منشورة (PyPI، v0.5.0) |
| `qorechain-sdk` (وحدة Go) | Go | `go get github.com/qorechain/qorechain-sdk/packages/go/...` | منشورة (وكيل Go، v0.5.0) |
| `qorechain-sdk` | Rust | `cargo add qorechain-sdk` | منشورة (crates.io، v0.5.0) |
| `io.github.qorechain:qorechain-sdk` | Java | `io.github.qorechain:qorechain-sdk:0.5.0` | منشورة (Maven Central، v0.5.0) |
| `@qorechain/evm` | TypeScript (محول EVM) | `npm i @qorechain/evm viem` | منشورة (npm، v0.5.0) |
| `@qorechain/svm` | TypeScript (محول SVM) | `npm i @qorechain/svm @solana/web3.js` | منشورة (npm، v0.5.0) |
| `@qorechain/react` | TypeScript (مجموعة React) | `npm i @qorechain/react` | منشورة (npm، v0.5.0) |
| `create-qorechain-dapp` | CLI | `npm create qorechain-dapp` | منشورة (npm، v0.5.0) |

> يُثبَّت توزيع Python باسم `qorechain-sdk` لكن **يُستورد باسم
> `qorsdk`**. جميع العملاء منشورون في سجلاتهم — راجع
> [التثبيت](/sdk/install) للأوامر الخاصة بكل لغة.

نواة TypeScript (`@qorechain/sdk`) هي الأساس للأمثلة في هذه
الوثائق. يصل عملاء Python وGo وRust وJava إلى **تكافؤ كامل مع السلسلة الأصلية**
مع TypeScript: إعدادات الشبكة المسبقة، وأدوات الفئات النقدية/العناوين، واشتقاق الحسابات الهرمي الحتمي
(native/EVM/SVM)، وتوقيع PQC (ML-DSA-87)، ومؤلِّفات الرسائل المُنمَّطة
لكل وحدة مخصصة بالإضافة إلى وحدات Cosmos القياسية، وعملاء الاستعلام المُنمَّطين،
ودورة حياة المعاملة الكاملة (الغاز التلقائي، وفك ترميز الأخطاء، وتتبع المعاملات،
والبحث في الكتل/المعاملات)، والمعاملات الهجينة ما بعد الكمومية، واشتراكات
WebSocket. جميع هؤلاء العملاء **منشورون**: TypeScript إلى npm
(`@qorechain/sdk` 0.5.0)، وPython إلى PyPI (`qorechain-sdk` 0.5.0، استيراد
`qorsdk`)، وGo إلى وكيل الوحدة (`.../packages/go` 0.5.0)، وRust إلى
crates.io (`qorechain-sdk` 0.5.0)، وJava إلى Maven Central
(`io.github.qorechain:qorechain-sdk` 0.5.0). أما محولات تنفيذ EVM/SVM
(`@qorechain/evm` و`@qorechain/svm`، كلاهما 0.5.0)، ومجموعة `@qorechain/react`
(0.5.0)، وأداة CLI لإنشاء الهيكل `create-qorechain-dapp` فهي خاصة بـ TypeScript فقط
ومنشورة بالمثل إلى npm.

أضاف إصدار v0.4 عمليات سحب الـ rollup (`MsgExecuteWithdrawal`، وهو مسار الخروج L2→L1)،
وعملاء الاستعلام المُنمَّطين لوحدات `multilayer` و`rdk` و`bridge`،
ورسائل إدارة الجسر، ومساعدات عالية المستوى للسلاسل الجانبية/سلاسل الدفع والـ rollup
عبر اللغات الخمس جميعها.
