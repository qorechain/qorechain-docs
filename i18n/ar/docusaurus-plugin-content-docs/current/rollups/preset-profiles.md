# الإعدادات المسبقة

يأتي الـ RDK بـ **إعدادات مسبقة** توفّر تهيئات rollup جاهزة للاستخدام مضبوطة لفئات التطبيقات الشائعة. يجمع الإعداد المسبق وضع تسوية، ووضع مرتِّب، وخلفية لتوفّر البيانات، ومعاملات تنفيذ، بحيث يمكنك إطلاق rollup دون اختيار كل خيار يدويًا.

يُمرَّر الإعداد موضعيًا إلى `create-rollup`:

```bash
qorechaind tx rdk create-rollup [rollup-id] [profile] [stake-amount]
```

:::note
تتطابق القيم لكل إعداد مسبق أدناه مع القيم الافتراضية المُضمَّنة في إعدادات **`@qorechain/rdk`**، والتي تعكس جدول الإعدادات المنشور للشبكة. وقد تتطور هذه القيم مع نضج الـ RDK — استعلم عن معاملات الوحدة الحية باستخدام `qorechaind query rdk config` (أو `RdkClient.params()` من الـ SDK) للحصول على التهيئة المرجعية، وتحقّق على شبكة الاختبار **`qorechain-diana`** قبل الشبكة الرئيسية.
:::

## الإعدادات المسبقة

يجمع كل إعداد مسبق نموذج تسوية (ونظام الإثبات الذي تتطلبه تسويته)، ووضع مرتِّب، وخلفية لتوفّر البيانات، ونموذج رسوم غاز، وجهازًا افتراضيًا:

| الإعداد | التسوية (الإثبات) | المرتِّب | DA | نموذج الغاز | VM | حالة الاستخدام المقصودة |
| ------- | ------------------ | --------- | -- | --------- | -- | ----------------- |
| **`defi`** | zk (SNARK) | dedicated | native | EIP-1559 | EVM | تطبيقات DeFi وتطبيقات بأسلوب AMM — أسواق الإقراض، والبورصات اللامركزية، والمشتقات حيث تهمّ النهائية السريعة والرسوم المتوقّعة |
| **`gaming`** | based | based | native | flat | custom | حالة لعبة عالية الإنتاجية ومنخفضة زمن الوصول واقتصادات داخل اللعبة |
| **`nft`** | optimistic (fraud) | dedicated | native (Celestia DA مخطط له) | standard | CosmWasm | سَكّ الـ NFT، والأسواق، والمقتنيات الرقمية |
| **`enterprise`** | based | based | native | subsidized | EVM | عمليات نشر مأذونة وكونسورتيوم برسوم مدعومة (مُعانة) |
| **`custom`** | معامَلة بالكامل (الافتراضات: optimistic / fraud) | معامَلة بالكامل | معامَلة بالكامل | معامَلة بالكامل | معامَلة بالكامل (الافتراضي: EVM) | كل حقل يحدّده المستخدم — ابدأ من الصفر واضبط كل خيار بنفسك |

تنبثق بعض القيود من [مصفوفة التسوية ← الإثبات](/rollups/overview): تستخدم تسوية `optimistic` إثباتات `fraud`، وتستخدم `zk` إثبات `snark` (أو `stark`)، ولا تحمل `based` و`sovereign` أي إثبات. تقترن تسوية `based` دائمًا بوضع المرتِّب `based`. يُسوّي الإعداد المسبق `nft` أصليًا اليوم مع **تخطيط لـ Celestia DA**.

:::note
تم التحقق من تهيئة كل إعداد مسبق حيًا على إصدار السلسلة **v3.1.74**، حيث يطبّق `create-rollup` الإعداد المسبق للنموذج تلقائيًا: **`defi` = zk + EVM، `gaming` = based + custom VM، `nft` = optimistic + CosmWasm، `enterprise` = based + EVM، `custom` = optimistic + EVM (الافتراضات)**. يترك الإعداد المسبق `custom` كل حقل مفتوحًا — القيم المعروضة هي افتراضاته الأولية.
:::

عامِل الإعدادات المسبقة الأربعة الخاصة بالمجالات على أنها نقاط بداية معقولة والإعداد **`custom`** على أنه الخيار المفتوح بالكامل. قد تتغير المعاملات المُضمَّنة الدقيقة بين الإصدارات — استعلم عن `rdk config` (أدناه) للحصول على القيم المرجعية، ثم ابدأ من أقرب إعداد مسبق وحسّنه.

تنشئ واجهة سطر الأوامر [`create-qorechain-rollup`](/rollups/deploying-a-rollup#scaffold-a-project-with-create-qorechain-rollup) مشروعًا أوليًا قابلًا للتشغيل — قالب واحد لكل إعداد (`defi-rollup`، `gaming-rollup`، `nft-rollup`، `enterprise-rollup`، `custom-rollup`) — بحيث يمكنك الانتقال من إعداد إلى تعليمات برمجية عاملة للإنشاء/الاستعلام بأمر واحد.

## الحصول على توصية: `suggest-profile`

إذا لم تكن متأكدًا من أي إعداد مسبق يناسبك، يأخذ استعلام `suggest-profile` وصفًا بلغة بسيطة لحالة الاستخدام لديك ويُعيد إعدادًا مسبقًا موصى به.

```bash
qorechaind query rdk suggest-profile [use-case]
```

**مثال:**

```bash
qorechaind query rdk suggest-profile "a lending protocol with predictable fees"
```

التوصية نقطة بداية مفيدة — راجِع التوصية مقابل متطلباتك المحددة (ضمانات التسوية، ونموذج ثقة المرتِّب، واحتياجات توفّر البيانات، والجهاز الافتراضي) قبل الالتزام بتهيئة.

## فحص تهيئة الإعداد المسبق على السلسلة

نظرًا لأن تفاصيل الإعداد المسبق تُحلّ على السلسلة، فإن الطريقة المرجعية لمعرفة ما يُحلّ إليه الإعداد هي الاستعلام عن الوحدة وعن الـ rollup المُنشأ:

```bash
# Module-level parameters that govern rollup creation and defaults
qorechaind query rdk config

# After creation, inspect the resolved configuration of a specific rollup
qorechaind query rdk rollup [rollup-id]

# List all registered rollups
qorechaind query rdk list-rollups
```

يتيح لك هذا النمط — الاستعلام عن `config` قبل النشر، ثم الاستعلام عن `rollup` بعده — تأكيد ما أنتجه إعدادك المسبق المختار بالضبط، بدلًا من الاعتماد على قيم موثّقة قد تتطور.
