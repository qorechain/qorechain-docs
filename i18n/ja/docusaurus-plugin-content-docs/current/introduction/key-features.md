# 主な機能

次の表は、QoreChain のすべての主要機能を、それが導入されたリリースフェーズ別に整理して一覧化したものです。現在のチェーンバージョンは **v3.1.80** で、メインネット（`qorechain-vladi`、EVM チェーン ID 9801）は 2026 年 6 月 7 日から稼働しており、並行してテストネット（`qorechain-diana`、EVM チェーン ID 9800）も稼働しています。

| 機能                       | 導入バージョン      | 説明                                                                                                                                                                                                                                                                                                                                                                                           |
| -------------------------- | ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| PQC ハイブリッド署名（デフォルトで必須） | v1.1.0 (Security)   | すべての cosmos パストランザクションでのデュアル署名: 従来の secp256k1（ECDSA）署名と ML-DSA-87（Dilithium-5）をペアにします。v3.1.71 以降、ネットワークのデフォルトは**必須**（`hybrid_signature_mode = required`、`allow_classical_fallback = false`）です。従来型のみの cosmos トランザクションは拒否されます。ジェネシス gentx と PQC 鍵の登録/移行トランザクションのみが免除されます。EVM トランザクションは別の `eth_secp256k1` パスを使用し、影響を受けません。3 つのガバナンス制御の強制モード（disabled / optional / required）が引き続き利用可能です。TX 拡張の自動登録によるシームレスなウォレットオンボーディング。 |
| SHAKE-256 デフォルトハッシュ | v1.1.0 (Security)   | SHA-3 ファミリーの拡張可能出力関数（XOF）。v3.1.73 以降、SHAKE-256（`qorehash` パッケージ経由）は**デフォルトのアプリケーションハッシュ**であり、PQC ベースライン（Dilithium-5 + ML-KEM-1024 + SHAKE-256）を完成させます。可変長ハッシュ、固定 32 バイト出力、Merkle 内部ノード連結、ドメイン分離ハッシュを提供します。すべて FFI 依存なしの純粋な Go で実装されています。 |
| TEE および FL インターフェース | v1.1.0 (Security)   | Trusted Execution Environment アテステーション（SGX、TDX、SEV-SNP、ARM CCA）および連合学習（Federated Learning）協調（FedAvg、FedProx、SCAFFOLD 集約方式）のための本番グレードのインターフェース仕様。暗号学的保証を伴うハードウェアエンクレーブ AI 推論とプライバシー保護分散モデル学習を可能にします。 |
| オンチェーン RL コンセンサス（PRISM） | v1.0.0 (Genesis) | Go ネイティブの固定小数点 MLP（73,733 パラメータ）が、ブロックライフサイクル内で直接 PPO 推論を実行します。PRISM 最適化レイヤーは、外部オラクルなしでブロックタイム、ガス上限、バリデータープールの重みを動的に調整します。決定論的なテイラー級数数学により、すべてのバリデーターにわたって同一の結果が保証されます。4 つの動作モード: shadow、conservative、autonomous、paused。安全のためのサーキットブレーカー保護。 |
| トリプルプール複合 PoS     | v1.0.0 (Genesis)    | バリデーターは、QoreChain コンセンサスエンジン上で 1,000 ブロックごとに RPoS（評判加重）、DPoS（委任加重）、PoS（標準）プールに分類されます。プール加重のソーティションにより、純粋なステーク優位を超えてブロック生成が多様化されます。カスタムボンディングカーブは、自己ボンドステーク、ロイヤルティ期間、評判品質、プロトコルフェーズを考慮します。 |
| QDRW ガバナンス            | v1.0.0 (Genesis)    | 評判加重を伴う二次委任（Quadratic Delegation with Reputation Weighting）。投票力は、シグモイド評判乗数によって減衰された平方根関数を使用し、クジラによる支配を防ぎつつ、長期的で誠実な参加に報います。100 倍のステーク優位は約 10 倍の投票力をもたらします。xQORE 保有は投票の重みを 2 倍にします。 |
| バーンエンジン             | v1.0.0 (Genesis)    | 10 の異なるバーンチャネル: トランザクション手数料、ガバナンスペナルティ、スラッシング、ブリッジ手数料、スパム抑止、エポック超過、手動バーン、コントラクトコールバック、クロス VM 手数料、ロールアップ作成バーン。徴収された手数料は、**37% がバリデーターへ、30% が永久にバーン、20% がトレジャリーへ、10% がステーカーへ、3% がライトノードへ**分割されます。 |
| xQORE ステーキング         | v1.0.0 (Genesis)    | QOR をロックして 1:1 の比率で xQORE をミントし、QDRW 投票で 2 倍のガバナンス重みを得ます。段階的離脱ペナルティ（30 日未満で 50%、30〜90 日で 35%、90〜180 日で 15%、180 日後は 0%）は、PvP リベースを通じて残りの保有者に再分配され、確信に報い、短期資本にペナルティを課します。 |
| 固定供給発行               | v1.0.0 (Genesis)    | 4,500,000,000 QOR の固定総供給量（TGE で 80,000,000 がバーン）と、590,000,000 QOR の有限ステーキング報酬予算: 1 年目は 8〜12% APY（127,500,000 QOR）、2 年目は 6〜10% APY（106,250,000 QOR）、3〜4 年目は 5〜8% APY（年あたり 85,000,000 QOR）、5 年目以降はガバナンスで決定（残り約 186,000,000 QOR）。バーンエンジンと組み合わせることで、QOR はトランザクション量の増加に伴い純デフレ的な挙動に収束します。 |
| EVM ランタイム             | v1.0.0 (Genesis)    | EIP-1559 ガス価格設定、ポート 8545 での JSON-RPC（`eth_`、`web3_`、`net_`、`txpool_`、`qor_` 名前空間）、標準ツールサポート（Hardhat、Foundry、Remix）を備えた完全な Ethereum 互換性。既存の Ethereum ワークフローを使用して Solidity コントラクトをデプロイおよび操作できます。 |
| CosmWasm ランタイム        | v1.0.0 (Genesis)    | Rust ベースのコントラクト向け WebAssembly スマートコントラクトエンジン。完全なライフサイクルサポート: instantiate、execute、query、migrate。コントラクトは決定論的実行を備えたサンドボックス化された Wasm 環境で実行されます。 |
| SVM ランタイム             | v1.0.0 (Genesis)    | Rust ベースのエグゼキューターによる BPF プログラムのデプロイと実行。ポート 8899 での Solana 互換 JSON-RPC サーバーは `getAccountInfo`、`getBalance`、`getSlot` などをサポートします。既存の Solana クライアントとツールは変更なしで動作します。 |
| クロス VM ブリッジ         | v1.0.0 (Genesis)    | 3 つすべての VM 間でのシームレスな相互運用性。EVM コントラクトはプリコンパイル経由で CosmWasm を呼び出し、CosmWasm コントラクトはカスタムメッセージ経由で EVM を呼び出し、SVM プログラムは非同期イベントベースのブリッジングを通じて参加します。単一チェーン内での同期的な EVM-CosmWasm 呼び出しと非同期の SVM メッセージング。 |
| クロスチェーン接続性       | v1.2.0 (Interop)    | 8 つの IBC チャネル（Cosmos Hub、Osmosis、Noble、Celestia、Stride、Akash、Babylon、Injective）に加え、**36 の外部チェーンをカバーする 37 の QCB 設定**（ネイティブループバックとしての QoreChain 自体を含む）。PQC 署名のバリデーターアテステーション、チェーンごとの確認深度、サーキットブレーカーのボリューム上限。現在テストネット / 保留中のステータスであり、まだ本番ではありません。 |
| BTC リステーキング         | v1.2.0 (Interop)    | Bitcoin ファイナリティ保証のための Babylon Protocol 統合。バリデーターは BTC ステーキングポジション（最小 100,000 サトシ）を登録します。QoreChain のエポック状態ルートは、IBC リレーされた Babylon エポックを介して定期的に Bitcoin にチェックポイントされ、BTC ハッシュレートに支えられた二次的なファイナリティレイヤーを提供します。 |
| アカウント抽象化           | v1.2.0 (Interop)    | プロトコルレイヤーでのプログラマブルなスマートアカウント（ERC-4337 に類似）。3 つのアカウントタイプ: マルチシグ、ソーシャルリカバリー、セッションベース。きめ細かい権限と有効期限を持つセッションキー、アカウントごとの日次およびトランザクションごとの支出ルール、スコープ付き denom 許可リスト、コンセンサスでの自動ルール強制。 |
| MEV 保護                   | v1.2.0 (Interop)    | 暗号化メンプール向けの FairBlock しきい値 ID ベース暗号化（tIBE）フレームワーク。トランザクションは、含まれるまでブロックプロポーザーに対して暗号学的に不透明であり、フロントランニングとサンドイッチ攻撃を排除します。FairBlockDecorator ante ハンドラーは配線され準備が整っています。tIBE しきい値復号は鍵セレモニーのデプロイ後に有効化されます。 |
| ガス抽象化                 | v1.2.0 (Interop)    | マルチトークンガス支払いにより、トランザクション手数料のためにネイティブ QOR を保有する必要がなくなります。ユーザーは、受け入れられた IBC 転送トークンで支払うことができます: ibc/USDC は 1:1 のレート、ibc/ATOM は 10:1 のレート。GasAbstractionDecorator は、標準的な手数料控除の前に非ネイティブ手数料 denom を検証および変換します。 |
| 5 レーン優先順位付け       | v1.2.0 (Interop)    | ブロックスペースは、5 つの優先度レーンに静的に分割されます: PQC（優先度 100、15% スペース）、MEV（90、20%）、AI（80、15%）、Default（50、40%）、Free（10、10%）。セキュリティ上重要なトランザクションが、大量の標準トラフィックによって締め出されることは決してありません。 |
| オンチェーン流動性（AMM）  | v1.2.0 (Interop)    | ネイティブな自動マーケットメーカー（`x/amm`）が、プロトコルレイヤーでオンチェーンの流動性プールとスワップを提供します。 |
| RDK ロールアップ           | v1.3.0 (Rollups)    | 4 つの決済パラダイム（optimistic、zk、based、sovereign）、5 つのプリセットプロファイル（defi、gaming、nft、enterprise、custom）、SHA-256 ブロブストレージと自動プルーニングを備えたネイティブ DA ルーター、設定可能な作成バーン率を備えた bank エスクローライフサイクル、EndBlocker 自動ファイナライズ、コンセンサスモジュール経由の PRISM 支援設定を備えた Rollup Development Kit。ロールアップ機能はホストチェーンフレームワークとして提供されます。 |
| チェーンライセンス         | v1.3.0 (Rollups)    | `x/license` モジュールは、プロトコルネイティブなチェーンライセンスを提供します。 |

## バージョン履歴

<details>

<summary>v1.0.0 — ジェネシスリリース</summary>

ポスト量子暗号（Dilithium-5、ML-KEM-1024）、PRISM オンチェーン強化学習コンセンサスレイヤー、クロス VM メッセージングを備えたトリプル VM ランタイム（EVM、CosmWasm、SVM）、固定供給トークノミクスエンジン（burn、xQORE、有限発行予算）、トリプルプール複合 PoS バリデーター選択、QDRW 二次ガバナンス、AI トランザクション処理パイプラインを備えたコアプロトコルを確立しました。

</details>

<details>

<summary>v1.1.0 — セキュリティ強化リリース</summary>

従来の secp256k1（ECDSA）署名と ML-DSA-87 をペアにするハイブリッド署名アーキテクチャ、3 つのガバナンス制御の強制モード、将来の Merkle ツリー置き換えのための SHAKE-256 ポスト量子ハッシュ基盤、TEE アテステーション（SGX、TDX、SEV-SNP、ARM CCA）および連合学習協調（FedAvg、FedProx、SCAFFOLD）のための本番グレードのインターフェース仕様を導入しました。

</details>

<details>

<summary>v1.2.0 — 相互運用性と UX リリース</summary>

クロスチェーン接続性（8 つの IBC チャネル + 36 の外部チェーンをカバーする 37 の QCB 設定、現在テストネット/保留中）、Babylon Protocol を介した BTC リステーキング、セッションキーとソーシャルリカバリーを備えたスマートアカウント抽象化、FairBlock MEV 保護フレームワーク、マルチトークンガス抽象化、オンチェーン流動性（`x/amm`）、5 レーンブロックスペース優先順位付けを追加しました。

</details>

<details>

<summary>v1.3.0 — ロールアップエコシステムリリース</summary>

4 つの決済パラダイム（optimistic、zk、based、sovereign）、5 つのプリセットデプロイプロファイル（defi、gaming、nft、enterprise、custom）、ネイティブ DA ルーター、bank エスクローライフサイクル管理、EndBlocker 駆動の自動ファイナライズ、PRISM 支援ロールアップ設定、チェーンライセンス（`x/license`）を備えた Rollup Development Kit を出荷しました。自動サイドチェーン登録と状態アンカリングのためのマルチレイヤーアーキテクチャモジュールとの深い統合。

</details>
