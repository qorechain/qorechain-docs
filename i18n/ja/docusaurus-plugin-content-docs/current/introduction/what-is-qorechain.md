# QoreChain とは？

QoreChain は、ジェネシスにおけるポスト量子暗号、AI ネイティブなトランザクション処理、そして単一チェーン上で EVM、CosmWasm、SVM プログラムを実行するトリプル VM ランタイムを備えて構築された、世界初のレイヤー 1 ブロックチェーンです。既存のプロトコルに量子耐性を後付けするのではなく、QoreChain は、現代の汎用ブロックチェーンに期待される開発者体験と相互運用性を提供しながら、古典的および量子的な攻撃者の両方に対して安全であるよう、ゼロから設計されました。

メインネット（`qorechain-vladi`、EVM チェーン ID **9801**）は 2026 年 6 月 7 日から稼働しており、チェーンバージョン **v3.1.77** を実行しています。パブリックテストネット（`qorechain-diana`、EVM チェーン ID **9800**）は、ステージングと統合テストのために並行して稼働しています。ネイティブトークンは **QOR**（表示）/ **uqor**（基本単位、10^6）であり、Bech32 プレフィックスはアカウント用が `qor`、バリデーター用が `qorvaloper` です。チェーンは Cosmos SDK v0.53 上に構築されています。

## コアイノベーション

### 1. ポスト量子暗号

QoreChain は、デジタル署名に NIST 標準化された ML-DSA-87（Dilithium-5）、鍵カプセル化に ML-KEM-1024、デフォルトのアプリケーションハッシュに SHAKE-256 を使用し、古典的コンピュータと量子コンピュータの両方からの攻撃に対するセキュリティを提供します。ハイブリッド署名は、cosmos トランザクションパスで**デフォルトで必須**となりました。すべての cosmos パストランザクションは、従来の secp256k1（ECDSA）署名と*ともに*、Dilithium-5（ML-DSA-87）署名をトランザクション拡張として保持する必要があります。従来型のみの cosmos トランザクションは拒否されます。ダウングレードパスは閉じられています（ジェネシス gentx と PQC 鍵の登録/移行トランザクションのみが免除されます）。EVM トランザクションは影響を受けません。これらは別の `eth_secp256k1` ante パス（QoreChain EVM エンジンパス）を使用し、ハイブリッド署名を必要としません。3 つのガバナンス制御の強制モード（disabled、optional、required）が引き続き利用可能ですが、現在のネットワークデフォルトは**必須**です。アルゴリズムアジリティフレームワークにより、暗号標準の進化に応じて署名スキームをガバナンス提案を通じてアップグレードできます。

### 2. AI ネイティブ処理

オンチェーン強化学習エージェント（73,733 パラメータの PPO MLP）が、ブロックライフサイクル内で直接決定論的な固定小数点推論を実行し、ブロックタイム、ガス上限、バリデータープールの重みなどのコンセンサスパラメータを動的に調整します。この最適化レイヤーは **PRISM**（Policy-driven Reinforcement-learning for Intelligent State Machines）としてブランド化されています。統計的アイソレーションフォレスト異常検知と多次元リスクスコアリングが ante ハンドラーチェーン内ですべてのトランザクションを評価し、実行前に不正なパターンをフラグ付けします。動的手数料最適化は、リアルタイムのネットワーク状態に基づいてベース手数料を調整します。すべての AI 推論はバリデーター間で完全に決定論的です。同一の入力は、外部オラクル依存なしに同一の出力を生成します。

### 3. トリプル VM ランタイム

QoreChain は、1 つのコンセンサス内で 3 つの仮想マシンをネイティブに実行する唯一のレイヤー 1 です:

* **EVM** — EIP-1559 ガス価格設定とポート 8545 での JSON-RPC を備えた完全な Ethereum 互換性。標準ツール（Hardhat、Foundry、Remix）を使用して Solidity コントラクトをデプロイします。
* **CosmWasm** — 完全なライフサイクルサポート（instantiate、execute、query、migrate）を備えた、Rust で記述された WebAssembly スマートコントラクト。
* **SVM** — ポート 8899 での Solana 互換 JSON-RPC サーバーを備えた BPF プログラムのデプロイと実行。既存の Solana クライアントとツールがそのまま動作します。

クロス VM メッセージングにより、3 つすべてのランタイムが通信できます: EVM コントラクトはプリコンパイル経由で CosmWasm を呼び出し、CosmWasm コントラクトはカスタムメッセージ経由で EVM を呼び出し、SVM プログラムは非同期イベントベースのブリッジングを通じて参加します。

### 4. 固定供給トークノミクス

10 の異なるバーンチャネル（トランザクション手数料、ガバナンスペナルティ、スラッシング、ブリッジ手数料、スパム抑止、エポック超過、手動バーン、コントラクトコールバック、クロス VM 手数料、ロールアップ作成バーン）が中央のバーン会計モジュールに供給されます。徴収された手数料は、**37% がバリデーターへ、30% が永久にバーン、20% がトレジャリーへ、10% がステーカーへ、3% がライトノードへ**分割されます。xQORE ガバナンスステーキングメカニズムにより、ユーザーは QOR をロックして PvP リベース再分配で 2 倍のガバナンス重みを得られます。早期離脱ペナルティは残りの保有者に再分配され、確信に報います。

QoreChain は、永続的なパーセンテージインフレーションではなく、有限の発行予算を備えた**固定供給**モデルを使用します。総供給量は **4,500,000,000 QOR** に固定されており、そのうち **80,000,000（1.78%）** が TGE でバーンされました。ステーキング報酬は、複数年スケジュールにわたる専用の **590,000,000 QOR** プールから支払われます:

| 期間 | 目標 APY | 発行予算 |
| --- | --- | --- |
| 1 年目 | 8〜12% | 127,500,000 QOR |
| 2 年目 | 6〜10% | 106,250,000 QOR |
| 3〜4 年目 | 5〜8% | 年あたり 85,000,000 QOR |
| 5 年目以降 | ガバナンスで決定 | 残り約 186,000,000 QOR |

10 のバーンチャネルと組み合わせることで、固定供給設計は、トランザクション量の増加に伴い純デフレ的な挙動に収束します。

### 5. クロスチェーン接続性

QoreChain は、2 つの補完的なプロトコル、すなわちネイティブ IBC と QoreChain Bridge（QCB）を通じて、幅広いブロックチェーンエコシステムに接続するよう設計されています。ブリッジレイヤーは、**37 の QCB チェーン設定**（ネイティブループバックとしての QoreChain 自体を含む）に加え、**8 つの IBC チャネル**を定義しており、合計で **36 の外部チェーン**をカバーします。クロスチェーンレイヤーは現在**テストネット / 保留中のステータスであり、まだ本番ではありません**。以下の数値は対象とするカバレッジを示しています。

* **8 つの IBC チャネル** — Cosmos Hub、Osmosis、Noble、Celestia、Stride、Akash、Babylon、Injective。クライアント更新、不正検出、自動パケットクリアリングを備えた事前設定済みのリレーヤーテンプレート。
* **37 の QCB 設定（36 の外部チェーン + QoreChain ループバック）** — 各エンドポイントは、タイプごとのアドレス検証、設定可能な確認深度、サーキットブレーカーのボリューム上限、PQC 署名のバリデーターアテステーションを含むよう設計されています。対象とする外部チェーンは次のとおりです:
  * **ベースライン (10):** Ethereum、Solana、TON、BSC、Avalanche、Polygon、Arbitrum、Optimism、Base、Sui
  * **EVM ファミリー (14):** zkSync Era、Linea、Scroll、Blast、Mantle、Hyperliquid、Berachain、Sonic、Sei、Monad、Plasma、Filecoin FVM、Cronos、Kaia
  * **非 EVM (5):** Starknet、XRP Ledger、Stellar、Hedera、Algorand
  * **保留中 (7):** NEAR、Bitcoin、Cardano、Polkadot、Tezos、Tron、Aptos

アーキテクチャは、あらゆる主要チェーンタイプ（EVM、Solana（SVM）、Move ベース（Sui、Aptos）、Cosmos/IBC、UTXO、その他の非 EVM ファミリー）にまたがり、エコシステム全体にわたる幅広い相互運用性を提供します。

### 6. Rollup Development Kit

`x/rdk` モジュールは、QoreChain ホストチェーン上で直接アプリケーション固有のロールアップをデプロイするためのプロトコルネイティブなフレームワークです。ロールアップサポートはホストチェーンフレームワークとして提供されます。特定のデプロイに関する主張は、対象とする機能として扱う必要があります。4 つの決済パラダイムがサポートされています:

* **Optimistic** — 7 日間のチャレンジウィンドウを備えた不正証明、EndBlocker による自動ファイナライズ。
* **ZK（ゼロ知識）** — 検証時に即時ファイナリティを得る SNARK または STARK 証明。
* **Based** — 約 2 ホストブロックでファイナリティを得る L1 シーケンスされたトランザクション。
* **Sovereign** — データ可用性のためにのみ QoreChain を使用する独立したチェーン。

5 つのプリセットプロファイル（**defi、gaming、nft、enterprise、custom**）により、事前設定された決済モード、ブロックタイム、VM の選択、DA バックエンド、ガスモデルでのワンクリックデプロイが可能になります。ネイティブ DA ルーターは、設定可能な保持期間と自動プルーニングを備えた SHA-256 コミット済みブロブストレージを提供します。PRISM コンセンサスモジュールは、AI 支援ロールアップ設定のためのアドバイザリーメソッドを提供します。

### 7. アカウントおよびガス抽象化

3 つのプログラマブルタイプ（マルチシグ、ソーシャルリカバリー、セッションベース）を備えたスマートアカウントは、きめ細かい権限と有効期限を持つセッションキー、アカウントごとの支出ルール、denom 許可リストをサポートします。これにより、標準アカウントでは不可能なウォレット UX パターンが可能になります: モバイル向けの dApp セッションキー、ファーストクラスのアカウントタイプとしてのソーシャルリカバリー、コンセンサスで強制されるプログラマブルな支出上限。ガス抽象化により、手数料のためにネイティブ QOR を保有する必要がなくなります。ユーザーは、USDC や ATOM などの受け入れられた IBC 転送トークンで支払うことができます。

## エコシステム

QoreChain は、**20 以上のカスタムモジュールを含む 45 以上のジェネシスモジュール**を備えて出荷されており、セキュリティ（pqc）、AI（ai、reputation、rlconsensus）、コンセンサス（qca）、仮想マシン（vm、svm、crossvm）、トークノミクス（burn、xqore、inflation）、流動性（amm）、ライセンス（license）、ブリッジ（bridge、babylon、multilayer）、ガバナンス拡張（abstractaccount、fairblock、gasabstraction）、ロールアップ（rdk）をカバーします。最近の追加には、AMM / オンチェーン流動性のための `x/amm` と、チェーンライセンスのための `x/license` が含まれます。チェーンはオープンコアアーキテクチャに従っています。プロトコルレイヤーは完全にオープンソースであり、エンタープライズデプロイ向けのオプションのプロプライエタリ拡張があります。
