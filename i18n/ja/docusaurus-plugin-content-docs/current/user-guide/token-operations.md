# トークン操作

このガイドでは、QOR トークン、トークンの送受信方法、残高の照会方法、そして QoreChain における手数料分配モデルについて説明します。

:::note
以下のコマンドは **`qorechain-diana`** テストネット（EVM チェーン ID **9800**）を使用しています。メインネット（**`qorechain-vladi`**、EVM チェーン ID **9801**）は 2026 年 6 月 7 日以降稼働しており、チェーンバージョン **v3.1.82** を実行しています。メインネットで取引を行う場合は、**メインネットへの接続** ページにあるメインネットのチェーン ID とエンドポイントに置き換えてください。
:::

## トークン情報

| プロパティ                | 値                            |
| ------------------------ | ----------------------------- |
| **表示単位**             | QOR                           |
| **基本単位**             | uqor                          |
| **換算**                 | 1 QOR = 1,000,000 uqor (10^6) |
| **チェーン ID**          | `qorechain-vladi`（メインネット）/ `qorechain-diana`（テストネット） |
| **Bech32 プレフィックス** | `qor`（例: `qor1abc...xyz`）   |

すべてのオンチェーン金額は **uqor** で表されます。トランザクションを送信する際は、常に金額を uqor で指定してください。

## トークンの送信

QOR トークンをあるアカウントから別のアカウントへ転送するには:

```bash
qorechaind tx bank send <from_address> <to_address> <amount>uqor \
  --chain-id qorechain-diana \
  --fees 500uqor
```

**例:** 別のアドレスに 5 QOR（5,000,000 uqor）を送信する場合:

```bash
qorechaind tx bank send qor1sender... qor1recipient... 5000000uqor \
  --chain-id qorechain-diana \
  --fees 500uqor
```

送信者には、生のアドレスの代わりにキー名を使用することもできます。

```bash
qorechaind tx bank send mykey qor1recipient... 5000000uqor \
  --chain-id qorechain-diana \
  --fees 500uqor
```

## 残高の照会

任意のアカウントの残高を確認します。

```bash
qorechaind query bank balances <address>
```

**例:**

```bash
qorechaind query bank balances qor1abc...xyz
```

**サンプル出力:**

```yaml
balances:
- amount: "15000000"
  denom: uqor
pagination:
  next_key: null
  total: "0"
```

これは、アカウントが 15 QOR（15,000,000 uqor）を保有していることを示しています。

## 手数料の構造

QoreChain のトランザクション手数料は、ネットワークのインセンティブを整合させるために、5 つの宛先に分配されます。

| 宛先             | 割合  | 目的                                                           |
| --------------- | ----- | -------------------------------------------------------------- |
| **バリデーター**  | 37%   | ブロック生成者に報酬を与え、ネットワークを保護します                |
| **バーン**       | 30%   | 供給から永久に削除され、デフレ圧力を生み出します                   |
| **トレジャリー**  | 20%   | プロトコル開発とエコシステムへの助成金に資金提供します             |
| **ステーカー**   | 10%   | すべての委任者に比例配分されます                                 |
| **ライトノード**  | 3%    | ネットワークデータを提供するライトノードオペレーターに報酬を与えます  |

## バーンチャネル

QoreChain はマルチチャネルのバーンメカニズムを実装しています。QOR トークンは、10 個の異なるチャネルを通じて流通から永久に削除されます。

| チャネル              | 説明                                                                |
| -------------------- | ------------------------------------------------------------------- |
| `tx_fee`             | 各トランザクション手数料の 30% バーン分                               |
| `governance_penalty` | ガバナンス提案がクォーラムに達しなかった場合、または拒否された場合にバーンされます |
| `slashing_burn`      | スラッシングされたバリデーターステークのバーン分                       |
| `bridge_fee`         | クロスチェーンブリッジ転送でバーンされる手数料                         |
| `spam_deterrent`     | スパムとしてフラグ付けされたトランザクションに適用される追加バーン       |
| `epoch_excess`       | エポック境界でバーンされる目標を超える超過発行分                       |
| `manual_burn`        | ガバナンス提案を通じたコミュニティ主導のトークンバーン                  |
| `contract_callback`  | スマートコントラクトのコールバック実行でバーンされる手数料              |
| `cross_vm_fee`       | クロス VM（例: EVM から CosmWasm）呼び出しの実行時にバーンされます      |
| `rollup_create`      | 新しいロールアップをデプロイする際にバーンされる最小ステークの 1%       |

すべてのチャネルにわたるバーン総額を照会できます。

```bash
qorechaind query bank total --denom uqor
```

## ヒント

:::caution
トークンを送信する前に、受信者のアドレスを常に再確認してください。QoreChain でのトランザクションは取り消せません。
:::

:::tip

* `--dry-run` フラグを使用すると、ブロードキャストせずにトランザクションをシミュレートできます。
* `--gas auto` を使用すると、トランザクションに必要なガスをノードに推定させることができます。
* 標準的な転送の最低手数料は **500 uqor** です。

:::
