# UX エディション — Web ダッシュボード

**UX（User eXperience）**エディションは、SX エディションと同じライトノードデーモンを実行しますが、ブラウザでノードとネットワークを監視できるように**組み込みの Web ダッシュボード**を追加します。バイナリは `lightnode-ux` です。SX エディションと同様に、これはライトノードの**v3.1.1**系列です（チェーンバージョンとは別の、独自のバージョン）。

UX エディションは、デスクトップでの利用や、コマンドラインよりも視覚的なインターフェースを好むオペレーターにとって適切な選択です。

## インストール

### ソースからのビルド

UX エディションは**Go 1.26.1**を必要とし、ポスト量子ネイティブライブラリのために CGO を有効にしてビルドします。

```bash
CGO_ENABLED=1 go build -o build/lightnode-ux ./cmd/lightnode-ux/
```

これにより `build/lightnode-ux` が生成されます。

### Docker

UX サービスは `Dockerfile.ux` からビルドされます。

```bash
docker compose up lightnode-ux
```

UX コンテナは、`/root/.qorechain-lightnode` にある名前付きボリュームにデータを永続化し、チェーン RPC アドレスを `QORECHAIN_RPC_ADDR` 環境変数から読み取ります。

## 実行

UX ノードを起動します。

```bash
build/lightnode-ux start
```

これは、デーモンと組み込みダッシュボードサーバーを一緒に起動します。UX エディションは常にダッシュボードを有効にします。起動時、バイナリはダッシュボードの URL を出力します。

UX エディションは SX エディションとセットアップを共有します。`~/.qorechain-lightnode` から同じ `config.toml` を読み取り、同じ Dilithium-5 キーリングを使用します。まだノードを設定していない場合は、まず SX ウィザード（`lightnode-sx onboard`）を実行して設定を書き込み、キーをインポートまたは生成してください — [SX エディション](/light-node/sx-edition)を参照してください。

## ポート 8420 の Web ダッシュボード

ダッシュボードは**ポート 8420**で公開されます。これは `lightnode-ux` Docker イメージが宣言するポート（`EXPOSE 8420`）であり、バイナリがバインドするデフォルトでもあるため、Docker で実行している場合、ダッシュボードは `8420` で公開されます。

```
http://localhost:8420
```

:::caution compose のポートマッピングを確認してください
他の箇所の記述では、ダッシュボードに関してポート 8080 が参照されている場合があります。正式な値は**8420**です — これがイメージが実際に公開し、デーモンがデフォルトでバインドするものです。独自の `docker-compose.yml` やリバースプロキシを調整する場合は、8080 ではなく**8420**にマッピングしてください。
:::

## ダッシュボードが表示する内容

ダッシュボードは次のビューに整理されています。

- **Overview** — ブロック高さとノード状態の概要。
- **Validators** — ボンドされたバリデーターセット。
- **Delegation** — 現在の委任とその分割。
- **Network** — ライブのネットワークテレメトリと最近同期したヘッダー。
- **Bridge** — クロスチェーンブリッジのテレメトリ。
- **Tokenomics** — トークン経済のテレメトリ。
- **Settings** — ノードの有効な設定。

テレメトリはリアルタイムで更新され、デーモンはバリデーター、ネットワーク、ブリッジ、トークノミクスのデータを独立した間隔で更新します（`config.toml` の `[telemetry]` の下で設定可能）。

### ローカルのみバナー

ノードに**チェーン RPC エンドポイントが設定されていない**場合、ダッシュボードは**ローカルのみモード**で実行され、その状態を説明する目立つバナーを表示します。PQC スタックは検証されますが、ノードはどのチェーンも同期しないため、ブロック高さは `0` のままです。バナーは、ホスト上でオンボーディングウィザードを実行するよう促します。

```bash
lightnode-sx onboard
```

ウィザードは PQC セルフテストを実行し、チェーンエンドポイントを尋ね、バリデーターキーをインポートまたは生成します。エンドポイントが設定されると、ノードを再起動することでダッシュボードはライブのチェーンデータを表示し始めます。
