# バージョン履歴

QoreChain のパブリックバージョン履歴です。最新リリースは **v3.1.77** で、メインネット **`qorechain-vladi`**（EVM チェーン ID **9801**、2026 年 6 月 7 日から稼働）上で動作しています。テストネット **`qorechain-diana`**（EVM チェーン ID **9800**）はプレリリースビルドを追跡します。

:::note
以下のエントリーは高レベルの機能サマリーです。以前の `v1.x` エントリーは、メインネットに先行したテストネットリリースラインの歴史的記録として保持されています。
:::

---

## v3.1.77 — 現在のメインネットリリース

**リリースの焦点:** クロスチェーンおよび供給モジュールの読み取り専用 REST アクセス。

* **Bridge REST エンドポイント** — ブリッジモジュールの読み取り専用 HTTP クエリエンドポイント。gRPC に加えて標準 REST でブリッジ状態を公開します。
* **Burn REST エンドポイント** — バーンモジュールの読み取り専用 HTTP クエリエンドポイント。手数料分配と供給データを標準 REST でクエリ可能にします。

## v3.1.76 — SVM ツールチェーンの近代化

**リリースの焦点:** Solana 仮想マシン互換性のリフレッシュ。

* **現行ツールチェーンプログラムのサポート** — SVM 実行が近代化され、現行の Solana ツールチェーンでビルドされたプログラムが QoreChain SVM ランタイム上で動作するようになりました。

## v3.1.75 — SVM JSON-RPC をデフォルトで有効化

**リリースの焦点:** すぐに使える Solana 互換 RPC。

* **Solana 互換 JSON-RPC** — SVM JSON-RPC サーバーがデフォルトで有効化（ポート **8899**）され、ノードとともに自動的に起動するようになり、SVM ツール向けの Solana 互換 RPC インターフェースを提供します。

## v3.1.74 — ロールアッププロファイルプリセット

**リリースの焦点:** Rollup Development Kit のユーザビリティと決済。

* **プロファイルプリセットの適用** — ロールアップ作成時に選択したプロファイルのプリセット（DeFi、ゲーミング、NFT、エンタープライズ、または完全カスタム）が適用されるようになり、新しいロールアップがそのユースケースに適したデフォルトを継承します。
* **オプティミスティック決済** — オプティミスティック決済パス（バッチ送信とチャレンジ）がエンドツーエンドで稼働しています。

## v3.1.73 — ポスト量子ハッシュベースライン

**リリースの焦点:** デフォルトのポスト量子暗号ベースラインの完成。

* **SHAKE-256 デフォルトハッシュ** — SHAKE-256（SHA-3 ファミリー）がデフォルトのアプリケーションハッシュとして採用され、**ML-DSA-87 (Dilithium-5)** 署名、**ML-KEM-1024** 鍵カプセル化、**SHAKE-256** ハッシュからなるデフォルトのポスト量子ベースラインが完成しました。

## v3.1.72 — 安定性とメンテナンス

**リリースの焦点:** 定期的な安定性とビルドパイプラインのメンテナンス。

* **安定性の改善** — 外部から見える動作変更を伴わない、内部の安定性、依存関係、ビルドパイプラインのメンテナンス。

## v3.1.71 — PQC ハイブリッド署名をデフォルトで強制

**リリースの焦点:** Cosmos トランザクションパスでデフォルトで有効なポスト量子セキュリティ。

* **ハイブリッド署名がデフォルトで必須** — ポスト量子ハイブリッド署名が Cosmos トランザクションパスでデフォルトで強制されるようになりました。各トランザクションは古典的な **secp256k1** 署名とともにポスト量子 **ML-DSA-87 (Dilithium-5)** 署名を持ちます。
* **ガバナンス制御の強制** — 強制モードはガバナンス制御のままで、デフォルトは **required** に設定されています。

## v3.1.70 — 本番環境ハードニング

**リリースの焦点:** 稼働中のメインネットのための本番環境ハードニングとコンセンサス最適化。

* **PRISM コンセンサス最適化** — サーキットブレーカーの安全制御を備え、稼働中のネットワーク状況下での適応的パラメータチューニングのための PRISM 強化学習最適化レイヤーの継続的な改善。
* **パフォーマンスと安定性** — バリデータとフルノード全体でのスループット、レイテンシ、リソース使用量の改良。
* **運用ツール** — メインネットオペレーター向けの監視、クエリ、ノード運用の使い勝手の改善。
* **トークノミクス v2.1 アライメント** — 固定供給、有限発行の経済モデルに合わせた手数料分配と発行メカニズム。

## v3.0.0 — メインネットジェネシス

**リリースの焦点:** メインネットローンチとトークン生成イベント。

* **メインネットジェネシス** — QoreChain メインネット（`qorechain-vladi`、EVM チェーン ID 9801）が **2026 年 6 月 7 日** にローンチされ、ジェネシス時にトークン生成イベント（TGE）が行われました。
* **5 分割手数料配分** — バリデータ、バーン、トレジャリー、ステーカー、ライトノード全体でのプロトコル手数料分配（**37 / 30 / 20 / 10 / 3**）で、専用のライトノードシェアを追加。
* **オンチェーン AMM** — オンチェーン流動性プールとスワップのためのネイティブな自動マーケットメーカーモジュール（`x/amm`）。
* **チェーンライセンシング** — プロトコル権限を登録および管理するためのオンチェーンライセンスモジュール（`x/license`）。
* **ハードニングされた決済パラダイム** — RDK 決済モードが optimistic、zk、based、sovereign として確定。

## v1.4.0 — プレメインネット拡張

**リリースの焦点:** メインネットに先立つクロスチェーンカバレッジとリリース候補の安定化。

* **クロスチェーンカバレッジの拡大** — より広範な外部ネットワークへの追加の IBC およびブリッジ接続。
* **ライトノードの参加** — ライトノードと、その手数料シェア報酬の基盤を導入。
* **リリース候補ハードニング** — メインネットジェネシスに向けて、すべてのコアモジュールにわたる広範なテスト、監査、安定化。

## v1.3.0 — Rollup Development Kit

**リリースの焦点:** ソブリンおよび共有セキュリティロールアップデプロイのためのネイティブロールアップインフラ。

* **x/rdk モジュール** — 4 つの決済パラダイム（optimistic、zk、based、sovereign）を備えた完全な Rollup Development Kit
* **5 つのプリセットプロファイル** — DeFi、ゲーミング、NFT、エンタープライズ、完全カスタムのユースケース向けの事前設定済みロールアップテンプレート
* **ネイティブデータ可用性** — ブロブストレージ、保持管理、プルーニングライフサイクルを備えたオンチェーン DA レイヤー
* **EndBlocker 自動確定** — チャレンジウィンドウの期限切れ時に、オペレーターの介入なしにバッチを自動確定
* **AI 支援プロファイル選択** — 意図されたユースケースに基づいて最適なロールアップ構成を推奨する `suggest-profile` クエリ
* **マルチレイヤー統合** — ロールアップはマルチレイヤーアーキテクチャ内でレイヤーとして登録され、ルーティング、アンカリング、チャレンジメカニズムを継承
* **Bank エスクローライフサイクル** — オペレーターステークはロールアップ運用中にエスクローで保持され、正常なシャットダウン時に解放、スラッシング時に没収

## v1.2.0 — IBC とブリッジ

**リリースの焦点:** クロスチェーン接続性と高度なアカウント抽象化。

* **25 のクロスチェーン接続** — 8 つの IBC チャネルと、外部ネットワークへの 17 の QoreChain Bridge (QCB) 接続
* **x/babylon モジュール** — Bitcoin 保有者が QoreChain ステーキングセキュリティに参加できるようにする BTC リステーキング統合
* **x/abstractaccount モジュール** — プログラム可能な支出ルール、セッションキー、カスタム認証ロジックを備えたスマートアカウントフレームワーク
* **x/fairblock モジュール** — MEV 耐性トランザクション暗号化のためのしきい値 ID ベース暗号（tIBE）
* **x/gasabstraction モジュール** — ネイティブ QOR、IBC ブリッジ USDC、IBC ブリッジ ATOM をサポートするマルチトークンガス支払い
* **5 レーン TX 優先順位付け** — 優先順位順に並べられたトランザクションレーン: システム、ガバナンス、ステーキング、ブリッジ、一般
* **IBC リレーヤー構成** — サポートされるすべての IBC チャネルの事前設定済みリレーヤーセットアップ
* **Bridge から burn への統合** — ブリッジ手数料はバーンモジュールの手数料分配を通じてルーティング

## v1.1.0 — PQC ハイブリッド署名

**リリースの焦点:** ポスト量子暗号セキュリティとアルゴリズムアジリティ。

* **デュアル secp256k1 (ECDSA) + ML-DSA-87 署名** — すべてのトランザクションが古典署名とポスト量子署名の両方を持ち、AnteHandler チェーンで検証
* **3 つの強制モード** — 設定可能なハイブリッド署名強制: オフ（モード 0）、許可的（モード 1、PQC は任意）、必須（モード 2、PQC は必須）
* **自動登録** — PQC 公開鍵は最初のハイブリッドトランザクションで自動的に登録され、別の登録ステップが不要に
* **SHAKE-256 ハッシュ基盤** — すべての PQC 関連ハッシュ操作は量子耐性アドレス導出のために SHAKE-256（SHA-3 ファミリー）を使用
* **TEE アテステーションインターフェース** — PQC 鍵生成の完全性を証明するための信頼実行環境アテステーションサポート
* **アルゴリズムアジリティフレームワーク** — チェーンアップグレードなしにガバナンスを通じて将来の PQC アルゴリズムを追加できるプラガブルなアルゴリズムレジストリ

## v1.0.0 — ジェネシス（トークノミクスエンジン）

**リリースの焦点:** 完全なトークノミクス、マルチ VM 実行、AI 支援運用を備えた初期プロトコルローンチ。

* **x/burn モジュール** — バリデータ、バーン、トレジャリー、ステーカー全体での 4 分割配分を備えたマルチチャネル手数料バーンメカニズム
* **x/xqore モジュール** — 段階的な早期アンロックペナルティと PvP リベース再分配を備えたガバナンスステーキングデリバティブ
* **x/inflation モジュール** — 有限発行の経済モデルによって管理される、年次減衰を伴うエポックベースの発行
* **PRISM コンセンサスレイヤー** — サーキットブレーカーの安全制御を備えた、動的チェーンパラメータチューニングのための強化学習最適化（PPO）
* **トリプルプール CPoS** — レピュテーションスコアで加重された Emerald、Sapphire、Ruby バリデータプールを備えた分類型プルーフ・オブ・ステーク
* **QDRW ガバナンス** — プール全体の報酬分配に対するガバナンス承認済みの調整を可能にする動的報酬重み付けシステム
* **EVM + CosmWasm + SVM ランタイム** — 3 つの同時実行環境: QoreChain EVM エンジン、CosmWasm スマートコントラクト、Solana 仮想マシン
* **クロス VM ブリッジ** — 単一ブロック内での EVM、CosmWasm、SVM ランタイム間のメッセージパッシングと資産転送
* **ポスト量子暗号** — 高性能 PQC ライブラリに支えられた量子耐性署名
* **QCAI** — 不正検知、手数料推定、ネットワーク最適化のための、オプションのオフチェーンサイドカーを備えたオンチェーンヒューリスティック分析
* **コンテナ化デプロイ** — サイドカーサービスとブロックインデクサーを備えた完全なマルチバリデータテストネットデプロイ
* **ブロックインデクサー** — 履歴クエリと分析のための永続ストレージを備えたブロックリスナー
