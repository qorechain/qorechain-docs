# バージョン履歴

QoreChain の公開バージョン履歴です。最新リリースは **v3.1.80** で、メインネット **`qorechain-vladi`**（EVM チェーン ID **9801**、2026 年 6 月 7 日より稼働中）で動作しています。テストネット **`qorechain-diana`**（EVM チェーン ID **9800**）はプレリリースビルドを追跡します。

:::note
以下のエントリは、上位レベルの機能概要です。より古い `v1.x` のエントリは、メインネットに先行したテストネットのリリースラインの歴史的記録として保持されています。
:::

---

## v3.1.80 — マルチレイヤー状態アンカークエリ（現行メインネットリリース）

**リリースの焦点:** ロールアップ向けの、読み取り可能でオフライン検証可能な決済アンカー。

* **アンカー読み取りクエリ** — `x/multilayer` クエリサービスが `Anchor`（レイヤーの最新状態アンカー）と `Anchors`（レイヤーのアンカー履歴）を公開するようになり、クライアントはレイヤーの決済アンカーを取得して独立して検証できるようになりました。
* **マルチレイヤー向け REST ゲートウェイ** — すべてのマルチレイヤークエリ（`params`、`layers`、`layers/{layer_id}`、`anchor/{layer_id}`、`anchors/{layer_id}`、`routing-stats`）が gRPC に加えて REST 経由でも利用可能になりました。
* **耐量子決済レシートのブロック解除** — 各アンカーは、その正規フィールドに対する **ML-DSA-87 (Dilithium-5)** 署名を保持しており、Rollup Development Kit のオフライン決済レシート検証のためのオンチェーン基盤を提供します。

## v3.1.79 — ブリッジネットワーク向けバリデーター自動プロビジョニング

**リリースの焦点:** ライセンスを保有するバリデーターによる、接続済みネットワークへのターンキー参加。

* **ネットワークドライバーフレームワーク** — 宣言的なドライバーフレームワークにより、該当する `validator_<chain>`（または `qcb_bridge`）ライセンスを保有する QoreChain バリデーターは、対応する外部ネットワーククライアントを QoreChain のオーケストレーションのもとで同じノード上にプロビジョニング、設定、実行させることができます（ライセンスがアクティベートされた場合に限ります）。
* **37 すべてのブリッジネットワーク向けドライバー** — カバレッジは接続済みのすべてのネットワークに及び、参加モデル（パーミッションレスバリデーター、上限付き/選出/承認制、L2 フルノード、非ステーキング/トラストリストロール）ごとに分類されています。外部ネットワークのステークと署名キーはネットワークごとにオペレーターが提供するものとして維持され、QoreChain はフレームワークと強制されるライセンスゲートを提供します。

## v3.1.78 — デプロイ前準備

**リリースの焦点:** ウォレット、ブリッジ、IBC、ライセンスのすべてがローンチ時に動作する — デプロイ後のガバナンスは不要。

* **トラストレスなデプロイ後ブリッジアクティベーション** — `bridge_admin` キー（または `qcb_bridge` ライセンス保有者）は、単一の署名済みトランザクション（`tx bridge update-chain-config` / `set-verifier-bootstrap`）で、接続済みの任意のチェーンのブリッジをアクティベートできます — コントラクトアドレス、確認数、アーキテクチャ、ステータス、アクティブな検証者、検証者トラストルートを設定 — ガバナンス提案やチェーンアップグレードは不要です。
* **バリデーターネットワークライセンスゲート** — オーケストレーターは、外部ネットワーククライアントを起動する前に `validator_<chain>` / `qcb_bridge` ライセンスを強制するようになりました（フェイルクローズ）。
* **ウォレット統合パッケージ** — `@qorechain/wallet-adapter` と `@qorechain/connect` を npm（v0.1.0）に公開し、ワンコールでの MetaMask ネットワーク登録（EIP-3085、EVM レール上の **18 桁** ネイティブ QOR）と Keplr のガス価格設定を追加しました。
* **IBC ターンキーリレイヤー** — 8 つの IBC カウンターパーティ向けの、すぐに実行可能なリレイヤー設定とチャネルブートストラップツール。これにより、特注のセットアップなしでデプロイ後にチャネルが立ち上がります。

## v3.1.77 — ブリッジ & バーン REST エンドポイント

**リリースの焦点:** クロスチェーンおよびサプライモジュール向けの読み取り専用 REST アクセス。

* **ブリッジ REST エンドポイント** — ブリッジモジュール向けの読み取り専用 HTTP クエリエンドポイントで、gRPC に加えて標準 REST 経由でブリッジ状態を公開します。
* **バーン REST エンドポイント** — バーンモジュール向けの読み取り専用 HTTP クエリエンドポイントで、手数料分配およびサプライデータを標準 REST 経由でクエリ可能にします。

## v3.1.76 — SVM ツールチェーンのモダナイゼーション

**リリースの焦点:** Solana 仮想マシンの互換性リフレッシュ。

* **現行ツールチェーンのプログラムサポート** — SVM 実行をモダナイズし、現行の Solana ツールチェーンでビルドされたプログラムが QoreChain SVM ランタイム上で動作するようにしました。

## v3.1.75 — SVM JSON-RPC をデフォルトで有効化

**リリースの焦点:** 箱から出してすぐに使える Solana 互換 RPC。

* **Solana 互換 JSON-RPC** — SVM JSON-RPC サーバーがデフォルトで有効になり（ポート **8899**）、ノードとともに自動的に起動するようになり、SVM ツール向けの Solana 互換 RPC インターフェースを提供します。

## v3.1.74 — ロールアッププロファイルプリセット

**リリースの焦点:** Rollup Development Kit のユーザビリティと決済。

* **プロファイルプリセットの適用** — ロールアップ作成時に、選択したプロファイルのプリセット（DeFi、ゲーミング、NFT、エンタープライズ、または完全カスタム）が適用されるようになり、新しいロールアップはそのユースケースに適したデフォルト値を継承します。
* **オプティミスティック決済** — オプティミスティック決済パス（バッチ送信とチャレンジ）がエンドツーエンドで動作します。

## v3.1.73 — ポスト量子ハッシュベースライン

**リリースの焦点:** デフォルトのポスト量子暗号ベースラインの完成。

* **SHAKE-256 デフォルトハッシュ** — SHAKE-256（SHA-3 ファミリー）をデフォルトのアプリケーションハッシュとして採用し、**ML-DSA-87 (Dilithium-5)** 署名、**ML-KEM-1024** 鍵カプセル化、**SHAKE-256** ハッシュからなるデフォルトのポスト量子ベースラインを完成させました。

## v3.1.72 — 安定性 & メンテナンス

**リリースの焦点:** 定常的な安定化とビルドパイプラインのメンテナンス。

* **安定性の向上** — 外部から見える動作の変更を伴わない、内部の安定性、依存関係、およびビルドパイプラインのメンテナンス。

## v3.1.71 — PQC ハイブリッド署名をデフォルトで強制

**リリースの焦点:** Cosmos トランザクションパスでデフォルト有効のポスト量子セキュリティ。

* **ハイブリッド署名をデフォルトで必須化** — ポスト量子ハイブリッド署名が Cosmos トランザクションパスでデフォルトで強制されるようになりました。各トランザクションは、古典的な **secp256k1** 署名とともに、ポスト量子 **ML-DSA-87 (Dilithium-5)** 署名を保持します。
* **ガバナンス制御による強制** — 強制モードはガバナンス制御のままで、デフォルトは **required** に設定されています。

## v3.1.70 — プロダクションハードニング

**リリースの焦点:** 稼働中のメインネット向けのプロダクションハードニングとコンセンサス最適化。

* **PRISM コンセンサス最適化** — 稼働中のネットワーク条件下での適応的パラメータチューニングのための PRISM 強化学習最適化レイヤーの継続的改善（サーキットブレーカー安全制御付き）。
* **パフォーマンスと安定性** — バリデーターおよびフルノード全体にわたるスループット、レイテンシ、リソース使用量の改善。
* **運用ツール** — メインネットオペレーター向けのモニタリング、クエリ、ノード運用のエルゴノミクスを改善。
* **トークノミクス v2.1 への整合** — 手数料分配と発行メカニズムを、固定供給・有限発行の経済モデルに整合させました。

## v3.0.0 — メインネットジェネシス

**リリースの焦点:** メインネットローンチとトークン生成イベント。

* **メインネットジェネシス** — QoreChain メインネット（`qorechain-vladi`、EVM チェーン ID 9801）が **2026 年 6 月 7 日** にローンチし、ジェネシス時にトークン生成イベント（TGE）が行われました。
* **5 分割手数料スプリット** — バリデーター、バーン、トレジャリー、ステーカー、ライトノードにまたがるプロトコル手数料分配（**37 / 30 / 20 / 10 / 3**）。専用のライトノードシェアを追加しました。
* **オンチェーン AMM** — オンチェーンの流動性プールとスワップのためのネイティブな自動マーケットメーカーモジュール（`x/amm`）。
* **チェーンライセンシング** — プロトコル権限を登録・管理するためのオンチェーンライセンスモジュール（`x/license`）。
* **ハードニングされた決済パラダイム** — RDK 決済モードを optimistic、zk、based、sovereign として確定しました。

## v1.3.0 — Rollup Development Kit

**リリースの焦点:** ソブリンおよび共有セキュリティロールアップデプロイメントのためのネイティブロールアップインフラストラクチャ。

* **x/rdk モジュール** — 4 つの決済パラダイム（optimistic、zk、based、sovereign）を備えた完全な Rollup Development Kit
* **5 つのプリセットプロファイル** — DeFi、ゲーミング、NFT、エンタープライズ、完全カスタムのユースケース向けの事前構成済みロールアップテンプレート
* **ネイティブデータ可用性** — ブロブストレージ、保持管理、プルーニングライフサイクルを備えたオンチェーン DA レイヤー
* **EndBlocker 自動ファイナライゼーション** — チャレンジウィンドウの期限切れ時に、オペレーターの介入なしでバッチを自動的にファイナライズ
* **AI 支援によるプロファイル選択** — 意図したユースケースに基づいて最適なロールアップ構成を推奨する `suggest-profile` クエリ
* **マルチレイヤー統合** — ロールアップはマルチレイヤーアーキテクチャ内でレイヤーとして登録され、ルーティング、アンカリング、チャレンジメカニクスを継承
* **バンクエスクローライフサイクル** — オペレーターのステークはロールアップ運用中にエスクローで保持され、クリーンシャットダウン時に解放されるか、スラッシング時に没収されます

## v1.2.0 — IBC & ブリッジ

**リリースの焦点:** クロスチェーン接続性と高度なアカウント抽象化。

* **25 のクロスチェーン接続** — 外部ネットワークへの 8 つの IBC チャネルと 17 の QoreChain Bridge (QCB) 接続
* **x/babylon モジュール** — Bitcoin 保有者が QoreChain のステーキングセキュリティに参加できるようにする BTC リステーキング統合
* **x/abstractaccount モジュール** — プログラム可能な支出ルール、セッションキー、カスタム認証ロジックを備えたスマートアカウントフレームワーク
* **x/fairblock モジュール** — MEV 耐性のあるトランザクション暗号化のためのしきい値 ID ベース暗号化（tIBE）
* **x/gasabstraction モジュール** — ネイティブ QOR、IBC ブリッジ経由の USDC、IBC ブリッジ経由の ATOM をサポートするマルチトークンガス支払い
* **5 レーン TX 優先順位付け** — 優先順位順に並べられたトランザクションレーン: system、governance、staking、bridge、general
* **IBC リレイヤー設定** — サポートされるすべての IBC チャネル向けの事前構成済みリレイヤーセットアップ
* **ブリッジからバーンへの統合** — ブリッジ手数料はバーンモジュールの手数料分配を通じてルーティングされます

## v1.1.0 — PQC ハイブリッド署名

**リリースの焦点:** ポスト量子暗号セキュリティとアルゴリズムアジリティ。

* **secp256k1 (ECDSA) + ML-DSA-87 デュアル署名** — すべてのトランザクションは古典署名とポスト量子署名の両方を保持し、AnteHandler チェーンで検証されます
* **3 つの強制モード** — 構成可能なハイブリッド署名の強制: off（モード 0）、permissive（モード 1、PQC はオプション）、mandatory（モード 2、PQC は必須）
* **自動登録** — PQC 公開鍵は最初のハイブリッドトランザクション時に自動的に登録され、別途の登録ステップが不要になります
* **SHAKE-256 ハッシュ基盤** — すべての PQC 関連のハッシュ操作は、量子耐性のあるアドレス導出のために SHAKE-256（SHA-3 ファミリー）を使用します
* **TEE アテステーションインターフェース** — PQC 鍵生成の整合性を証明するための Trusted Execution Environment アテステーションサポート
* **アルゴリズムアジリティフレームワーク** — 将来の PQC アルゴリズムをチェーンアップグレードなしにガバナンス経由で追加できるプラグ可能なアルゴリズムレジストリ

## v1.0.0 — ジェネシス（トークノミクスエンジン）

**リリースの焦点:** 完全なトークノミクス、マルチ VM 実行、AI 支援運用を備えた初期プロトコルローンチ。

* **x/burn モジュール** — バリデーター、バーン、トレジャリー、ステーカーにまたがる 4 分割分配を備えたマルチチャネル手数料バーンメカニズム
* **x/xqore モジュール** — 段階的な早期アンロックペナルティと PvP リベース再分配を備えたガバナンスステーキングデリバティブ
* **x/inflation モジュール** — 有限発行経済モデルに支配される、年次減衰を伴うエポックベースの発行
* **PRISM コンセンサスレイヤー** — サーキットブレーカー安全制御を備えた、動的チェーンパラメータチューニングのための強化学習最適化（PPO）
* **トリプルプール CPoS** — レピュテーションスコアで重み付けされた Emerald、Sapphire、Ruby バリデータープールを備えた Classified Proof-of-Stake
* **QDRW ガバナンス** — プール間の報酬分配に対するガバナンス承認済みの調整を可能にする Dynamic Reward Weighting システム
* **EVM + CosmWasm + SVM ランタイム** — 3 つの同時実行環境: QoreChain EVM Engine、CosmWasm スマートコントラクト、Solana 仮想マシン
* **クロス VM ブリッジ** — 単一ブロック内での EVM、CosmWasm、SVM ランタイム間のメッセージパッシングと資産転送
* **ポスト量子暗号** — 高性能 PQC ライブラリに支えられた量子耐性のある署名
* **QCAI** — 不正検知、手数料見積もり、ネットワーク最適化のための、オプションのオフチェーンサイドカーを備えたオンチェーンヒューリスティック分析
* **コンテナ化デプロイメント** — サイドカーサービスとブロックインデクサーを備えた完全なマルチバリデーターテストネットデプロイメント
* **ブロックインデクサー** — 履歴クエリと分析のための永続ストレージを備えたブロックリスナー
