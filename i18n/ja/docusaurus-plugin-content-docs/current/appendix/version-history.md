# バージョン履歴

QoreChain の公開バージョン履歴です。最新リリースは **v3.1.82** で、メインネット **`qorechain-vladi`**（EVM チェーン ID **9801**、2026年6月7日より稼働）上で動作しています。テストネット **`qorechain-diana`**（EVM チェーン ID **9800**）はプレリリースビルドを追跡しています。

:::note
以下のエントリは機能の概要をまとめたものです。初期の `v1.x` のエントリは、メインネットに先行したテストネットリリース系列の歴史的記録として保持されています。
:::

---

## v3.1.82 — SVM 上のネイティブ QOR 稼働 + インテグレーター支援（現行メインネットリリース）

**リリースの焦点:** 両ネットワークで稼働する SVM ネイティブ QOR 統合と、取引所やインテグレーターが接続に必要とするすべての要素。

* **3 つのインターフェースすべてで統一ネイティブ QOR 残高が稼働** — SVM 統合（v3.1.81）がメインネットとテストネットの両方で稼働していることが確認されました。同一アカウントが単一の残高を保持し、Cosmos 上では `uqor`（6 桁の小数）、EVM 上では wei 形式の 18 桁の小数、Solana 互換インターフェース上では lamports（9 桁の小数、1 uqor = 1,000 lamports）として参照できます。
* **検証済みの公開エンドポイント** — 両ネットワークにおけるコンセンサス RPC、REST、EVM JSON-RPC、SVM JSON-RPC の公開 HTTPS エンドポイントに加え、公開[ブロックエクスプローラー](https://explore.qore.network)を提供。詳細は[ネットワーク](/appendix/networks)を参照してください。
* **ダウンロード** — バージョン付きノードバイナリのバンドル、メインネットのジェネシス、最新のチェーンデータスナップショット（SHA-256 チェックサム付き）を [download.qore.host](https://download.qore.host) で公開。
* **クライアントスタック全体での決定論的ポスト量子署名** — `@qorechain/pqc` 0.1.1 は 6 つすべての言語バインディングで ML-DSA-87 を決定論的（FIPS-204 §3.4）に署名し、チェーンが受理する形式と一致します。`@qorechain/wallet-adapter` 0.1.2 はこれを基盤としてハイブリッドトランザクション署名を実現します。
* **インテグレーターガイド** — 3 つのインターフェースにまたがる入金、出金、ノード運用をカバーする新しい[取引所・インテグレーターガイド](/developer-guide/exchange-integration)を追加。

## v3.1.81 — SVM ネイティブ QOR 統合

**リリースの焦点:** Solana 互換インターフェース上でネイティブ QOR をファーストクラスの資産として扱う。

* **SVM 上のネイティブ QOR** — SVM ランタイムは、SVM 専用の別残高を管理するのではなく、アカウントのネイティブ QOR 残高を直接（lamports 単位で）公開するようになりました。`getBalance` と `getSignaturesForAddress` はネイティブ資金に対して動作し、System Program の送金はネイティブ QOR を移動します。
* **SVM アドレスマッピング** — アカウントの SVM アドレスは、その 20 バイトのアカウントバイト列から導出されます(右側を 32 バイトまでパディングし base58 エンコード)。これにより、1 つの鍵の Cosmos、EVM、SVM アドレスはすべて同じ資金を指します。

## v3.1.80 — マルチレイヤー・ステートアンカークエリ

**リリースの焦点:** ロールアップ向けの、読み取り可能かつオフライン検証可能な決済アンカー。

* **アンカー読み取りクエリ** — `x/multilayer` クエリサービスが `Anchor`（レイヤーの最新ステートアンカー）と `Anchors`（レイヤーのアンカー履歴）を公開するようになり、クライアントはレイヤーの決済アンカーを取得して独立に検証できます。
* **マルチレイヤー向け REST ゲートウェイ** — すべてのマルチレイヤークエリ（`params`、`layers`、`layers/{layer_id}`、`anchor/{layer_id}`、`anchors/{layer_id}`、`routing-stats`）が gRPC に加えて REST でも利用可能になりました。
* **量子安全な決済レシートのブロック解除** — 各アンカーは正規化されたフィールドに対する **ML-DSA-87 (Dilithium-5)** 署名を保持し、Rollup Development Kit のオフライン決済レシート検証のオンチェーン基盤を提供します。

## v3.1.79 — ブリッジネットワーク向けバリデーター自動プロビジョニング

**リリースの焦点:** ライセンスを保有するバリデーターによる、接続ネットワークへのターンキー参加。

* **ネットワークドライバーフレームワーク** — 宣言的なドライバーフレームワークにより、該当する `validator_<chain>`（または `qcb_bridge`）ライセンスを保有する QoreChain バリデーターは、対応する外部ネットワーククライアントを同じノード上で QoreChain のオーケストレーションの下でプロビジョニング、設定、実行させることができます — ただしライセンスが有効化された後に限ります。
* **37 のブリッジネットワークすべてに対応するドライバー** — すべての接続ネットワークをカバーし、参加モデル（パーミッションレスバリデーター、上限付き/選出/承認制、L2 フルノード、非ステーキング/トラストリスト役割）で分類されています。外部ネットワークのステークと署名鍵はネットワークごとにオペレーターが用意し、QoreChain はフレームワークと強制されるライセンスゲートを提供します。

## v3.1.78 — デプロイ前レディネス

**リリースの焦点:** ウォレット、ブリッジ、IBC、ライセンシングがローンチ時点ですべて機能 — デプロイ後のガバナンスは不要。

* **トラストレスなデプロイ後ブリッジ有効化** — `bridge_admin` 鍵（または `qcb_bridge` ライセンス保有者）は、署名済みトランザクション 1 件（`tx bridge update-chain-config` / `set-verifier-bootstrap`）で任意の接続チェーンのブリッジを有効化できます — コントラクトアドレス、確認数、アーキテクチャ、ステータス、有効なベリファイア、ベリファイアのトラストルートを設定でき、ガバナンス提案やチェーンアップグレードは不要です。
* **バリデーターネットワークのライセンスゲート** — オーケストレーターは、外部ネットワーククライアントを起動する前に `validator_<chain>` / `qcb_bridge` ライセンスを強制（フェイルクローズド）するようになりました。
* **ウォレット統合パッケージ** — `@qorechain/wallet-adapter` と `@qorechain/connect` を npm に公開（v0.1.0）。ワンコールでの MetaMask ネットワーク登録（EIP-3085、EVM レール上の **18 桁小数**のネイティブ QOR）と Keplr のガス価格設定を追加。
* **IBC ターンキーリレイヤー** — 8 つの IBC カウンターパーティ向けのすぐに実行可能なリレイヤー設定とチャネルブートストラップツールにより、個別のセットアップなしにデプロイ後にチャネルが立ち上がります。

## v3.1.77 — ブリッジ & バーン REST エンドポイント

* **バーン REST エンドポイント** — バーンモジュールの読み取り専用 HTTP クエリエンドポイントにより、手数料分配とサプライデータを標準 REST でクエリ可能に。

## v3.1.74 — ロールアッププロファイルプリセット

## v3.1.73 — ポスト量子ハッシュベースライン

* **SHAKE-256 をデフォルトハッシュに採用** — SHAKE-256（SHA-3 ファミリー）がデフォルトのアプリケーションハッシュとして採用され、**ML-DSA-87 (Dilithium-5)** 署名、**ML-KEM-1024** 鍵カプセル化、**SHAKE-256** ハッシュというデフォルトのポスト量子ベースラインが完成しました。

## v3.1.71 — PQC ハイブリッド署名をデフォルトで強制

**リリースの焦点:** Cosmos トランザクションパスでポスト量子セキュリティをデフォルトで有効化。

* **ハイブリッド署名をデフォルトで必須化** — Cosmos トランザクションパスでポスト量子ハイブリッド署名がデフォルトで強制されるようになりました。各トランザクションは、古典的な **secp256k1** 署名と並んで、ポスト量子 **ML-DSA-87 (Dilithium-5)** 署名を保持します。
* **ガバナンスによる強制制御** — 強制モードは引き続きガバナンスで制御され、デフォルトは **required** に設定されています。

* **Tokenomics v2.1 との整合** — 手数料分配とエミッションのメカニズムを、固定供給・有限エミッションの経済モデルに整合。

## v3.0.0 — メインネットジェネシス

## v1.3.0 — Rollup Development Kit

**リリースの焦点:** ソブリンおよび共有セキュリティのロールアップデプロイメントのためのネイティブロールアップインフラストラクチャ。

* **x/rdk モジュール** — 4 つの決済パラダイム（オプティミスティック、zk、ベースド、ソブリン）を備えた完全な Rollup Development Kit
* **5 つのプリセットプロファイル** — DeFi、ゲーミング、NFT、エンタープライズ、完全カスタムのユースケース向けに事前構成されたロールアップテンプレート
* **ネイティブデータ可用性** — ブロブストレージ、保持管理、プルーニングライフサイクルを備えたオンチェーン DA レイヤー
* **EndBlocker による自動ファイナライゼーション** — チャレンジウィンドウの満了時にバッチを自動的にファイナライズ。オペレーターの介入は不要
* **AI 支援のプロファイル選択** — 想定ユースケースに基づいて最適なロールアップ構成を推奨する `suggest-profile` クエリ
* **マルチレイヤー統合** — ロールアップはマルチレイヤーアーキテクチャのレイヤーとして登録され、ルーティング、アンカリング、チャレンジのメカニズムを継承
* **バンクエスクローライフサイクル** — オペレーターのステークはロールアップ運用中エスクローに保持され、正常なシャットダウン時に解放、スラッシング時には没収

## v1.2.0 — IBC & ブリッジ

**リリースの焦点:** クロスチェーン接続性と高度なアカウント抽象化。

* **25 のクロスチェーン接続** — 外部ネットワークへの 8 つの IBC チャネルと 17 の QoreChain Bridge (QCB) 接続
* **x/babylon モジュール** — Bitcoin 保有者が QoreChain のステーキングセキュリティに参加できる BTC リステーキング統合
* **x/abstractaccount モジュール** — プログラマブルな支出ルール、セッションキー、カスタム認証ロジックを備えたスマートアカウントフレームワーク
* **x/fairblock モジュール** — MEV 耐性のあるトランザクション暗号化のためのしきい値 ID ベース暗号（tIBE）
* **x/gasabstraction モジュール** — ネイティブ QOR、IBC ブリッジされた USDC、IBC ブリッジされた ATOM に対応するマルチトークンガス支払い
* **5 レーンのトランザクション優先度付け** — 優先度順のトランザクションレーン: システム、ガバナンス、ステーキング、ブリッジ、一般
* **IBC リレイヤー設定** — サポートされるすべての IBC チャネル向けに事前構成されたリレイヤーセットアップ
* **ブリッジとバーンの統合** — ブリッジ手数料はバーンモジュールの手数料分配を通じてルーティング

## v1.1.0 — PQC ハイブリッド署名

**リリースの焦点:** ポスト量子暗号セキュリティとアルゴリズムアジリティ。

* **secp256k1 (ECDSA) + ML-DSA-87 のデュアル署名** — すべてのトランザクションが古典署名とポスト量子署名の両方を保持し、AnteHandler チェーンで検証
* **3 つの強制モード** — 設定可能なハイブリッド署名の強制: オフ（モード 0）、パーミッシブ（モード 1、PQC は任意）、必須（モード 2、PQC 必須）
* **自動登録** — PQC 公開鍵は最初のハイブリッドトランザクションで自動的に登録され、別途の登録ステップが不要に
* **SHAKE-256 ハッシュ基盤** — PQC 関連のすべてのハッシュ操作は、量子耐性のあるアドレス導出のために SHAKE-256（SHA-3 ファミリー）を使用
* **TEE アテステーションインターフェース** — PQC 鍵生成の完全性を証明するための Trusted Execution Environment アテステーションのサポート
* **アルゴリズムアジリティフレームワーク** — チェーンアップグレードなしにガバナンス経由で将来の PQC アルゴリズムを追加できるプラガブルなアルゴリズムレジストリ

## v1.0.0 — ジェネシス（トークノミクスエンジン）

**リリースの焦点:** 完全なトークノミクス、マルチ VM 実行、AI 支援運用を備えた初期プロトコルローンチ。

* **x/burn モジュール** — バリデーター、バーン、トレジャリー、ステーカーの 4 分割配分によるマルチチャネル手数料バーンメカニズム
* **x/xqore モジュール** — 段階的な早期アンロックペナルティと PvP リベース再分配を備えたガバナンスステーキングデリバティブ
* **x/inflation モジュール** — 有限エミッションの経済モデルに準拠した、年次減衰付きのエポックベースエミッション
* **PRISM コンセンサスレイヤー** — サーキットブレーカーによる安全制御を備えた、動的チェーンパラメータチューニングのための強化学習最適化（PPO）
* **トリプルプール CPoS** — レピュテーションスコアで重み付けされた Emerald、Sapphire、Ruby のバリデータープールを持つ Classified Proof-of-Stake
* **QDRW ガバナンス** — プール間の報酬分配をガバナンス承認で調整できる Dynamic Reward Weighting システム
* **EVM + CosmWasm + SVM ランタイム** — 3 つの並行実行環境: QoreChain EVM Engine、CosmWasm スマートコントラクト、Solana Virtual Machine
* **クロス VM ブリッジ** — 単一ブロック内での EVM、CosmWasm、SVM ランタイム間のメッセージパッシングと資産移転
* **ポスト量子暗号** — 高性能 PQC ライブラリに支えられた量子耐性署名
* **QCAI** — 不正検知、手数料推定、ネットワーク最適化のための、オプションのオフチェーンサイドカー付きオンチェーンヒューリスティック分析
* **コンテナ化デプロイメント** — サイドカーサービスとブロックインデクサーを備えた完全なマルチバリデーターテストネットデプロイメント
* **ブロックインデクサー** — 履歴クエリと分析のための永続ストレージ付きブロックリスナー
