# ステーキングとバリデーター

**バリデーター**ページでは、ネットワークのバリデーターを確認し、そこへ委任することで QOR をステーキングできます。委任はネットワークの安全確保に役立ち、ステーキング報酬を獲得できます。委任と報酬の背景にある概念については、[ステーキングと委任](/user-guide/staking-and-delegation)をご覧ください。

ステーキングするにはウォレットを接続してください。詳しくは[概要とはじめに](/dashboard/overview#connect-your-wallet)をご覧ください。

## バリデーターを確認する

このページを開くと、アクティブなバリデーター数、ボンドされた QOR の合計、平均コミッション、平均アップタイムのサマリーカードが表示されます。その下にはバリデーターのリストがあります。各バリデーターには次の内容が表示されます。

- **ランク**とバリデーターの**モニカー**（名前）、そのアドレスとコピーボタン。
- **投票力** — バリデーターのボンド済みステークと、全体に占めるその割合。
- **コミッション** — バリデーターが報酬から差し引く割合。
- **APY** — 委任に対する年間利回りの見積もり。
- **ステータス** — 例：active または jailed。
- リージョン、アップタイム、提案したブロック数、ソフトウェアバージョン、最終確認時刻などの運用上の詳細。

検索ボックスで、バリデーター名またはアドレスによりリストを絞り込めます。

## バリデーターを選ぶ

委任先のバリデーターを選ぶ際は、次の点を考慮してください。

- **コミッション** — 料率が低いほど自分に残る報酬は多くなりますが、持続可能な運用者には妥当な取り分が必要です。
- **アップタイムとステータス** — アップタイムが優れたアクティブなバリデーターを選びましょう。jailed のバリデーターは報酬を得ていません。
- **投票力** — 複数のバリデーターにステークを分散させることは、分散化を支えます。

## 委任、再委任、請求

ウォレットを接続すると、次のことができます。

- バリデーターに QOR を**委任**して報酬の獲得を開始する。
- ステークをあるバリデーターから別のバリデーターへ**再委任**する。
- ステークの引き出しを開始するために**委任解除**する。
- 委任から発生した**報酬を請求**する。

:::note アンボンディング期間
委任解除した QOR は、再び使えるようになるまでアンボンディング期間を経ます。その間は報酬を得られません。詳しくは[ステーキングと委任](/user-guide/staking-and-delegation)をご覧ください。
:::
